Aerospike parameters

This section describes the Aerospike parameters.

Namespaces

The Aerospike parameters are related to the these namespaces:

dsc

This is the default namespace and is used :

  • On SM, all of the records are stored in this namespace.
  • On ME, any record that is not assigned to other namespaces is stored in this namespace.

Two-way XDR replication is automatically enabled for this namespace by configuring the Mate Site parameter between the two MEs or two IGs . The Mate Site parameter is defined in the SPS section of the Inputs.xlsx file. However, for SMs, this namespace is always replicated, and the mate system is used for SM only for replication related to User and Role Management data (ckey/cmdb installed with geo-redundancy), not to enable XDR replication.

dsclocal

Only ME uses it. This namesapce stores the records provisioned by SM that are not subscriber related (that is, service definition data).

XDR cannot be enabled for this namespace.

dscglobal

Only ME uses it. This namespace stores N-way replicated records (that is, global mapping tables).

N-way XDR replication across all MEs is automatically enabled for this namespace.

spspd

Only ME uses one or several namespaces for the partitions data. These namespaces store subscriber/devices related data.

Two-way XDR replication between the two MEs associated with the partition is automatically enabled for each of these namespaces.

spsundologs

SM and ME use it. This namespace stores all the transactions' undo logs information that is used to revert a transaction if a rollback occurs.

This is a memory only namespace.

XDR cannot be enabled for this namespace.

Aerospike namespace parameters

Aerospike namespace’s filesize and memorysize parameters are configured in accordance with:

  • The filesize and memorysize parameters defined for a specified namespace correlate.
  • The values for the memorysize parameters must be defined and cannot be empty.
  • The values for the filesize are optional.
  • You must update these parameters for a live deployment.
  • You must contact Nokia Professional Services to determine the correct values for these parameters.
  • The default values are for the lab trial setups only.
For the example of the aerospike configuration for geo-replicarion, see Topology parameters.
