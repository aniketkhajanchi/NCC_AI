Credit-Control-Failure-Handling (CCFH) AVP support in the NCC Gy interface

The credit-control client uses the information in CCFH AVP to handle the credit control server failure.

NCC supports the CCFH AVP in Credit-Control-Answer (CCA) command with the values TERMINATE(0), CONTINUE (1), RETRY_AND_TERMINATE (2), and IGNORE (3).

Note:
CCFH AVP is not applicable for IEC calls.

NCC performs the following logic based on the values set:

  • If the value of CCFH is configured as IGNORE in active RSV, then CCFH AVP is not returned in CCA.

  • If the value is NULL, then CCFH AVP is returned in CCA as TERMINATE.
