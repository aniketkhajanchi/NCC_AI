NCC scheduled backup

The scheduled backup of NCC Aerospike database and CSDC components are not enabled by default. After installing NCC, scheduled backup of these components may be enabled. For more information, see Scheduled backups

The default schedule for scheduled backup cronJobs for Aerospike database sps-admincli and CSDC are set to the same value.
0 2 * * *

This indicates the scheduled backup will begin daily at 2am.

Update schedules for scheduled backups

To update the cronJob schedules defined at install time for either of the NCC Backup and restore policy created by NCC, a helm upgrade must be performed. This provides the new values for the cronJob schedules and reuses old values for all other properties. Then, scheduled backups must be re-enabled. For more information regarding enabling scheduled backups, see Scheduled backups.

As an example, the following helm upgrade command changes the schedules in the policy to run the backup for the Aerospike database daily at 3am. As autoEnableCron and autoUpdateCron are set to true, the changes will be immediately effective.

helm upgrade ig-admincli admincli-chart-*.tgz  -n $NAMESPACE --set cbur.cronSpec="0 3 * * *",cbur.autoEnableCron=true,autoUpdateCron=true --reuse-values --wait --timeout=1200s

In another example, the following helm upgrade changes the schedules in the policy to run the backup for CSDC daily at 3am.

helm upgrade ig-csdc csdc-chart-*.tgz  -n $NAMESPACE --set cbur.cronSpec="0 3 * * *" --reuse-values --wait --timeout=1200s

To make this change for CSDC effective, the scheduled backups have to be enabled. If the scheduled backups were already enabled before doing this upgrade, they must be disabled and re-enabled for the changes to the policy to have effect.
