SS7 to NCC-DH: diameter integration

This section describes the configuration needed for the DH service to make the initial connection with NCC. You can edit the configurations via the OSP UI or the SS7 Manager GUI. Details are found in the NCC SS7 User Guide.

Service object

This object is required to be configured to make the DH service UP and running.

Diameter object

This object is used to store the diameter configuration such as origin-host, origin-realm, destination-host, destination-realm, CCR max retransmission allowed, and the Service context ID used in various diameter AVP. This object is preconfigured with default values.

Note:
The value for the Diam Configuration Name must be diam1. It is mandatory to create one instance of Diameter Configuration named diam1. This configuration is used to fill CER and DWR. Origin Host and Origin Realm should be configured with your system-specific values. Destination Host and Destination Realm should match the NCC configuration.

Server List object

This object lists the different type of servers available for the service and their generic behaviors. One tuple list is provided for a Diameter server type. The object is preconfigured with a Type value of SDIAM.

Server object

This object describes a server diameter. The physical view of the server (IP address, port) is implemented in the object (TCP_CFG). Common attributes (time-out, support) are stored in this object. Ideally, four primary and four secondary servers for four systems are preconfigured in the server object for reference.

TCP_CFG object

This object manages the TCP-IP information of a server (IP address, port). A TCP_CFG object is linked with a Server object that manages the protocol used to exchange data. A TCP array is built in each SEP memory. This array is local to the SEP to allow it to manage calls faster. Ideally, this object is preconfigured with four primary and four secondary TCP nodes, which are linked with their respective Server object.

SEP Server object

This object is used to link the server to the DH service SEP. One instance contains a SEP name and a server name (with status diameter). Several instances can be created in order to link multiple servers to a SEP. When the TCP array is created or refreshed, the SEP_SERVER_Interconnection object (SEPSRV) is used to know which server is allowed to be used for this SEP.

Ideally all the DH service SEP of all the systems are pre-configured and associated with one primary and one secondary preconfigured server with their respected systems. If new server and TCP configuration objects are recreated, the SEP Server object can be configured with the association of system-wise DH service SEP and with their respective server.

AVP List object

This object is used to contain the list of AVPs and their values which are sent in diameter messages for CER, DWR, and CCR. These values must be configured as per defined agreement and compatibility with NCC and NCC-SS7 nodes.
