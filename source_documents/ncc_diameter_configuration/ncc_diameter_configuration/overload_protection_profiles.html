Overload protection profiles

System overload protection profiles capture the overload parameters for queue levels, shedding percentages, etc. Multiple overload protection profiles may be defined and distributed using the SM.

NCC supports a separate profile for each monitored resource such as Diameter message latency, and Diameter message count, as well as a profile for the shedding actions to be applied when the system goes into overload.

The following overload protection profiles can be configured:
  • Overload shedding profile
  • CPU usage threshold profile
  • Message count threshold profile
  • Latency threshold profile

To configure an overload protection profile, in SM GUI navigate to Connectivity → ☰ → OVERLOAD → Overload Protection. Select PROFILES tab. Click + and select a profile to be created. Configure the parameters of the selected overload protection profile.
