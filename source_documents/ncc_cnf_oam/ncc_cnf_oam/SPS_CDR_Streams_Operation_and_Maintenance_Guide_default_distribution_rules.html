Default distribution rules

If no distribution rules are configured, only file output is enabled. The file characteristics are taken from the node parameters as described in this table. The same default values apply if the configured file distribution rule does not define some of the characteristics.

Table: Default distribution rules for file output
Default behavior Node parameter
Output identifier

None. Empty string is used to replace possible output identifier in the file name mask.
Output directory CG Distributor Node / TargetDirectory (connection parameter)
Maximum number of records per file CG Mapper Node / RecordsPerBatch
Maximum number of seconds to wait before closing a file CG Mapper Node / SecondsPerBatch
Sequence ID All output files share the same set of sequence numbers.

If a Kafka distribution rule does not define the topic name, the default topic name in the node parameters is used, as described in this table.

Table: Default distribution rules for Kafka output
Default behavior Node parameter
Topic name CG Kafka Distributor Node / KafkaDefaultTopicName
Failure log directory

CG Log Distributor Node / SFTPRemoteDirectory
