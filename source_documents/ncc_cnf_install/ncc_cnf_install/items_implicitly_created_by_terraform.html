Items implicitly created by Terraform

Once the Terraform apply is completed, it creates these Kubernetes objects that can be used by NCC.

  • Restricted PSP to enforce this on pods installed on this cluster. This is the default policy applied to all installed on components. This is implicitly applied to all pods installed unless they specifically use a privileged role.
  • Privileges PSP for pods that need and explicitly declare they need higher privileges.
  • AWS specific storage classes which could be used in NCC. (https://kubernetes.io/docs/concepts/storage/storage-classes/#aws-ebs). NCC has High Throughput Storage Class in the Inputs.xlsx file which is used by Aerospike, if configured.
  • Some role binding needed for essential kube-system pods.
  • ncc-node-critical priority class which is used by NCC critical pods which need to be placed on Kubernetes nodes. This is used by Fluentd and Prometheus node exporter daemonset pods.
  • Deploys cluster node auto scaler for AWS EKS.
  • Generates a <your-cluster>.conf file in the local directory which is can be used by the kubectl/helm command to access your cluster.
