Disabling mutual TLS for intrasystem Aerospike communication

This procedure provides instructions on how to disable intrasystem mutual TLS (mTLS) communication on a particular system that occurs as server-to-server between aerospike pods, and as client-to-server between aerospike pods (servers) and the other pods (clients). This procedure is used post-installation when NCC is already deployed. For instructions on how to configure mTLS for intrasystem Aerospike communication during installation, see the Installation and Upgrade Guide in the NCC documentation set.

  1. Set the aerospike.network.intraCluster.exposeInsure parameter in the values.yaml file to true.
  2. Run helm upgrade with -f option passing the input values.yaml file to apply the changes.
  3. Restart the aerospike server pods to upgrade the servers.
  4. Turn off mTLS for clients by setting the global.aerospike.mtls.client.enabled parameter to false in the values.yaml file.
  5. Run helm upgrade with -f option passing the input values.yaml file to apply the changes.
  6. Set the parameters in the values.yaml file as follows:
    • aerospike.network.intraCluster.exposeInsure to true

    • aerospike.network.intraCluster.mtls.enabled to false

  7. Run helm upgrade with -f option passing the input values.yaml file to apply the changes.
  8. Restart the aerospike server pods to upgrade the servers.
    Note:
    Aerospike communication uses a certificate for the aerospike-credential secret. When the current certificate expires or is rotated, the ongoing client-to-server connections are not affected. However, new connections require the updated certificate. To apply the updated certificate, the application and aerospike pods must restart.
