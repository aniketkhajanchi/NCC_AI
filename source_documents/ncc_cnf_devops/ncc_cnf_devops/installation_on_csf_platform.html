Installation on CSF Platform

Note:

CSF Platform has RBAC enabled for different User/Namespace, and does not allow containers to run as the root user. Consequently, the test execution container is forced to run with UID:10000 and GID:10000.

If user Docker images are provided, ensure that a non-root user is added with UID:10000 and GID:10000 and that all tests can be executed with UID:10000 and GID:10000.

Follow the CSF Platform UI prompts and customize the default values as necessary, for example, resources and proxy setting.

Note:

If ingress is enabled (default), CCTF hostname and CCTF Jenkins hostname should be configured in the following format:

  • CCTF hostname:

    <tenant name>-<instance name>-cctf-<https port>.<namespace>.<CSF Platform FQDN>

  • CCTF Jenkins hostname:

    <tenant name>-<instance name>-jenkins-8080.<namespace>.<CSF Platform FQDN>

After a successful installation, all pods should be up and running after a short duration. Users should be able find the CCTF URL from instance endpoints if ingress is enabled. If not, add bindings manually, as follows:
