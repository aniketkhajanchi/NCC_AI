Rule, variable, and rule system version provisioning

Rule authoring management

Within an NCC system there is one active rule system at any given time, which contains the rules and variables that are currently in effect. NCC allows the creation of multiple Rules System Versions (RSV) to support having rule systems for different purposes, such as in a lab where different scenarios are being explored, and to maintain previous versions of rule systems as they evolve.
Note:
If an RSV is more than 5MB in size, it is not possible to perform an export of the RSV using a Chrome browser. This is a limitation of the browser. Nokia suggests that the Mozilla Firefox browser is used for an operation of this magnitude. To do so, the default setting in Mozilla Firefox must be changed:
  • Navigate to "about:config" in the Firefox address bar.
  • Search for "dom.storage.default_quota", which is set to 5120 by default.
  • Set the value to 7000 (to export an RSV of 7MB) or an appropriate size that is higher to be able to export the RSV.
NCC supports functionality through rules to create a condition-based decision with associated triggers with one or more Rule Groups. Triggers are a message type (for example, database calls - Gy CCR). Rules and rule variables can be configured by using the rule text editors or by importing an existing RSV. Rule groups may be re-ordered within a trigger or re-order rules within a rule group by dragging the rules up or down. These options are available to manage an RSV:

  • Edit

    Available when the RSV is in a Draft state. If the RSV is in a Release state, it may be reverted to Draft if it has not been distributed to an ME subsystem.

  • Delete

    Available when the RSV is in a Draft state. If the RSV is in a Release state, it may be reverted to Draft if it has not been distributed to an ME subsystem.

  • Import

    To Import Rule Group, tap Add Rule Group, then tap the IMPORT button in the Rule Group window. Add a name for the rule group and browse to find the file to import.

    To reference an existing Rule Group in a different Trigger, select a new Trigger, tap the Add Rule Group button, then tap the REFRENCE RULE GROUP button. Select the name of the rule group and add the new trigger.

    Notice:
    NCC does not support RSV exported from a previous release version and imported into a higher release version by using the export/import functionality. However, for the same release, RSV export/import is supported.

  • Export

    Export is supported using the export button that appears in the RSV table when the cursor is hovered over a line item in the table. The name of the file being imported in this release cannot be changed.

  • Clone

    Clone is supported using the Clone button that appears in the RSV table when the cursor is hovered over a line item in the table. The name of the file being cloned in this release cannot be changed.

  • Distribute or provision to an ME subsystem

    The Release version of the RSV is the one that must be distributed to all of the MEs in a Geo-cluster or a single ME in a standalone system.

  • Activate on the ME subsystem

    After the Release version of the RSV has been distributed or provisioned, it must be activated.

When the Rules and Variables section is selected from the Charging panel of Global Rules, there are two options for RSVs to be created, one for Charging and another for Recharge.

By default, the Charging RSV under the Rules and Variables section is opened. The charging application provides a default RSV. You can configure a new RSV with a different configuration of rules with triggers and rule variables.

Similar functionality is available if you select RECHARGE under Rules and Variables.

The Rules and Variables section displays the total number of Active and Draft RSVs respectively when you select CHARGING or RECHARGE. The section also lists Active RSV Details as a summary and inactive RSVs you have created for testing purpose.

The Rules and Variables section also allows for searching, exporting, and importing RSVs, as well as creating and configuring new RSVs.

To create a new RSV, you must:
  1. Provide a name for the RSV.
  2. Optionally, provide a description.
  3. Save and Close.

New RSVs are in the Draft state automatically. If an RSV must be edited, ensure the RSV is in a Draft state. After it is created, the RSV may be edited to change the state to Release, and distributed to the ME subsystems where it can then be activated.
