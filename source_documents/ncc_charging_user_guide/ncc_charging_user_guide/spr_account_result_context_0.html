Note:
We should be able to add hard coded value of TPGs as well as using rule variables as shown above in the last example.

Set-Custom-Data-expiryTime LOAN_PRE_PROCESSING This action is used to convert a non-timed custom data to timed custom data for the target account or device.

It takes custom data name and expiry time as arguments:

Set-Custom-Data-expiryTime(<custom_data_name>, <expiryTime>)

Expiry time is optional.

If expiry time is not provided, then custom data definition and the calculated value of expiry time are used.

If expiry time is provided, then the expiry date is set in msecs of the attached custom data. The new value of expiry time results in recalculation of TEM.

If the expiration time is in the past, then the custom data expires immediately.

If the expiry time is set to 0, then the custom data becomes non-timed, and the current value of custom data remains unchanged.

Stop-Custom-Data-expiryTime LOAN_PRE_PROCESSING This action is used to stop the expiry time of custom data.

If the custom data is timed, then the timer stops and custom data retains its current value. If this action is configured, then the expiry time of custom data becomes null, and TEM is removed.

This action is applicable only for timed custom data.

It takes custom data name as an argument:

Stop-Custom-Data-expiryTime(<custom_data_name>)
