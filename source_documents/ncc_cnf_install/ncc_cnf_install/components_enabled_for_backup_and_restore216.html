Components enabled for backup and restore

This section describes the Helm charts and the charts' components that NCC enables for backup-and-restore when a user selects cbur as value for the Features parameter in the Inputs.xlsx file.
Table: Components enabled for backup-and-restore
Helm chart YAML file Volume
ckey brpolicy.yaml backup (volumeClaimTemplates)
mariadb-cbur-policy.yaml backupdir (volumeClaimTemplates) or datadir (volumeClaimTemplates) / backup
csdc policy.yaml tmpdatavol (emptyDir volume type)
cpro brpolicy.yaml storage-volume (volumeClaimTemplates)
cpro-gen3gppxml brPolicy.yaml gen3gppxml-data (volumeClaimTemplates)
cpro-grafana grafana-brpolicy.yaml appdata (emptyDir volume type) or storage (persistentVolumeClaim)
mariadb-cbur-policy.yaml backupdir or datadir (volumeClaimTemplates) / backup
