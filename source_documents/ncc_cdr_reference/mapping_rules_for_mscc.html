Mapping rules for MSCC information

Table: Mapping rules for EDR MSCC information
Output field name Input field name Conversion rule
merchantId <parent>.merchantId

Value mapping (case-insensitive):

  • SMSALL -> 0 (smsall)
  • MMSALL -> 1 (mmsall)

If input exists but does not match any of the above, use the default value:

100 (others)
serviceType <parent>.serviceType

Value mapping (case-insensitive):

  • MO -> 0 (mobileOrig)
  • MT -> 1 (mobileTerm)

If input exists but does not match any of the above, use the default value:

100 (others)
intlIndicator <parent>.intIndicator

Direct mapping:

  • 0 -> 0 (false)
  • 1 -> 1 (true)
recipientNumber <parent>.recipientAddress Direct mapping.
originatorNumber <parent>.originatorAddress Direct mapping.
result <parent>.result

Value mapping (case-insensitive):

  • Success -> 0 (success)
  • Fail, Failed, Failure -> 1 (failure)
reason <parent>.reason Direct mapping.
