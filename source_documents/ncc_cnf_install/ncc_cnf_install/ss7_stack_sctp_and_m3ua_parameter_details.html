The ASPM timer is governed by following high, low bound parameter type and a threshold counter.

Till SWTHRESH count, TIME_LOW shall be in use, while after that TIME_HIGH is used - till max retransmission of ASPM messages.

Parameter Type:

M3UA_PROT_RETRAN_TIME_LOW This parameter is used to change the initial low time of the re-transmission timer. Time is in seconds.

M3UA_PROT_RETRAN_TIME_HIGH It changes the switch over high time of the re-transmission timer. Time in seconds.

M3UA_PROT_RETRAN_TIMER_SWTHRESH It changes the switch over count of the re-transmission timer.

If the parameters mentioned in the table are configurable and if the parameter location is in the sigtran.prov file then follow these steps to update these parameters.

  1. Navigate to the directory that you unpackaged the Input Generator files.
    • cd /input-generator/charts/
  2. Unpackage the ss7em chart.
    • tar -xf ss7em-chart-22.9-4.185.tgz
  3. Navigate to the template directory inside the ss7em directory.
    • cd /input-generator/charts/ss7em/templates
  4. Open the sigtran-configmap.yaml file and edit the desired parameters as mentioned in table.
  5. Repackage this file again with the same name as it was before and follow the standard chart installation procedure.
  6. The parameter mentioned as edit from OSP GUI can be updated after the installation of the charts.
  7. If the parameter is mentioned as "CLI (Command Line Interface) Command", then the parameter can be updated from the ss7cli pod after installation.

Overriding the default SCTP parameters

The supported configurable SCTP parameters can be overwritten during installation via a custom ss7-values.yaml file. Perform these steps to generate the override file.

  1. Create a ss7-values.yaml file with the configurable parameter override values.
    # Note - Properties set under global section of this file are not used while generating SS7 Helm overrides files.
# Global properties set under each sub-chart's global section form the 'global' section in that subchart's Helm
# overrides file can be used to set Helm overrides for SS7 charts

ss7em:
  global:    
    M3UA_SCTP_CONFIG_PARAMS:
       rto_init: 1000
       rto_min: 500
       rto_max: 3000
       rto_alpha_percent: 25
       rto_beta_percent: 25
       valid_cookie_lifetime: 60000 
       assoc_max_retrans: 8
       path_max_retrans: 5
       max_init_retrans: 8
       hb_interval: 30000
       bundling_time: 100
       sack_period: 2
       sack_frequency: 2 
       mtu_size: 1500

    labels:
      chartname: ss7em
  2. Add the ss7-values.yaml file name to the Overrides value field in the SS7 section of the namespace worksheet in Inputs.xlsx.
  3. Run this command. 
    ./generate.py -i Inputs.xlsx

The generated override file is located in the output folder of ss7em.yaml.
