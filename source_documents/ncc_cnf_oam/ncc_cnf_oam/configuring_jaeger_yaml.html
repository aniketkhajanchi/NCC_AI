Jaeger configurations in the values.yaml

Overview

Jaeger configurations may be made in the values.yaml file as part of the OpenTracing configurations.

The following is an example of the OpenTracing and Jaeger configurations in the values.yaml.
opentracing:
    headers: "x-request-id,x-b3-traceid,x-b3-spanid,x-b3-parentspanid,x-b3-sampled,x-b3-flags,b3"
    jaeger:
      endpoint: "http://jaeger-collector.istio-system.svc.cluster.local:14268/api/traces"

Header configuration

In the values.yaml, the headers property can be used to configure which headers are propagated.
opentracing:
    header: <your_headers_here>

Where <your_headers_here> is a comma separated list of headers to propagate.

For example, the following is configured to propagate headers x-request-id, x-b3-traceid, x-b3-spanid, x-b3-parentspanid, x-b3-sampled, x-b3-flags, and b3.
opentracing:
    headers: "x-request-id,x-b3-traceid,x-b3-spanid,x-b3-parentspanid,x-b3-sampled,x-b3-flags,b3"

Endpoint configuration

In the values.yaml, the endpoint property can be used to configure endpoints, which tell Jaeger where to send traces collected in NCC.
opentracing:
    headers: "x-request-id,x-b3-traceid,x-b3-spanid,x-b3-parentspanid,x-b3-sampled,x-b3-flags,b3"
    jaeger:
      endpoint: <endpoint_address>

Where <endpoint_address> is the address of the endpoint to which Jaeger will send traces.

For example, the following is configuration sends traces to the endpoint at http://jaeger-collector.istio-system.svc.cluster.local:14268/api/traces.
opentracing:
    headers: "x-request-id,x-b3-traceid,x-b3-spanid,x-b3-parentspanid,x-b3-sampled,x-b3-flags,b3"
    jaeger:
      endpoint: "http://jaeger-collector.istio-system.svc.cluster.local:14268/api/traces"
