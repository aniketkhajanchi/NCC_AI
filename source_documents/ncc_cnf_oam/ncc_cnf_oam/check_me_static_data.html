check-me-static-data

Description

check-me-static-data.sh enables users to compare the static data of ME subsystems by dumping the contents to files which may then be compared. The tool dumps the contents in the namespace dsclocal of the ME subsystem from which the tool is run.

This tool generates the following files.
  • <PID>_<ME_name>: This file contains the record info, or edigest, from the database for the IG or ME system with name <ME_name> and process ID <PID>. This file is generated when using the -d flag.
  • <PID>_Detailed_<ME_name>: this file contains a full record of information from the database for the IG or ME system with name <ME_name> and process ID <PID>. This file is generated when using the -d flag.
  • <PID>_missing_records: this file contains a record of differences found between input ME subsystems. This file is generated when using the -c flag.

Syntax

check-me-static-data.sh [-d <ME_name>] [-p <path_to_dump>] [-e <list_of_excluded_sets>] [-l]
check-me-static-data.sh [-c <file_having_dump_1>,<file_having_dump_2>]
check-me-static-data.sh [-h/-t]
Option Description Option arguments
-h or -? Displays usage/help information.  
-d <ME_name>

Dumps the static data of the provided ME subsystem, which is used to generate the output files.

 <ME_name> is the name of the target IG or ME system.
-p <path_to_dump> Indicates the path where the file should be dumped. <path_to_dump> is the path where the file should be dumped. Default value is /opt/tpa/logs.
-e <list_of_excluded_sets>

Indicates a list of individual sets to exclude by the tool.

 Comma separated list comprised of any set intended for exclusion by the tool.
-c <file_having_dump_1>,<file_having_dump_2>

Compares dumped data present in the provided files (generated with -d option).

 Comma separated list comprised of filenames of the dumped data. All files which need to be compared should be provided in this list.
-t Runs the tool in DEBUG mode, which provides a more detailed terminal output.  
-l Displays the default entities name that will not be searched.  

Examples

This example shows check-me-static-data run with option -h.

[tpaadmin@...]$ check-me-static-data.sh -h
  Dump/Compare ME static Data.

  Usage:
  check-me-static-data.sh [-d <ME_name>] [-p <path_to_dump>] [-e <list_of_excluded_sets>] [-l]
  check-me-static-data.sh [-c <file_having_dump_1>,<file_having_dump_2>]
  check-me-static-data.sh [-h/-t]

where:
  -d : Dump Static Data of the Provided ME.
  -p : Path where file should be dumped (Default: /opt/tpa/logs).
  -e : List of Sets to exclude (comma separated Set Names)
  -c : Compare dumped data present in the provided files (generated with -d option).
  -h : This help.
  -t : Run with Debug. (by default off)
  -l : Display the default entities name that will not be searched.

O/p Files:

With -d option:
  <PID>_<ME NAME> : This file will contain the record info (edigest) from the database.
  <PID>_Detailed_<ME NAME> : This file will contain full record info from the database

With -c option:
    <PID>_missing_records : Information on records differences.
