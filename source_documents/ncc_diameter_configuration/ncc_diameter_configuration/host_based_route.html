Host based routes

To view the list of host-based route, click Connectivity → ☰ → DIAMETER ROUTES → Host Based Routes. Click + to create a new host-based route.

Note:
All Diameter-related configuration areas can also be configured using APIs.
Table: Host Based Routing parameters
Parameter Description Mandatory / Optional
Route Id

Specify a unique identifier for the route that can be used to distinguish between multiple routes with the same destination identity name but different priorities.

NCC automatically creates this ID in case no ID is mentioned. However, if multiple MEs are registered to SM, then SM sends this data with ID created at the SM side to track the provisioning status on each ME.

If the route name is not specified, then it is auto-generated as a concatenation of the values in Next Hop, Destination Identity, and Application.

 Mandatory
Description Specify a description for the Diameter route. Optional
Next Hop

Specify the hostname of the server to which NCC has to forward the incoming message.

The valid values are host-name, diameter-peer, or FQDN.

 Mandatory
Destination realm

For Diameter destination routes, specify the value which is received in the Destination Host AVP of the incoming message. If NCC is not directly connected to this peer (to the peer mentioned in Destination realm), then NCC forwards the message to the peer specified in Next Hop.

The asterisk * can be used as a wildcard to match multiple routes for patterns. NCC uses the highest priority route that matches the pattern.

 Mandatory
Application

Specify a list of one or more application types to reach a specified host or realm. Select an application type from the drop-down to add it to the list.

By default, all applications are valid for a route. Keep the field blank to enable all applications.

 Mandatory
Priority Specify the Diameter route priority. Mandatory
