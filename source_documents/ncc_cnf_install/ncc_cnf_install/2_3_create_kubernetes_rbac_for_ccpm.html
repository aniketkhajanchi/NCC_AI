Create the Kubernetes role-based access for Container Package Management

Create a Helm override file named ccpm-resource.yaml with these values:

ccpm:
  # Update this list with the namespaces that Container Package Management is constrained to. In this example, Container Package Management only manages pods in the IG namespace.
  targetNamespaces: ["ig1"]
rbac:
  # By setting this value to true, a clusterrole is created with the ability to create resources at the cluster scope.
  createElevatedPermissions: false
   # Set this value to true when installing Istio using operator.
  createIstioPermissions: false
# The name to be used while creating the Kubernetes serviceAccount for operator. By default, chart creates serviceAccount name automatically if no value is provided.
# This service account name should be passed at the time of Container Package Management operator install.
serviceAccountName: ccpm-resource-sa
Set this variable:
TARGET_CHF_SOLUTION_VERSION=23.2.1-XXX

Run this command to install the chart:

helm upgrade --install -n ncc-operators ccpm-resource <helm_repo_name>/ccpm-resource --version ${TARGET_CHF_SOLUTION_VERSION} -f ccpm-resource.yaml --wait
