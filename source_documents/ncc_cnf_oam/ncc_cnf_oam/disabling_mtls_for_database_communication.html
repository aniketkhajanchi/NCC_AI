Disabling TLS for intersystem Aerospike XDR communication

This procedure provides instructions on how to disable TLS protocol for Aerospike XDR communication between the two geo-redundant systems of the same type. This procedure is used post-installation when NCC has already been installed. For instructions on how to disable TLS for intersystem Aerospike communication during installation, see the Installation and Upgrade Guide. In this procedure, the two systems are called System 1 and System 2.

  1. On System 1 and System 2, set the xdrproxy.exposeInsecure parameter to true in the input file and then run helm upgrade with -f option passing the input values.yaml file.
  2. On System 1 set the aerospike.network.outboundXdr.mtls.enabled parameter to false in the input file and then run helm upgrade with -f option passing the input values.yaml file.
  3. On System 2 set the xdrproxy.mtls.enabled parameter to false and the aerospike.network.outboundXdr.mtls.enabled to false in the input file and then run helm upgrade with -f option passing the input values.yaml file.
  4. On System 1 set the xdrproxy.mtls.enabled parameter to false bin the input file and then run helm upgrade with -f option passing the input values.yaml file.
