Validate your AWS EKS cluster

  1. Go to the AWS console and check for EKS clusters and it should be active
  2. All of your node groups in the cluster should be ready and active.
  3. Export the generated <your-cluster>.conf file as the KUBECONFIG and run kubectl commands to check connectivity and cluster health as well.
  4. Check all of the AWS storage classes are present. They are called (io1,st1,sc1). (kubectl get sc)
  5. Check for the presence of restricted and privileges PSP (kubectl get psp).
  6. Check for the presence of ncc-node-critical priority class.(kubectl get pc).
  7. Check your nodes in different node groups to make sure they have the label and taints you provided by describing them.
