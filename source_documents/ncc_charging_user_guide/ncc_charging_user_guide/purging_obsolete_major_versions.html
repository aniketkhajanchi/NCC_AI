Purging obsolete major versions

Purging is available only for the following entities:

  • Service
  • Charging Logic

Purging permanently deletes draft and obsolete versions older than a selected major version and older than a pre-defined number of days. The number of days is defined in the SM application preference History Period (days).

Purging does not delete the contributing draft version. For example, if you select 5.0.0 and perform a purge, then it deletes 4.0.0 and any version before it. It does not delete version 4.1 because it is the contributing draft version of 5.0.0.

The older versions are deleted under the following conditions:

  • The major number of the older version is less than the selected major version.​
  • The state of the entity is not ACTIVE, TEST_ACTIVE, or SCHEDULED.​
  • The version is expired. That is, start time of the version is before the number defined in HistoryPeriod​.

Procedure:

  1. Go to the Service/Charging Logic listing page.
  2. Click in the respective row of the Service/Charging Logic. The Service/Charging Logic details page appears.
  3. In the left side of the page, from the Select version drop-down list, select a major version (for example, 5.0.0).
  4. Click PURGE OLDER VERSIONS.
    Note:
    This (action) is only available for major versions.
  5. A confirmation screen appears with a list of all versions to be deleted. Click PURGE to complete the action.
