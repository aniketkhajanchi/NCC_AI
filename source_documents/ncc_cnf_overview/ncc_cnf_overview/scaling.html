Scaling

Both CG Rejection Collector Node and CG Rejection Distributor Node can be scaled by increasing the number of worker threads transferring files to/from the SFTP server. The number of worker threads is defined with node parameter WorkerThreads. Each worker thread transfers a separate file and uses a separate SFTP connection.

In addition to using multiple worker threads, reprocessing can be scaled by running multiple instances of the Reprocessor stream. The implementation prevents multiple instances of CG Rejection Collector Node from transferring the same file. The coordination between node instances is implemented using per-file locks in the database.

Note:
To reprocess rejected records in multiple streams, each CG Rejection Collector Node must be scheduled individually.
