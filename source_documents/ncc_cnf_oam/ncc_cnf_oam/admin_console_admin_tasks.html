Admin console administrative tasks

Overview

The bulk of administrative tasks are done through the User and Role Management admin console. The console may be accessed directly at https://$SM_FQDN/auth/admin with the user credentials created in the add-user-keycloak script in the bin directory. This would further redirect to the User and Role Management admin console.
Note:
: If the master realm user forgets their password, the add-user-keycloak script may be used to create a new administrator of the master realm.

The right-most drop-down menu on the admin console allows the administrator to view the user account or logout from the current session. To inquire about a certain feature, button, or field within the User and Role Management admin console, simply hover the mouse over any question mark (?) icon which pops up the tool-tip text to describe the area of the console of interest.

This section allows the selection of the sps-auth realm, configure realm settings, clients, client scopes, roles, identity providers, user federation, and authentication using the User and Role Management admin console.
Note:
When a Geo-redundant pair of SM/IG systems are installed and changes are done on the User and Role Management for any of these systems, the caches must be cleared (realm, user, and key cache) on the other system to reflect the same changes. The only time that this does not need to be done is the creation and deletion of users, roles, and user-role mapping which gets reflected without the need to clear the cache. The changes may include creation, deletion and modification on password policies, user attributes, clients, events, and so on. To clear the caches, perform these steps:
  1. Select the realm on which the changes were made.
  2. Go to Realm Settings -> Cache.
  3. Click the Clear button for all three caches (Realm Cache, User Cache, and Keys Cache).
