log-level

Description

The log-level command provides the admin with get/set/reset log levels for an individual pod. This tool helps the admin get/set/reset log levels without any modification in log4j.xml.

The admin can easily switch back and forth from these logging levels, which are common to Log4j:
  • TRACE
  • DEBUG
  • INFO
  • WARN
  • ERROR
  • FATAL

The log-level command must be run from the individual pod whose logging level is to be changed. To access the individual pod, run this command from a control node.

kubectl exec -it <pod_name> bash
Where:
  • <pod_name> is the name of the pod which is to have its logging levels modified.

For example, this command issued on a control node initiates login to the pod charging-pod.

kubectl exec -it charging-pod bash
To get the level of a logger, run this command on an individual pod:
log-level -n localhost:3333 <logger name>
To set the level of a logger, run this command on an individual pod:
log-level -n localhost:3333 <logger name> <level>
To reset the levels of all loggers or a specific logger back to default, run this command on an individual pod:
log-level -n localhost:3333 -r [<logger name>]
Note:
Changing the logger level using this utility does not persist after an application server restart. For a persistent change, add a "logger" entry in log4j.xml.

Examples of get, set and reset all operations

To get log level:

get operation
Note:
A value of 'null' means that a level has not been explicitly set for the given logger. When a level of 'null' is obtained, a recursive attempt is made on the parent logger in order to determine the inherited log level.

To set log level:

set operation
Note:
A previous level of "INFO" is assumed if a level has not been explicitly set for the given logger.

To reset all:

reset all operation

Note:
If no loggers need to be reset, the log4j.xml configuration file will be read and any changes from the file will be applied.

Syntax

log-level -n localhost:3333 -r [<logger name>]
Option Description
No arguments Displays usage/help information.
-h Displays usage/help information.
-n localhost:3333 Indicates the server host name, which will always be localhost:3333.
<logger name> Provides the logger name. Example:

log-level -n localhost:3333 <logger name>
<level>

Indicates the logging level. INFO, TRACE, DEBUG and WARN.

Valid values for the logging levels are:
  • TRACE
  • DEBUG
  • INFO
  • WARN
  • ERROR
  • FATAL

Example:

log-level -n localhost:3333 <logger name> <level>
-r <logger name> Resets the levels of all loggers or sets a specific logger back to the default. Example: 
log-level -n localhost:3333 -r [<logger name>]
