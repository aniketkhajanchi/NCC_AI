Removing NCC by using the NCOM API

Only deployments which were created by using NCOM can be removed by using NCOM.

  1. Edit the deployNS.conf file located in the input_generator/output/SITE_TYPE/examples/NCOM directory.
    You need to add the values for NS_PACKAGE_ID and NS_INSTANCE_ID. These values can either be obtained from the NCOM GUI or by using the script deployNS.sh. For more information on the tool, see Using the deployNS.sh tool.
  2. Run this command to terminate the NS instance. 
    ./deployNS.sh -f deployNS.conf -R instance

    If you encounter errors, use the --force flag.

  3. Run this command after removing the NS instance to deactivate the package before removing it. 
    ./deployNS.sh -f deployNS.conf --only deactivateNSPackage
  4. Run this command to delete the package. 
    ./deployNS.sh -f deployNS.conf -R package
