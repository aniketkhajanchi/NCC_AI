Flushing database

In some situations it may be necessary to empty the (Redis) database in a controlled manner. You can do so by executing the FLUSH_DATA instance of CG Configuration Node in the Configuration stream.

For more information on flushing of data, see the Overview Guide.

  1. In Charging Gateway Function UI, navigate to your Configuration stream, then select the FLUSH_DATA instance of CG Configuration Node.
  2. Open the node parameters for editing and edit the FlushMode parameter. Make sure that the parameter has been set to value Immediate.
    Note that once the flushing is started, incoming traffic is stopped.
  3. Click Execute now to start the flushing process.
    The alerts displayed for the node indicate how the flushing proceeds. Info message CG057 indicates that the flushing process has been completed.
  4. When the flushing is complete, edit the value of node parameter FlushMode again. Select value None and execute the FLUSH_DATA instance of CG Configuration Node again.

    Executing the node with FlushMode set to None is required for the system to start accepting incoming traffic again.

Expected outcome

Info message CG056 indicates that the flushing process has ended.
