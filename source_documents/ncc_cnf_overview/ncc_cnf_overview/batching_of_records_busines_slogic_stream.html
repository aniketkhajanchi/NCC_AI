Batching of output records

When a batch of input records is available for the Business logic stream (batch has been closed in Kafka input stream), CG Mapper Node reads every record in the batch and maps the input records into output records. The node then creates batches of output records for the output stream to process.

Record batches can be configured in terms of the number of records in a batch and the time that passes between creating batches. These are configured with node parameters RecordsPerBatch and SecondsPerBatch respectively.

Node parameter RecordsPerBatch defines the maximum number of records that CG Mapper Node can collect into an output batch. The minimum value is 1, and the default value is 10000. With node parameter SecondsPerBatch, a maximum wait time can be set, after which CG Mapper Node creates a new batch of records. Setting the parameter to 0 (minimum value) means that the wait time functionality is disabled.

Normally, CG Mapper Node creates a new batch when the number of records defined with RecordsPerBatch are ready. If this maximum number is not reached before the time defined with SecondsPerBatch has passed, it creates a new batch anyway. The default value for SecondsPerBatch is 600.
