Recovery from data loss (non-catastrophic disaster)

There are two possible scenarios:

  • the data that was lost in one SM was replicated on other SMs (for example WebSSO data and Aerospike data), or
  • the data that was lost in one SM was unique to that SM (for example data related to telemetry applications).

For the second scenario, it is assumed that a restore of the latest Backup and Recovery backup is acceptable, as the data does not change that frequently and/or the application is completely functional even if some data cannot be recovered. See NCC Telemetry data recovery.

For the first scenario, a procedure is outlined here to recover the current data from another site.
