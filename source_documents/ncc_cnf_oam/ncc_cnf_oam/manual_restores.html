Restore a backup

The procedure to restore a specific backup is described in detail for each component. It is performed by executing a curl command from the admincli pod. This is a generic example.

CBUR_MASTER_NAMESPACE=naperil110vzwcchf-y-nk-sm-001   
NAMESPACE=naperil110vzwcchf-y-nk-sm-001
RELEASENAME=ckey
BR_POLICY=ckey-ckey

# Initiate Restore
curl -vkL --header 'Accept: application/json' -X POST --header 'Content-Type: application/x-www-form-urlencoded' -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/restore/$NAMESPACE/$RELEASENAME?helm_version=3

# In alternative to the command above, which will restore the latest backup taken for each BrPolicy defined for the specified release, 
# if more than one, a different URL can be used in the curl command to specify the name of a BrPolicy:

# Initiate Restore
# curl -vkL --header 'Accept: application/json' -X POST --header 'Content-Type: application/x-www-form-urlencoded' -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/restore/$NAMESPACE/$BR_POLICY?helm_version=3
