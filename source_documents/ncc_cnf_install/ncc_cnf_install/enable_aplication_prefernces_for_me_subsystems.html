Enable application preferences for ME subsystems

After you deploy the canary pods, you must set specific application preferences in the SM GUI when canary is enabled on ME subsystems.

  1. Login to the SM GUI.
  2. Click on Platform.
  3. Click on the information icon that appears when you highlight an SM in the Service Managers section.
  4. Click on the PREFERENCES tab.
  5. Scroll to the Canary Testing section or use the search bar and search for canary.
  6. Update properties in this section with these values:
    1. ME Canary Type: The default value is NONE. Update this value to match the canary type that the ME subsystem is using.
    2. Canary SM Connection Timeout (ms): The default value is 2000. Specify the amount of time, in milliseconds, that the canary client attempts to establish a connection with canary servicemanager before it times out.
    3. Canary SM Response Timeout (ms): The default value is 5000. Specify the amount of time, in milliseconds, that the canary client should wait for a response from the canary servicemanager before it times out.
  7. Save and exit.
