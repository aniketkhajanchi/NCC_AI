Pushing images to a customer repository

This procedure provides instructions on how to upload NCC and NCC SS7 images to a customer repository. The example script in this procedure does the equivalent of the following for every Docker tar file in the directory:
  1. Loads the file.

    Example:

    docker load < node_exporter_v1.0.1-2.tar

    where node_exporter_v1.0.1-2.tar is a file to push to a registry.

  2. Tags the file.

    Example:

    docker image tag csf-docker-delivered.repo.lab.pl.alcatel-lucent.com/cpro/registry4/node_exporter:v1.0.1-2 abc.com/cpro/registry4/node_exporter:v1.0.1-2
  3. Pushes the file to the registry.

    Example:

    docker push example.com/cpro/registry4/node_exporter:v1.0.1-2

    where example.com is the FQDN of the registry.

  1. On your Linux workstation, go to the directory with all the Docker images.
    Note:
    Your workstation might use a non-Linux system like Windows.
  2. Perform the equivalent of what this example script does. 
    #Set the repo to the customers repository

for dockerImage in *.tar;
do
    echo "Processing $dockerImage ";
    #import the docker image, it will output its full repo/name/tag information
    loadResult=`docker load < $dockerImage`
 
    fullTag=`echo $loadResult | cut -d " " -f3`
 
    #tag the docker image in your machine's docker for the target customer repository
    docker image tag ${fullTag} ${TARGET_REPO}/${fullTag}
  
    #push the newly tagged image to the customer repository
    docker push ${TARGET_REPO}/${fullTag}
done
