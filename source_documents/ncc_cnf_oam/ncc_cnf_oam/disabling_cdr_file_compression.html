Enabling/disabling CDR file compression

During stream deployment, compression of CDR files can be enabled or disabled through the connection parameters of the CG Distributor Node in the file output stream. To enable compression, an external command has to be defined for the connection parameter External Command In order to have the same file name in both the compressed file and its content, the -n option has to be added to the command. For example, /bin/gzip -n &FILE, where &FILE automatically points to the CDR file that is being processed. To disable compression, the value of the connection parameter of the External Command should be left empty.
