API methods

This table lists the API methods and CURL examples.
Table: API methods
Method Purpose Path Note CURL example
GET Loading data /cg/api/v1/lookup-service Condition is not mandatory here. If it is not added, all of the data is loaded. 
curl --location --request GET 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service' \
 --header 'Authorization: Bearer TOKEN' \ 
 --header 'Content-Type:application/json' \ 
 --data-raw '{"table":"lt_distr_rules_vzsps", "conditions": {"c1": "v1", "c2":[1,2,3]}}'
POST Saving data /cg/api/v1/lookup-service   
curl --location --request POST 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service' \
 --header 'Authorization:Bearer TOKEN' \
 --header 'Content-Type:application/json' \
 --data-raw '{"table":"lt_distr_rules_sps", "data":[{"priorityrule":"80", "rule_outputs":"DEFAULT_FILE","id":88}]}'
PUT Editing data /cg/api/v1/lookup-service Condition is not mandatory. If it is not added, all of the values for the columns in the data key are changed. 
curl --location --request PUT 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service' \ 
 --header 'Authorization: Bearer TOKEN' \
 --header 'Content-Type:application/json' \
 --data-raw '{"table":"lt_distr_rules_sps", "data": {"description":"updated descript"}, "Conditions":{"rulecriteria":"N40_FAILURE_RESULT_CODE"}}'
DELETE Deleting data /cg/api/v1/lookup-service Condition is not mandatory. If it is not added, all of the table is cleaned from the data. 
curl --location --request DELETE 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service' \
 --header 'Authorization:Bearer TOKEN' \
 --header 'Content-Type:application/json' \
 --data-raw '{"table":"lt_distr_rules_sps", "conditions": "id":130}}'
POST Publishing configuration /cg/api/v1/lookup-service/publish Publishing configuration rules through HTTP requests. Expected response: [{"orderId": 8013,"parameters": {}}] 
curl --location --request POST 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service/publish' \
 --header 'Authorization:Bearer TOKEN'
GET Get node events /cg/api/v1/lookup-service/result List of node warnings and errors.

Limit: Number of results per page, the default 10.

Page: Page number for navigating results.

 
curl --location --request GET 'http://vbox.nokia.com:8090/cg/api/v1/lookup-service/result' \
 --header 'Authorization: Bearer TOKEN' \
 --header 'Content-Type:application/json' \
 --data-raw '{"limit":5, "page":2}'
