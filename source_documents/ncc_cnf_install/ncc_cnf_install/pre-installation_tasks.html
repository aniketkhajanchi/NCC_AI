Pre-installation tasks

NCC assumes that much of the base AWS infrastructure is already installed and does not provide automation for some pre-installation tasks.

These items must already be present.

  • An AWS account with credentials
  • A Virtual Private Cloud (VPC) with at least two subnets available for EKS to use. The two subnets must be in two different availability zones. Note that NCC only uses a single availability zone when deploying.
  • In case multiple sites are being set up, each should have its own availability zone with two subnets and be within the same VPC network.
  • NCC container images loaded into a registry. This can be either the AWS service Elastic Container Registry (ECR) or some other container image registry.
    Note:
    NCC provides a script as part of the input-generator to help upload container images to ECR. This is described later.
  • The private subnet within the VPC must have internet connectivity, so that the EKS APIs can be reached during setup.
  • An elastic IP address. This is used to have a fixed target for the NCC application.
  • An FQDN registered with a DNS server associating a DNS name with the elastic IP address. This FQDN and the elastic IP address identifier are then used in the input-generator.
