Preparing NCS

This procedure provides instructions on how to prepare NCS for Helm charts installation with NCOM.

  1. Add RBAC resources.
    Note:
    The RoleBindings and new Role Definition need to be changed at the NCS level for NCOM to access CaaS.
    For information on how to add RBAC resources, see section Appendix M: Onboarding NCS details into NCOM as CaaS Resource in <NCOM Version x> Operating Guide.
  2. Add the CaaS.
    Note:
    The NCS cluster needs to be added so that NCOM can deploy the Network Services on this cluster.

    To add the Caas, choose Authentication type as Token and add the token from step 2. For more information on how to add a CaaS, see section Managing resources/Containers as a Service/ Adding a CaaS in <NCOM Version x> Operating Guide.

  3. Add Helm registries in NCOM for NCC Helm charts installation.
    Note:
    Although this step is optional, you need to perform the steps to Add Placement Constraints in the registries.

    For information on how to add a Helm registry, refer to section Managing resources/Registries/Adding Registries in <NCOM Version x> Operating Guide.
