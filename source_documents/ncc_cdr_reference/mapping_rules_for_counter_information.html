Mapping rules for counter information

Table: Mapping rules for EDR counter information
Output field name Input field name Conversion rule
counterName <parent>.counterName Direct mapping.
counterKey <parent>.counterKey Direct mapping.
counterType <parent>.counterType

Value mapping (case-insensitive):

  • Usage -> 0 (usage)
  • Cost -> 1 (cost)
  • Call -> 2 (call)
kindOfUnit <parent>.counterKindOfUnit

Value mapping (case-insensitive):

  • Volume -> 0 (volume)
  • Money -> 1 (money)
  • Time -> 2 (time)
  • Unit -> 3 (unit)
unitType <parent>.counterUnitType Direct mapping.
counterValueBefore <parent>.counterValueBefore

Validate that the input is of type Real. Raise a warning and do not map if it is not.

Do the conversion as described in Mapping of Real values.
counterValueAfter

<parent>.counterValueAfter

<parent>.counterValue

For Lifecycle EDRs, use input field counterValueAfter. For eCommerce EDRs, use input field counterValue.

Validate that the input is of type Real. Raise a warning and do not map if it is not.

Do the conversion as described in Mapping of Real values.
counterUsageLimit <parent>.counterUsageLimit

Validate that the input is of type Real. Raise a warning and do not map if it is not.

Do the conversion as described in Mapping of Real values.
counterScope <parent>.counterScope

Value mapping (case-insensitive):

  • ALL -> 0 (all)
  • SELF -> 1 (self)
  • HIERARCHY -> 2 (hierarchy)
  • MULTI_BUNDLE -> 3 (multiBundle)
  • SUBSCRIPTION -> 4 (subscription)
