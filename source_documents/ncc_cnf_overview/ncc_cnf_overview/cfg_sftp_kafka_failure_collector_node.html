CGF (DR) SFTP Kafka Failure Collector Node

This node can be scheduled to reprocess records files that failed to write to Kafka because of a Kafka brokers outage. It can be configured to retrieve the log files according to a predefined schedule or it can be executed manually.

When this node is triggered, it starts collecting records files logs from the SFTP directory only if the status of one of the Kafka brokers is up, which is observed by the Kafka Distributor node. These records are written to the JobQueueSuffix node parameter in the database and distributed to Apache Kafka again. The successfully collected records are moved to a another directory named processed. It is important that node parameter JobQueueSuffix is set to the same value in CG Kafka Distributor Node and CGF (DR) SFTP Kafka Failure Collector Node.
