Attaching SS7 roles with User and Role Management users

To configure a customized role to any User and Role Management user, a profile must be created in the OSP pfmaccess service. The same role must also be created in the User and Role Management server. Once the role is configured in User and Role Management, the same role must be attached to a user in the User and Role Management server. When this user logs in to the SS7 system through OSP GUI or through NCC-SS Web UI, the user is able to perform provisioning operations based on rights available on that role. Two or more SS7 roles can also be attached with a user. In this case, the user is able to perform provisioning based on the effective rights of all the attached SS7 roles.
