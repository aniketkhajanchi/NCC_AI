Input generator

The input generator prepares the values for all Helm charts across all systems in your deployment. The values are based on the inputs that a customer provides in the input file called Inputs.xlsx. The input generator stores different sets of default values and combines them with your choices to provide results that are ready for your deployment. The resulting files are not intended to be edited. However, if input changes, new files can be regenerated.

The input generator does not perform the installation and does not make any changes to Kubernetes or your system. It only prepares the inputs to the Helm charts. The input generator significantly reduces the amount of information that you need to provide. Any value for a Helm input generated by the input generator can be overridden by specifying the Helm property and value explicitly.

The input generator is a Helm chart published with the other NCC Helm charts and is tied to an NCC release.

Input file

The Inputs.xlsx file allows you to provide your deployment's specific values and override the default values for the Helm charts. It serves as a central planning tool that covers all of the sites across regions and data centers. A single input can be used to modify the behavior of the entire system for example, enabling Istio or Backup and Recovery. The example spreadsheet file is used to prepare your Inputs.xlsx file. The procedure Preparing Helm chart input values provides instructions on how to tailor the Inputs.xlsx file for deployment.

The prepared Inputs.xlsx file must contain the Common worksheet and a worksheet for each namespace in the network deployment.

The Common worksheet holds the values that apply to all the systems in the network deployment. Every namespace worksheet holds the values that apply only to one particular namespace from the deployment. Each BTEL section holds the values that apply to NCC Telemetry configuration of a namespace.

The example Inputs.xlsx file contains template namespace tabs and BTEL section for each supported system/subsystem type: SM, ME, and IG.

Changing values with a custom override YAML file

If you need to change a Helm chart value that is not exposed in the Inputs.xlsx file for example, a charging pod request CPU, you need to use the Overrides parameter of that particular Helm chart in the Inputs.xlsx file . For instructions on how to use this parameter and create a custom YAML file, see the procedure Providing Helm chart inputs directly.

Generated output files

All of the files that the input generator produces go into the input-generator/output folder. Inside the output folder is a subfolder that is created for each namespace worksheet in your version of the spreadsheet.

Each namespace output folder contains these items:

  • charts

    This folder contains the YAML files that are input to the Helm charts. Each file's name matches its target Helm chart name.

  • resources

    This folder contains other Kubernetes objects that need to be created before installation. Examples are the service accounts and namespace object.

  • examples

    This folder contains example commands that might be used for installation. Since the contents of this folder are intended as examples, running the entire scripts might not work in all environments.

  • site.conf

    This file contains variables that might be useful during installation.

Note:
  • Do not modify the output files. If a specific change is needed, use a custom override file.
  • Only the applicable YAML files are generated for each system. For example, since the User and Role Management Helm chart is installed on IG or SM, a User and Role Management YAML file is not generated for ME subsystems.
