userLocationInformation conversion

In 5G, userLocationInformation is an grouped object and NCC converts it into an octet string. Bits in the octet string depend on the parameters in userLocationInformation.

For 1st octet (Geographic Location Type bit correspondence) the following bits are used:

  • Bit 1: CGI
  • Bit 2: SAI
  • Bit 3: RAI
  • Bit 4: TAI
  • Bit 5: ECGI
  • Bit 6: LAI
  • Bit 7: Macro eNodeB ID
  • Bit 8: Extended Macro eNodeB ID
For subsequent octet(s) the corresponding value is used. For example, for ECGI, userLocationInfomation contain eutraLocation, nrLocation, and n3gaLocation. However, for CDR only one of these is used in the following priority order:
  1. eutraLocation - Only egci is printed. If ecgi is present, the octet string from plmnId and EutraCellId is printed.
  2. nrLocation - Only ncgi is printed, that is, if ncgi is present the octet string from plmnId and NrCellId is prented.
  3. n3gaLocation - Tai is printed if it is present.

CDR representation

userLocationInformation :: 1004F4110225BD6007, where

  • 10 - 00010000(5th bit set for ECGI)
  • 04F411 - MCCMNC Value (MCC2, MCC1, MNC3/F, MCC3, MNC2, MNC1)
  • 0- Spare
  • 225BD6007- eutraCellId
