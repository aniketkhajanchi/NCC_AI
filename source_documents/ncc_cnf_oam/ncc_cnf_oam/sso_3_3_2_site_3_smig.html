SM/IG 3

  1. Make note of the revisions of the Helm releases. They will be useful in case of rollback.
  2. NCC is at Helm revision 1.
  3. Perform a backup of MariaDB for User and Role Management.
  4. Helm upgrade NCC. These charts pick up SSO/LDAP configurations:
    1. chf-routing will make Servicemanager inaccessible by returning HTTP code 503 (service unavailable) to all incoming traffic.
    2. chf-resource will create SE/VS/DR for the remote identity provider.
    3. CKEY-CMDB.
    4. CKEY.
    5. sps-load and btel-load releases will upload sps-auth and btel-auth realms with SSO/LDAP configurations to User and Role Management.
    6. Metrics Viewer.
    7. Servicemanager.
    8. CGF (DR) chart (Helm releasename=dr) in case of IG system.
  5. NCC is at Helm revision 2.
  6. In the Inputs.xlsx file, set these properties and regenerate the Helm overrides files:
    1. Deployment Type = in-service upgrade in the Common tab's Global section.
    2. MariaDB Replication Enabled to false in SM/IG 3 tab's CKEY section.
    3. For SSO: Enable SSO = true in the Common tab's Global section.
    4. For LDAP: Enable = true in the Common tab LDAP - Authentication section.
  7. Upgrade NCC.
    1. chf-routing makes the Servicemanager accessible again for incoming traffic.
    2. MariaDB replication across sites is disabled.
    3. Backup cronjobs will be enabled for MariaDB for User and Role Management.
  8. NCC is at Helm revision 3.
  9. From the User and Role Management GUI, import users who are eligible to login directly to the application GUIs when authentication protocol service is inaccessible. For more information on importing users, see Import the exported users.
Note:
  • These items are impacted when the Servicemanager is inaccessible.
    • All provisioning traffic.
    • Servicemanager related REST API calls.
    • Servicemanager GUI access.
    • SS7 USSD requests are impacted because of its dependence on Servicemanager's service name.
    • SS7 communicates with Servicemanagers through its service name since performance degradation happens when primary and secondary Servicemanager FQDNs are used.
  • When the User and Role Management GUI is down, authentication requests and all GUIs are inaccessible.
  • When User and Role Management is down, CGF (DR)'s dr-ui pods try to reach User and Role Management and crashloopback after seven failed attempts. If that happens, delete and recreate those dr-ui pods.
  • When Reimport DR realm = true and Deployment Type = in-service upgrade in the Inputs.xlsx file, CGF (DR) chart (release name=dr) reupload DR realm to User and Role Management when an in-service upgrade is done. To avoid reimports in future in-service upgrades, set Reimport DR realm = false.
  • When Deployment Type = in-service upgrade in the Inputs.xlsx file, backup cronjobs are enabled for the MariaDB for User and Role Management.
