Supported logs

Log aggregation

Logs are processed by Fluentd and routed to Log Manager for processing by OpenSearch. They can then be viewed using the OpenSearch Dashboard. NCC utilizes the unified logging format, which is in JSON, and the logs are dumped at stdout and available in the containers. The standardized format helps in identifying the alarms and logs which are the same across all pods. Container logs can be fetched from the host machine.

Optional NCC Telemetry package

An optional component, the NCC Telemetry package, may be installed. NCC Telemetry supports components like telemetry logs, alarms, and metrics. If the operator chooses to forgo the NCC Telemetry package, the related components would not be accessible. The operator may need to facilitate log details when NCC is installed without the NCC Telemetry component.

You may choose to configure your own telemetry solution to capture telemetry logs with NCC.

The NCC Telemetry package offers a VES (Vendor Event Stream) option for streaming logs, alarms, and metrics off-board into a single event stream per the VES specification (supporting ONAP, Open Network Automation Platform), to be consumed by the operator's VES collector. To enable this option, see the CVEA section of the Input Generator spreadsheet.

Unified logging

NCC provides the logs in the same format for alarms and logs.

The format for all logs is as follows:
  • type: The type can be one of "log", "alarm", or "counter".
  • level: The following logging levels are supported:
    • 0=EMERG
    • 1=ALERT
    • 2=CRIT
    • 3=ERROR
    • 4=WARN
    • 5=NOTICE
    • 6=INFO
    • 7=DEBUG
  • process: Name and/or ID of the process producing the event.
  • service: Name of the service to which the event is related. For service in NCC, this field is correlated with the existing TAG field that is used in the SYSLOG to differentiate logs coming from the different applications.
  • system: System name of the system in which the event occurred.
  • systemid: System identifier assigned to the system in which the event occurred.
  • time: The local host time at which the event occurred.
  • timezone: Timezone of the local host's time.
  • message: The text of the log message.
  • process: Name of the program language method where the log is issued.
  • classname: Nome of the program language class where the log is issued.
  • stacktrace: A report of the active stack frames at a certain point in time during the execution of a program.
