Rule System Versioning

All rule groups and variables are part of a container called a Rule System Version (RSV). The RSV container supports rule operations such as rule or variable authoring, cloning, exporting, and importing. This is essential for the purpose of the rule logic being modified. An RSV is created on the SM subsystem and then the entire RSV is distributed to the ME subsystems.

When an RSV is activated, all rule groups that belong to the rule system version are evaluated and the rules can be executed by the Rule Engine as triggered in an order of precedence.

NCC supports the export and import of the entire RSV between NCC systems running the same version.

A read-only Default RSV is created automatically when NCC is started for the first time after installation. The Default RSV contains all of the triggers and rule groups that are required for the startup of an NCC and appears in the GUI in the Active state.

Any of the Draft versions or Release versions can be deleted. An Active rule system version cannot be modified or deleted.

The general workflow for the RSV is:
  • Create or modify rules in a Draft version.
  • Change the Draft version to a Release or an Active version.

    All rules attribute dictionaries, provisioned/predefined rules, and rule variables are loaded when a version is promoted to the Active version.

  • If necessary, change the Release version to the Active version.

    All rules attribute dictionaries, provisioned/predefined rules, and rule variables are loaded when a version is promoted to the Active version.

  • Provision and activate the RSV on each geo-redundantly related ME, as necessary.
  • Create subsequent versions as Draft versions and change the versions as required.

Only one rule system version can be in an Active state at any time. When a Release version is promoted to the Active state, the last Active version is automatically demoted to a Release state.

For more information about RSVs and provisioning rules and variables, see the Charging Rules section of the Charging User Guide.
