Repackaging an NS package

This procedure provides instructions on how to repackage an NS package. This procedure is used if the deployment of the NS fails or the installation of Helm charts fails.

  1. Navigate to the /output/<elementName>/examples/NCOM directory of the input generator.
  2. Edit the MANIFEST file.
    Remove the entire charts section in {tosca_name:....}, including the comma at the end.
    For example, if you do not need chf-test-certs, remove this section from the file.
    {
 "tosca_name": "chf-test-certs",
 "name": "chf-test-certs",
 "version": "22.0.1-109",
 "release_name": "chf-test-certs",
 "Values": "values_chf_test_certs"
 },
    Note:
    Ensure that you address other possible dependencies. That is, remove the tosca_name from the dependencies list in other charts.
  3. Ensure that the MANIFEST maintains its JSON structure and run the generate-network-servicedescriptor.sh script to generate a new NetworkServiceDescriptor(rendered).zip file.
