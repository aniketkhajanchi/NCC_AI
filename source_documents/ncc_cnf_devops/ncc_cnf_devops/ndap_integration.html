NCD Integration

Note:

If a Nokia Continuous Delivery (NCD) integration is required after the CCTF installation, the ingress controller (citm-ingress) must be customized.

There are two options for customizing citm-ingress:

  • Option A) Changing the citm defaultSSLCertificate parameter
  • Option B) Using scope concept of citm-ingress

Option A) Changing the citm defaultSSLCertificate parameter

Note that:

  • After the CCTF registration to NCD, TLS will be signed by NCD. And to be used by the CCTF instance, changing defaultSSLCertificate would impact other applications since the default TLS was changed.
  • A secret cctf/cctf-ndap-secret is created by CCTF automatically. Format of the secret: <cctf namespace>/<cctf release name>-ndap-secret.
  • If citm-ingress is already installed, delete it, and then install it in the same namespace as CCTF (here: cctf)

Example command:

helm install csf-helm-stable/citm-ingress --set global.registry=<local docker registy>,controller.defaultSSLCertificate=cctf/cctf-ndap-secret -n citm-ingress --namespace cctf`

Option B) Using scope concept of citm-ingress

Note that:

  • TLS ceritification change impacts only applications that are in the same namespaces/scope of cctf/citm-ingress.
  • Some port configuration might need to be changed if they are already used by any other ingress controller instance. Default values: 10254/80/443.

Example command:

helm install csf-helm-stable/citm-ingress --set global.registry=<local docker registy>,controller.defaultSSLCertificate=cctf/cctf-ndap-secret,controller.scope.enabled=True,controller.healthzPort=10254,controller.httpPort=8080,controller.httpsPort=8443 -n citm-ingress  --namespace cctf
