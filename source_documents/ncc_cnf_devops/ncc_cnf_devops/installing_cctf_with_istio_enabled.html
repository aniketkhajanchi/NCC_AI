Installing CCTF with Istio enabled

This procedure describes how to install CCTF with Istio enabled.

  1. Get the CCTF Helm chart (part of the NCC delivery).
  2. Create a namespace cctf: 
    kubectl create namespace cctf
  3. Label the namespace for auto injection: 
    kubectl label namespace cctf istio-injection=enabled
  4. Create an overrides file cctf-overrides.yaml, with the given content: 
    cat <<EOF > cctf-overrides.yaml
# If istio enabled, please also set cmdbmariadb.istio enabled and jenkins.istio.enabled to true
istio:
  # version of istio available in the environment, support 1.4 or >=1.5
  version: 1.9
  enabled: true
  cni:
    enabled: true
  mtls:
    enabled: true
  permissive: false
  virtualservice:
    hosts:
    - 'espoo-cctf.com'
  gateway:
    # Name of the istio gateway.
    name: cctf-gw
    # Should this be enabled or not.
    enabled: true
    hosts:
      - 'espoo-cctf.com'
      - 'espoo-jenkins.com'
ingress:
  enabled: false
ncd:
  enabled: false
cmdbmariadb:
  global:
    istioVersion: 1.9
  istio:
    enabled: true
jenkins:
  Master:
    RunAsUser: 1000
    FsGroup: 1000
    Ingress:
      enabled: false
  rbac:
    enabled: true
  istio:
    version: 1.9
    enabled: true
    mtls:
      enabled: true
    cni:
      enabled: true
    permissive: false
    sharedHttpGateway:
      name: cctf-gw
    virtualservice:
      hosts:
      - 'espoo-jenkins.com'
  Agent:
    runAsNonRoot: true
threeButtonsAPI:
  enabled: false
storageSize: "2Gi"
EOF
  5. Install CCTF, with the values defined in the override file: 
    helm install cctf-<version>.tgz --name cctf --namespace cctf -f cctf-overrides.yaml
