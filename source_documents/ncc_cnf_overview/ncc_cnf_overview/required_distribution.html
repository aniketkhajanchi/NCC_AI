Required distribution

NCC has special cases where pods are required to be separated, and not being able to satisfy this requirement stops deployment. This is anti-affinity with requiredDuringSchedulingIgnoredDuringExecution.

Aerospike uses this for database racks. Each pod is part of either rack 1 or rack 2 - where Aerospike internally makes sure every piece of data has a copy in both racks. Two pods of different racks are not permitted on the same worker node. This makes sure you do not get both copies of a database record on the same Kubernetes worker node.

Kafka and Zookeeper require this distribution of their pods. This separation ensures that the data and backup are not on the same worker node.
