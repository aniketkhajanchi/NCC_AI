Instantiating test package before test execution

A test package in CCTF is SUT independent, which makes it usable for different SUTs without needing to make changes to the test cases. To achieve this, a parameter file parameters.yaml is introduced with the SUT configuration information and other parameters used by each test case. Only when the data items in this file are filled with correct data for a specific SUT, the test cases will be triggered for execution. The users can edit or update the data items in this file through the CCTF UI.
