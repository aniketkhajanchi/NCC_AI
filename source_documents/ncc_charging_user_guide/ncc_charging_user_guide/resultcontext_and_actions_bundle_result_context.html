BUNDLE result context

Table: BUNDLE result context
Attribute name Trigger Description
Activation-Fee BUNDLE

It is used to set the activation fee for a BOU bundle.
Activation-Fee-Usage-Counted BUNDLE

Provides the amount of BOU activation fee to be added to the monetary counters that count the monetary usage of the account, and for threshold calculations.

The BOU activation fee is deducted as applicable but the value of BOU activation fee added to the monetary counters that count the monetary usage is based on the value of the attribute Activation-Fee-Usage-Counted.
Add-Subscription BUNDLE It is used to attach a subscription.
THRESHOLD
Bundle-Name BUNDLE It is the new bundle name to subscribe.
THRESHOLD
Discount-Percentage BUNDLE It is used to set the discount for a BOU bundle.
Activation-Event BUNDLE It is used to set the activation event for the BOU bundle.
Set-Activation-Count BUNDLE It is used to count the number of times a BOU bundle is activated.
Tariff-Time-Change BUNDLE It is the tariff switch time.
Align-With-Account-BC BUNDLE It is used to configure the BOU bundle to refer to billing information of the linked account for end datetime calculation.
Tariff-Time-Change-Multiple BUNDLE It is the multiple tariff change time points configured as the Tariff-Time-Change-Multiple in bundle rules and considered for all subscriptions which may be used in the call.
Grace-Period-Bucket BUNDLE

It defines the name of the grace bucket in the bundle rule. The bucket is a normal bucket of type volume and it is defined in tariff rules in charging logic.

If configured as BUNDLE.Grace-Period-Bucket = "<Bucket-Name>" in bundle rules, it triggers call logic to send grace bucket name to ChargeFeeAction. This action checks the status of bucket, that is, whether it is exhausted or not.

Sample rule:
BUNDLE.Activation-Fee = 8 and BUNDLE.Activation-Event = "activateBOU" and BUNDLE.Grace-Period-Bucket ="GB1"
Location-Counter-Status BUNDLE It returns the Location Policy Counter Status that is configured in bundle rule.
Roaming-Status BUNDLE It returns the Roaming Status that is configured in bundle rule.
Applicable-Period-Type BUNDLE This attribute indicates the applicable period type of the bundle. It can take the following values:
  • Daily
  • Continuous

Default value: Daily
Special-Date BUNDLE It returns the special date (for example: birthday) from the device string type custom data.

Multiple values can be configured using the parameter Multi Value. If Multi Value is enabled, then multiple dates can be configured for the device.

It is returned as a string in the following format:
  • "YYYYMMDD" (for example: 19800308)
    Note:
    Year information (YYYY) is not considered for validity check.
  • "MMDD" (for example: 0308)
Note:
If Special-Date is configured, then the applicable time frame is from 00:00:00 of the special date to 00:00:00 of the next day. Hence, if Special-Date is configured, then Week-Days, Applicable-Period-Type, and Start-End-Time attributes are not applicable.
Week-Days BUNDLE This attribute is a string which includes applicable week days of the bundle. Week days are separated with a delimiter ";".
For example:
  • "1;2;3;4;5" indicates from Monday (1) to Friday (5).
  • "0;6" indicates weekend. 0 means Sunday, and 6 means Saturday.

If Applicable-Period-Type attribute is continuous, then only the starting day of the period and ending day of period can be provided.

For example: "0;3" indicates from Sunday (0) to Wednesday (3). That is, the period includes Sunday (0), Monday (1), Tuesday (2), and Wednesday (3).
Start-End-Time BUNDLE

This attribute is a string which includes both start time and end time in following format:

"10:00:00,14:00:00" indicates that the applicable time is from 10:00 AM to 14:00 PM in a day.

Delimiter ";" is used to separate multiple time periods. For example: "10:00:00,14:00:00; 16:00:00,19:00:00" which indicates that the applicable time is from 10:00 AM to 14:00 PM, and from 16:00 PM to 19:00 PM in a day.

Note:
  • If Applicable-Period-Type attribute is continuous, then only one pair of start time and end time can be defined in the Start-End-Time attribute.
  • Overlap is not allowed when multiple time periods are configured.
  • Seconds are optional while defining the start time and end time.

If this attribute is not configured but Week-Days is configured, then start time is set to 00:00:00 of the day and end time is set to 00:00:00 of the next day.

For example: If Week-Days is set to “1;3”, Applicable-Period-Type is set to Daily, and Start-End-Time is not configured, then the period starts from 00:00:00 of Monday, and ends at 00:00:00 of Tuesday, then re-starts from 00:00:00 of Wednesday and ends at 00:00:00 of Thursday.

If only one timepoint (either start time or end time) is configured, then it is considered as start time, and the end time is set to 00:00:00 of the next day.

For example: If Week-Days is set to “1;3”, Applicable-Period-Type is set to Daily, and Start-End-Time is set to ”23:00:00”, then the period starts at 23:00:00 of Monday and ends at 00:00:00 of Tuesday, then re-starts from 23:00:00 of Wednesday and ends at 00:00:00 of Thursday.
VT-And-Time-Period-AF-Factor BUNDLE This attribute is the randomization factor (second) which is used to adjust VT and SNR triggering time based on the start and end time of the special period of the bundle.

This attribute is mandatory for special period bundle to spread out the traffic in NCC and PCRF.

In a Gy/5G call, if special period bundle is applicable, and the VT spans the end of special time period, then the VT is adjusted as follows:

VT = (end of special time period – call/current time) + RAND ( 1, VT-And-Time-Period-AF-Factor)
Start-Time-Impact-VT BUNDLE This attribute determines whether the bundle start time of the special period is the candidate VT time.

If the special period bundle is not applicable to the call, but Start-Time-Impact-VT is set to true, then start time of the special period is used for VT calculation.

If In-Advance-Mode is set to false:

VT = (Start of special time period – call/current time) + RAND( 1, VT-And-Time-Period-AF-Factor)

If In-Advance-Mode is set to true:

VT = (Start of special time period – call/current time - VT-And-Time-Period-AF-Factor) + RAND( 1, VT-And-Time-Period-AF-Factor)
In-Advance-Mode BUNDLE This attribute determines whether period start time should be one VT-And-Time-Period-AF-Factor in advance.

For example:

If Start-End-Time is “10:00:00,14:00:00”, VT-And-Time-Period-AF-Factor is 60 seconds, and In-Advance-Mode is true, then the period starts from 09:59:00 and ends at 14:00:00.

By default, the In-Advance-Mode is set to true.

Override-TimezoneId BUNDLE This attribute overrides the timezone at the time of BOU activation.

During BOU activation, Override-TimezoneId should be provided in the lifecycle action Reset Period Action to calculate BOU periodicity.

For example:
  • BUNDLE.Override-TimezoneId = "Europe/Prague"
  • BUNDLE.Override-TimezoneId = VARIABLES.Get-Variable ( "timezoneId")
  • BUNDLE.Override-TimezoneId = @timezoneID

This attribute is applicable only for BOU Activation Priority = ON_COMMIT type of BOU, and BOU Usage Priority = DEFAULT_SORTING_ORDER.
