Apply percentage discount on call

This feature allows operators to apply discount for a call. This can be configured in tariff. The RATING.Call-Discount action can be configured in tariff that contains the discount applicable for the call.

The discount is applied on the following configurations of the bundle if the priority order of the bundle is equal to or less than the bundle in which the discount is configured. The discount value represents the percentage value to be applied on call cost.
  1. Rate configured with the main balance, money-bucket and cost-bucket.
  2. Step-fee.
  3. Rate's connection cost.

Example:

The following example shows the configuration of Call-Discount. In this, the TARIFF:RATING.Call-Discount =30 is configured which indicates that 30% of discount is applicable for all monetary resources which comes in priority after the current bundle.

Discount on rate enhancement

If multiple discounts are active on the subscriber then the enhancement varies on the parameters, either Exclusive or Cumulative.

If... Then...
Exclusive The discount selection is based on discount with a higher priority. Only one discount is applied on the call.
Cumulative The cumulative of the discounts are applied.
Note:
If Exclusive is selected and multiple discounts have the same priority, then the selection is purely based on the Charging logic (charging service) priority rules.

Constraints on discount

Following are the constraints on discount:
  • The discount can be a decimal or integer value.
  • The discount value can be in the range 0-100 (excluding 0). Decimal values are supported. If a wrong value is configured, then the following error message appears: CHARGING_SERVICE_CALL_DISCOUNT_VALUE_NOT_IN_RANGE_ERROR: Invalid Configuration for Call-Discount: Tariff must have Call-Discount action value in range 0-100".
  • This discount is not applicable for non-monetary buckets.
  • This discount is not applied on secondary as well as external rate.
  • This discount is not applicable for BOU bundles including activation fee.

Example

A Device D1 is subscribed with four bundles having the following configuration:
  • Bundle B1 has CS1 with step bucket SBkt1. The priority of CS1 is highest among all.
  • Bundle B2 has CS2 with tariff having only call discount as 50. This is promotional bundle having priority lower than CS1.
  • Bundle B3 has CS3 with step bucket SBkt2. The priority of CS3 is lower than CS2.
  • Bundle B4 has CS4 with main balance. The priority of CS4 is least among all.

When a call is executed, CS1 is picked and since discount has not been evaluated yet, discount is not applied and the cost is deducted as per step fee. Once CS1 is exhausted and CS2 having discount configuration is evaluated. From now onwards, whenever charging happens either from step bucket SBkt2 or main balance, 50% discount is applied. The discount can be configured in any tariff pass. Hence, once a discount is encountered, all the bundles in all passes become eligible for discount.
