Configuring output file name pattern

The file name pattern for output files generated by the File output stream is defined with ASN.1 Encoder Node parameter OutputFileNameCreationMask. To configure the file name pattern, set this parameter for ASN.1 Encoder Node in File output stream.

Parameter OutputFileNameCreationMask can have the following variables:

Variable Description
&LAST(batchSeq) The batch/file sequence number (format defined in CG Mapper Node).
&LAST(batchSize) The number of output records in batch/file.
&LAST(batchCreation) The batch/file creation timestamp (format defined in CG Mapper Node).
&LAST(outputIdentifier) The part of file name defined in distribution rule configuration. If no value has been configured, this part will be empty.
@&LAST(suffix) The file name suffix used when transferring the file. This must always be defined, and included at the end of the file name mask. When the file has been transferred correctly, the suffix is removed.

The default value for OutputFileNameCreationMask is: 

CHF_-_&LAST(batchSeq).&LAST(batchCreation)&LAST(outputIdentifier)
@&LAST(suffix)

The value at the beginning is the file name prefix (CHF), and this can be changed using input generator. To change the value, modify recordFileNamePrefix in the cdrStreams section. For example, setting the recordFileNamePrefix value to NCC would change the value of OutputFileNameCreationMask to:

NCC_-_&LAST(batchSeq).&LAST(batchCreation)&LAST(outputIdentifier)@&LAST
(suffix)
