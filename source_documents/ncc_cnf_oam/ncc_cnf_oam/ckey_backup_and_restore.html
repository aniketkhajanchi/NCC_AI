User and Role Management backup and restore

These instructions assume that User and Role Management has been installed with NCC Backup and Recovery enabled. For more information, see the Deploying User and Role Management Helm chart section of the Installation and Upgrade Guide.
Note:
NCC Backup and Recovery Rest API calls must be called from the admincli container.
Note:
Kubectl commands must be executed from a control node. Kubectl commands may not be executed from the admincli container.

For both backup and restore operations, set the following variables.

CBUR_MASTER_NAMESPACE=<cbur_master_namespace>
NAMESPACE=<namespace>
RELEASENAME=<releasename>
Where:
  • CBUR_MASTER_NAMESPACE is the NCC Backup and Recovery master namespace. For OpenShift deployments, the app namespace NAMESPACE must be the same as CBUR_MASTER_NAMESPACE or backup and restore will fail.
  • NAMESPACE is the app namespace of the application to be backed up or restored.
  • RELEASENAME is the name of the release.

For example:

CBUR_MASTER_NAMESPACE=sps-btel-ig   #For Openshift, the app namspace MUST be the same as cbur-master namespace, or backup and restore will fail
NAMESPACE=sps-btel-ig
RELEASENAME=ckey

To backup User and Role Management

  1. Initiate the backup procedure by running the following command on the admincli container.

    curl -v --header 'Accept: application/json' -X POST http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/backup/$NAMESPACE/$RELEASENAME?helm_version=3

  2. By default, the NCC Backup and Recovery is installed with Celery/Redis to support asynchronous functions. Rest API calls return a taskId which may be used to query the status of the request with another Rest API call, such as the following which is run on the admincli container.

    curl -v --header 'Accept: application/json' -X GET http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/task/7b59914d-f521-41552b81b-765a45f85a42
To restore User and Role Management

  1. Initiate the restore procedure by running the following command on the admincli container.

    curl -vkL --header 'Content-Type: application/x-www-form-urlencoded' --header 'Accept: application/json' -X POST -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/restore/$NAMESPACE/$RELEASENAME?helm_version=3

  2. By default, the NCC Backup and Recovery is installed with Celery/Redis to support asynchronous functions. Rest API calls will return a taskId which may be used to query the status of the request with another Rest API call, such as the following which is run on the admincli container.

    curl -v --header 'Accept: application/json' -X GET http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/task/7b59914d-f521-41552b81b-765a45f85a42

  3. After the restore completes, delete all User and Role Management pods, which are identified by the labels mentioned in the -l flag. The pods will be restarted automatically by the Statefulset. Run the following command from a control node, not the admincli container, to delete all User and Role Management pods.

    kubectl delete pods -l "app.kubernetes.io/name=ckey-ckey" -n ${NAMESPACE}
