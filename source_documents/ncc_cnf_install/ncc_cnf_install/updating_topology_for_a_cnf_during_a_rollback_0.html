Updating topology when rolling back an addition of networked 1+1 ME pairs

This procedure provides instructions on how to update topology for any system or NCC Telemetry instance when you roll back an addition of networked 1+1 ME pairs.

CAUTION
This is a service-impacting action. Deleted pods can cause call drops.
If you are using the split chart:
Using the Helm charts and YAML files of the initial deployment, preform these actions:
  • Follow the procedure in Installing Helm charts manually to obtain the commands and perform a Helm upgrade to rollback the NCC Telemetry charts for a shared NCC Telemetry Design. If non-shared NCC Telemetry design was used, then NCC Telemetry charts were never updated and do not need to be rolled back.
  • Follow the procedure in Installing Helm charts manually to obtain the SM/ME/IG commands and perform a Helm upgrade to rollback the Helm charts. For SM/IG, the sps-load chart was not upgraded and does not need to be rolled back.
    Note:
    Use the original deployment files.
