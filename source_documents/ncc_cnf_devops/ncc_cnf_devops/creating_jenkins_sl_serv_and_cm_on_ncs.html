Creating Jenkins slave service and ConfigMap on NCS

This procedure describes how to create a Jenkins slave service and ConfigMap on NCS.

Execute the following commands from the Kubernetes NCS control node as root.

  1. Create an override file jenkins-slave-and-configmap.yaml with the given content: 
    cat <<EOF > jenkins-slave-and-configmap.yaml
apiVersion: v1
kind: Service
metadata:
  labels:
    jenkins: slave
  name: cctf-jenkins-slave
  namespace: cctf
spec:
  type: ClusterIP
  ports:
  - name: slavelistener
    port: 11200
    protocol: TCP
    targetPort: 11200
  - name: slavelistener2
    port: 11005
    protocol: TCP
    targetPort: 11005
  selector:
    jenkins: slave
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: citm-citm-ingress-tcp-11200
  namespace: cctf
data:
  11200: "cctf/cctf-jenkins-slave:11200"
  11005: "cctf/cctf-jenkins-slave:11005"
EOF

    In this file, the Jenkins slave service name is set to cctf-jenkins-slave and ConfigMap name to citm-citm-ingress-tcp-11200, both having the namespace cctf. Port 11005 is the default port of the HTTP2 test server. If needed, you can define additional ports.

  2. Create the Jenkins slave service and ConfigMap in namespace cctf using the override file: 
    kubectl create -n cctf -f jenkins-slave-and-configmap.yaml
  3. Verify the creation of the Jenkins slave service and ConfigMap:

    1. Check that the created Jenkins slave service exists in the cctf namespace: 
      kubectl get svc -n cctf
