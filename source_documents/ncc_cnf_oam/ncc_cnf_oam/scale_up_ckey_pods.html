Scale up ckey pods

When the replication status shows all of the SMs are in-synch, scale up the CKEY pods on SM4.

## Scale up ckey pods:
kubectl  -n $NAMESPACE scale statefulset $CKEY_STATEFULSET --replicas=$CKEY_REPLICAS

After the CKEY pods are all up and running, you should be able to login to the User and Role Management GUI by using the FQDN specified for SM4, and you should see all of the realms restored, as before the disaster occurred.
