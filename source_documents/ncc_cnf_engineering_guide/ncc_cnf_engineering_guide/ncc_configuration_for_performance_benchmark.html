NCC configuration for performance benchmark

Overview

This section includes characteristics related to the call processing and resources used for performing benchmark results in a Nokia lab environment (unless specified otherwise.) Your environments may have different configurations which can lead to different performance characteristics. As such, the information provided in this section can only be used as a reference on how NCC will perform on a given system.

Feature configuration

Default feature lists are as below unless noted otherwise:

Subscriber model

  • Each device has one bundle. Bundle has 3 buckets, and 3 counters and charging is done from 1 bucket and 1 counter is updated.
  • Total of one million instances of each object are populated to test.

Service model

The Service model used is as follows:

  • Default RSV
  • Charging Service

    The Charging Service used was named as CSD1 with priority 10. This Charging Service is comprised of three buckets of Unit Type as Byte each. Charging Step, Carry-Over, and Threshold were not selected in any of the attached buckets. Three Counters of Unit Type as Byte and Counter Type as Usage were defined. Thresholds were not defined, and these were not OCS Policy Counters.

    The Tariff defined for this Charging Service has only one rule with the following definitions:
    • RATING.Bucket-Selection = " Volume3_Bucket_Bytes"
    • RATING.Counter-Selection = " Volume3_Counter_Bytes"

    The amount is deducted from one bucket and for each deduction, one counter is updated.

  • Slicing profile
    Rule1 is defined in slicing profile. This rule defines the different parameters which are as follows:
    • DYNAMIC_SLICE.Algo-Name = AlgoName_DYNAMIC_ALGO and
    • DYNAMIC_SLICE.Max-Number-Of-Devices-In-Group = 500 and
    • DYNAMIC_SLICE.Validity-Time = 72001

    Algo-Name – Dynamic Slicing Algorithm was used for dimensioning calculations.

    Max-Number-Of-Devices-In-Group – One Device in One Group

    Validity-Time – Defined to 7200 and load was configured so that the request reaches NCC system before this value exhausts.

  • BundleThe bundle (4G_Data_Subscription) was created with DefaultSubscriptionLifeCycle, the fee of 10 and only one Charging Service (CSD1 defined as above).

Miscellaneous

  • Aerospike TLS ON (earlier, NCC 23.5 Aerospike TLS was OFF)
  • mTLS
  • CDR/EDR: Enabled

Load configuration

  • Load is designed to be high enough to drive system CPU usage to more than 50%
  • Load will last for at least two hours to become stable
