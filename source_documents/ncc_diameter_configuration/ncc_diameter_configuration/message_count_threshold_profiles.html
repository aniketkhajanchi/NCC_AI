Message count threshold profile

The message rate for the message queue backlog is configured as based on the aggregate traffic for all the charging pods. Each charging pod supports a message rate that is the configured rate divided by the number of active charging pods. For example, when there are five charging pods sharing the processing load, 10,000 requests per second is applied as 10000/5=2000 requests per second on each charging pod.

To configure a message count threshold profile, select PROFILES tab in the Overload Protection window. Click + and select Message count threshold profile. Configure the message count threshold profile parameters as described in the following table:

Table: Message count threshold profile parameters

Parameter

Description

Mandatory or Optional
Name Specify the profile name. Mandatory
Lower threshold Specify the threshold, in milliseconds, under which overload can be cleared. For message count, the threshold is in number of messages. Mandatory
Upper threshold Specify the threshold to trigger a major alarm. Mandatory
Resource critical threshold Specify the threshold to trigger a resource critical alarm. Mandatory
