Scale down ckey pods

The first step you need to perform is scaling down the CKEY pods, which you will scale up again once the data in the cmdb database has been restored.

## Determine the number of replicas before scaling down, so we can correctly scale them up once the DB restore is complete. 
## Use this command, where $CKEY_STATEFULSET is the name of the ckey statefulset in your deployment: 
CKEY_REPLICAS=$( kubectl  -n $NAMESPACE get statefulset $CKEY_STATEFULSET -o jsonpath="{.status.replicas}")

## Scale down ckey pods:
kubectl  -n $NAMESPACE scale statefulset $CKEY_STATEFULSET --replicas=0
