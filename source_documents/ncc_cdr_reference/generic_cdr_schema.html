Generic CDR schema

Generic CDRs follow a defined schema. The CDRs have a recordType tag, which is different for different types of CDRs. For SCUR CDRs, the recordType is set as OCSChargingRecord. As the CDR module is used by all NCC modules, there is a generic record module that contains all the records generated by the different NCC applications.

Note:
  • New CDR/EDR tags may be added in every release. Operators must configure their external systems to either ignore these tags or make necessary changes in the system to use the tags.
  • This document presents standard CDR/EDR structures generated by NCC. However, location of some tags may change inside a given group structure for different scenarios.

Schema

--GenericRecord 
GenericRecordTypes
DEFINITIONS IMPLICIT TAGS	::= 
BEGIN
GenericRecord ::= SET
{
  recordType         [0] OCTET STRING,
  recordElements     [1] SEQUENCE OF RecordElement OPTIONAL, 
  recordExtensions   [2] SEQUENCE OF GenericRecordExtension	OPTIONAL
}
GenericRecordExtension ::= SEQUENCE
{
  recordProperty       [0] OCTET STRING OPTIONAL, 
  recordElements       [1] SEQUENCE OF RecordElement OPTIONAL,   
  recordSubExtensions  [2] SEQUENCE OF GenericRecordExtension OPTIONAL
}
RecordElement ::= SEQUENCE
{
  varName    [0] OCTET STRING OPTIONAL, 
  varValue   [1] OCTET STRING OPTIONAL
}
END
