Defining X-button UI layout

The X-button UI layout is defined through the cctf_buttons.json file in the test package, where:

  • the array size of the JSON content in cctf_buttons.json defines the number of buttons displayed
  • testSuiteName defines the name of the test suite in the API. If testSuiteName does not exist, testType is used as the value of testSuiteName
  • buttonLabel defines the button label in the UI. If buttonLabel does not exist, testSuiteName is used
  • tests defines the count of tests in the test suite. If tests does not exist, -1 is used in the UI during the test suite execution.

Example:

The following example cctf_buttons.json file will result in 4 buttons displayed in the UI labeled as NEW FEATURE, PERFORMANCE, REGRESSION and REGRESSION-SET2.

[
  {
      "testSuiteName": "ChargingNewFeaturesSuite",
      "buttonLabel": "NEW FEATURE",
      "tests": 2,
      "command": "robot -i 'FUNCTIONAL' demoRobot/"
  },
  {
      "testType": "PerformanceChargingSuite",
      "buttonLabel": "PERFORMANCE",
      "tests": 2,
      "command": "robot -i 'performance' demoRobot/"
  },
  {
      "testSuiteName": "ChargingRegressionSuite",
      "testType": "Any suite name here will be ignored since testSuiteName exists",
      "buttonLabel": "REGRESSION",
      "tests": 3,
      "command": "robot -i 'REGRESSION' demoRobot/"
  },
  {
      "testSuiteName": "REGRESSION-SET2",
      "command": "robot -i 'REGRESSION-SET2' demoRobot/"
  }
]
