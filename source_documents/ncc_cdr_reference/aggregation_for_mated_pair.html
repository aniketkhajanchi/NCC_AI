Aggregation for mated pair

When a session is moving between different MEs (mated pair), CGF can get an indication from the following CDR tags that the session is moving between MEs:

  • prevMeName: This tag is added to all the generated CDR from the data calls, and its value is the previous ME name.
  • switchOver: This tag is added (with value = 1) if prevMeName tag has a value different from the current ME name. There are the following two scenarios to add this tag:
    • If the application preference CDR Generated per MSCC = true, then this tag is added just in the first generated CDR.
    • If the application preference CDR Generated per MSCC = false, then this tag is added in all the generated CDRs.

Once this happens, the secondary ME starts aggregation directly from recordSequenceNumber ≠ 1 or continues aggregation on top of the previous aggregation when prevMeName is different from meHostName and switchOver = 1.
