Mapping of involved party

The input value has a prefix indicating the type of the involved party. The prefix is “STRING:” followed by either “sip:” or “tel:”. The value after the first colon in the input is to be stored under the correct subfield, depending on the prefix:

  • If the input has prefix “sip:”, the subfield sIP-URI is to be used. For example, input “sip:+16328304020@ims.example.com” results in value “+16328304020@ims.example.com” being stored under <parent>.sIP-URI.
  • Otherwise, the subfield tEL-URI is to be used. For example, input “tel:+16328304020” results in value “+16328304020” being stored under <parent>.tEL-URI.
