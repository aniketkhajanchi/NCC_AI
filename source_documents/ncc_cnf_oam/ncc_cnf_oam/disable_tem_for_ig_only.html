Disable TEM for IG only

Before you start the application server, you need to stop TEM.

Run these command to mark TEM as non-vital:

# This also requires restart of centralmanagement which is already handled in the next step.
            kubectl get cm -n <namespace> | grep "installed"
            # output Example.  chf-configuration-me-installed-components-config   1      3h46m
            # Edit the configmap of the ME subsystem,  “chf-configuration-me-installed-components-config” and make tem vital as “false”.
            kubectl edit cm <config map> -n <namespace>
            "name": "tem",
            "vital": false

Run these commands to start the centralmanagement pod with replica=1 and disable TEM:

# While restoring centralmanagement, make replicas only as 1, not 2. Note, this is done ONLY for centralmanagement and other commands are same.
            # This is because we want some TEM to stop (available only at centralmanagement) but also want to keep 1 pod running of centralmanagement because it is the heartbeat.
            # Create only 1 pod of centralmanagement.
            kubectl scale --replicas=1 -n <namespace> statefulset $DEPLOYMENT_NAME_TYPE-centralmanagement 
            
            # Now, stop the TEM application service on centralmanagement pod. This can be done by running a twiddle command on the centralmanagement pod. 
            # These steps are required:
            kubectl exec <centralmanagement> -c <container> -n <namespace> -- /opt/tpa/bin/twiddle localhost invoke com.nokia.tpapps.spscommon.tem:service=TaskScannerService stop
            kubectl exec <centralmanagement> -c <container> -n <namespace> -- /opt/tpa/bin/twiddle localhost invoke com.nokia.tpapps.spscommon.tem:service=DDTEMTaskScannerService  stop
