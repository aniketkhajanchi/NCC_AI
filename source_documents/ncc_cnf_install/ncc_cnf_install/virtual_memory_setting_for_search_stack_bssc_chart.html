Virtual memory setting for Search Stack (BSSC) chart

The virtual memory setting should be set to 262144 on all nodes. To set the virtual memory maximum map count automatically for the NCS platform, edit the sysctl.yaml file and set the vm.max_map_count value to 262144.

apiVersion: ncm.nokia.com/v1alpha1
  kind: Sysctl
  metadata:
    labels:
      controller-tools.k8s.io: "1.0"
    name: sysctl-sample
  spec:
    sysctlRules:
      - "vm.max_map_count=262144"

You can also set the virtual memory maximum map count manually by running this command on all worker and controller nodes:

sysctl -w vm.max_map_count=262144

If the controller node has password-less access to all worker nodes via a root user, run this command:

kubectl get nodes --no-headers -l is_worker=true | awk '{print $1}' | while read node; do echo $node; ssh -qn -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null root@$node 'sysctl -w vm.max_map_count=262144 '; done
Note:
  • Replace is_worker=true with the actual worker label if this label is not in the system to identify the worker nodes.
  • For the case of worker node scale-out, upgrade, or any activity that might reset the setting; run the command again on the impacted worker node.

Run this command to validate the VM setting was applied:

kubectl get nodes --no-headers -l is_worker=true | awk '{print $1}' | while read node; do echo $node; ssh -qn -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null root@$node 'sysctl vm.max_map_count'; done
