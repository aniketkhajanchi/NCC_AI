Pod types

For all deployments, NCC delivers a baseline set of pod types. In addition, depending on the operator needs, NCC can deliver NCC Telemetry and also Nokia service mesh (Istio) to provide a telemetry infrastructure and a service mesh infrastructure.

Depending on the number of instances, NCC has three categories of pod types: singleton, redundant and non-scalable, and scalable.

  • Singleton is the type of pod that is present in the system as one pod instance.
  • Redundant and non-scalable pod types have multiple but fixed number of instances in the system that could be active/active or active/standby. If one instance of a redundant pod type fails, there is another instance of this type in the system.
  • Scalable pod types have a number of pod instances that can be scaled in or out as needed to match the need for CPU, disk, and/or memory resources in the system.
