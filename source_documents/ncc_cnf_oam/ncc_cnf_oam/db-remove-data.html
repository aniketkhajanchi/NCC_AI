db-remove-data

Description

This tool is used to delete data from a set (table) in the Aerospike database. At times, it may be required to empty a given data set. db-remove-data allows users to remove all records from a data set in a simple, efficient manner.

This tool may also be used to clean all data from a given database node the next time that node reboots. This is achieved by setting a flag (which is read during the next node reboot), which removes all database disk files and forces a clean boot (relying instead on the data from peer database nodes to populate the database on that node).

Notice:
This command removes ALL data from the specified data set. Once complete, this action cannot be reversed. Ensure this is what you want to do before running the db-remove-data tool.
Note:
  • db-remove-data does not run if there is an instance running of db-backup, db-restore, or db-remove-data for the same data set. In this case, an error message is displayed.
  • A clean boot is required anytime a node has been out-of-service for more than a couple hours, prior to bringing that node back to service, to ensure that no deleted records are mistakenly inserted back into the database cluster.

Syntax

db-remove-data -s <set> [-n namespace] [-r] [-w <node>]
Code Description
No arguments Displays usage/help information.
-h Displays usage/help information.
-s <set> Specifies the data set (table) from which to remove the data.
-n <namespace> Namespace for the specified set, or all. The default value is all and means to remove data from the given set from all namespaces that include a set by that name.
-r Remove records completely without the durable delete feature, leaving behind no tombstones.
CAUTION:
This can leave the system vulnerable to resurrected records that were previously deleted. This option should therefore NOT BE USED for most situations.

This option should only be used for specific use cases by users who completely understand the impacts.
-c <node> Use this for clean boot. This option wipes all data from the given node (where node is the hostname/podname) when it is booted.
-w <node> Wipe all data from the given node when the database is OOS.

Example

If the Aerospike pod/node name is ig-aerospike-0:
[tpaadmin@<ADMIN CLI POD> /]$/opt/tpa/bin/db-remove-data -c ig-aerospike-0
 
**** CAUTION: This will cause ALL DATA TO BE REMOVED from the node ig-aerospike-0 ****
Once the node has been booted, THIS CANNOT BE UNDONE.
 
Do you wish to continue? [y/n] y
 
Set the key ig-aerospike-0/config/aerospike/datastore/cleanboot to be true in sdc

Verification

sdcctl get system <AEROSPIKE POD NAME>/config/aerospike/datastore/cleanboot
As the Aerospike pod/node name is ig-aerospike-0:
[tpaadmin@<ADMIN CLI POD> /]$ sdcctl get system ig-aerospike-0/config/aerospike/datastore/cleanboot
true
