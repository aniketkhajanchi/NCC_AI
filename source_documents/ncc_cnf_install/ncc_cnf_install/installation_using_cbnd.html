NCC installation with Nokia Cloud Operations Manager (NCOM)

This section contains the procedures for installing NCC with Nokia Cloud Operations Manager (NCOM).

NCOM is a tool that runs outside of NCC and performs Helm lifecycle operations for NCC Helm charts such as install, update, and delete. NCOM is a convenient tool that offers many benefits to its users, including:
  • NCOM provides a centralized view of all the deployed components across namespaces, Kubernetes, and datacenters.
  • The NCOM deployment descriptor allows you to see what versions of different NCC components have been validated together.
  • NCOM automates the installation of Helm charts and eliminates customer interaction with a Helm client.

As the CaaS resource, NCOM uses Nokia Container Service (NCS). To use NCOM with NCC, you need a Network Service (NS) package. The NS package has the format NetworkServiceDescriptor(rendered).zip.

Note:
Before you use the NS package, you must perform this procedure: Generating Network Service Package in input generator.
For more information:
  • about NCOM, see the <NCOM Version x> Operating Guide.
  • about NS package, see the <NCOM Version x> Catalog Packaging Guide.
  • about Network Service Descriptor (NSD) in NS package, see the Network Service Descriptor Developer Guide in Nokia Discovery Center.
