Generating and verifying test/dummy alarms

This procedure can be used to verify your alarm configuration.

  1. Run this command to get the RabbitMQ pod. 
    kubectl get pods -n <btel-namespace> | grep crmq
  2. Run this command to login to the RabbitMQ container. 
    kubectl exec -it < RabbitMQ pod > -n <btel-namespace> bash
  3. Run the rabbitmqadmin command with your credentials and alarms JSON to trigger an alarm.
    Note:
    The values in this example must be changed to match your alarm requirements.
    rabbitmqadmin -u alma_mquser -p alma_mqpasswd publish exchange=cfw routing_key=event payload='{"system": "system","host": "host","service": "service","timezone": "Europe/Budapest","@class": "com.nsn.cam.alma.own.api.event.UnifiedLoggingAlarmEvent","time": "'$(date +%Y-%m-%dT%H:%M:%S.070Z)'","type": "alarm","level": "1","facility": "informational","process": "process","systemid": "systemid","container": "container","version": "1.0","extension":{ "Enterprise": "Enterprise", "VM_Name": "EXT-VM_Name", "VM_UUID": "EXT-VM_UUID", "MOC_ID": "EXT-MOC_ID", "MOC_Name": "MOC_Name", "ves_nfcNamingCode": "EXT-ves_nfcNamingCode", "ves_nfNamingCode": "EXT-ves_nfNamingCode", "ves_sourceType": "port", "ves_category": "EXT-ves_category", "ves_alarminterface": "EXT-ves_alarminterface" },"alarm":{ "id": "17890", "name": "CVEA DEBUG", "severity": "major", "task": "0", "key": "", "text": "", "event-type": "2", "probable-cause": "356", "data": "" }}'
  4. Verify the triggered alarm on the NCC Alarm Management GUI by using this URL format. 
    https://<fqdn>/calm-<element-name>/
