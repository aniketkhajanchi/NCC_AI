Committing the Aerospike persisted version

Prerequisites

  • This procedure must be done after all of the systems have been upgraded to the latest release.
  • Perform this update on one system at a time.
  • Once the database persisted version has been updated on one of the systems, that system cannot be rolled back.

  1. Set the Database Persisted Version value to Empty (no value) in the SPS section of the Common worksheet in the Inputs.xlsx file.
  2. Generate an override YAML file.
  3. Install the chf-configuration Helm chart by using the appropriate tool with the new chf-configuration.yaml override file.
  4. Log into the SM GUI.
  5. Update the SM/IG version for each system to the latest release.
