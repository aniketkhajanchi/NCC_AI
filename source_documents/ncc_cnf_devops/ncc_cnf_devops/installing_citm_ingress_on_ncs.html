Installing CITM ingress on NCS

This procedure describes how to install CITM ingress on NCS.

Execute the following commands from the Kubernetes NCS control node as root.

  1. Create an override file citm-ingress-overrides.yaml with the given content: 
    cat <<EOF > citm-ingress-overrides.yaml
controller:
  service:
    type: LoadBalancer
  # OPTIONAL ->
  #nodeSelector:
  #  kubernetes.io/hostname: <node name>
  #runOnEdge: false
  # <- OPTIONAL
EOF
    Note:

    The OPTIONAL part is needed if you are deploying, for example, NCC and CCTF in the same Kubernetes cluster. By default, CITM is installed on the Edge node. Set runOnEdge to false and choose the node where to schedule CITM by using nodeSelector (for example, Worker node, use kubectl get nodes).

  2. Install CITM ingress with name citm in namespace cctf, and with the values defined in the override file (in step 1): 
    helm install csf-helm-stable/citm-ingress --name citm --namespace cctf -f citm-ingress-overrides.yaml
  3. Verify the installation:

    1. Check that the CITM ingress pod exists in the cctf namespace: 
      kubectl get pods -n cctf

      kubectl get svc -n cctf
