Manual backups

In addition to having scheduled backups, you may need to perform a backup without waiting for a scheduled one. This can be done as a manual operation.

The procedure to perform a manual backup is described in detail for each component. This is a generic example. This procedure is performed by running a curl command from the admincli pod.

# Set variables, for example:
CBUR_MASTER_NAMESPACE=naperil110vzwcchf-y-nk-sm-001   
NAMESPACE=naperil110vzwcchf-y-nk-sm-001
RELEASENAME=ckey
BR_POLICY=ckey-ckey

# Initiate Backup
curl -v --header 'Accept: application/json' -X POST http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/backup/$NAMESPACE/$RELEASENAME?helm_version=3

# In alternative to the command above, which will backup the BrPolicy defined for the specified release, 
# a different URL can be used in the curl command to specify the name of a BrPolicy:

# Initiate Backup
curl -v --header 'Accept: application/json' -X POST http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/backup/$NAMESPACE/$BR_POLICY?helm_version=3
