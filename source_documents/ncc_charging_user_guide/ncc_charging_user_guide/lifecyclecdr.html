Lifecycle EDR

NCC generates lifecycle EDRs to track changes that occur in lifecycle entities. The lifecycle entities are accounts, subscriptions, devices, groups, and users. Every lifecycle EDR contains attributes that identify the triggering event that caused the change and all the information related to the change that took place, including the transitions and states used. Each lifecycle EDR corresponds to one triggering event but might contain records related to more than one entity.

Most commonly the lifecycle EDRs are triggered by lifecycle actions. For more information on CDRs, see the CDR/EDR Reference.
