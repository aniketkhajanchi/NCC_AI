NCC Telemetry data recovery

On SM4, restore the BTEL data from Backup and Recovery backups.

## After BTEL helm charts are installed, restore data from Backup and Recovery backups.
## On SM4, run the following command from admincli pod, where $RELEASENAME is the name of the release you are doing the restore for. 
## You will have to repeat this command for each BTEL component that has Backup and Recovery enabled in your deployment (typically cmdb, cpro and cpro-gen3gppxml), using the proper release name. 
curl -vkL  --header 'Accept: application/json' -X POST --header 'Content-Type: application/x-www-form-urlencoded' -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$NAMESPACE.svc.cluster.local:80/v2/helm/release/restore/$NAMESPACE/$RELEASENAME?helm_version=3

## By default Backup and Recovery master is installed with Celery/Redis to support asynchronous functions, so the restore Rest API calls will return a taskId
## which can be used to query the status of the restore with another Rest API call, like in the following command, that has an example taskId:
curl -v --header 'Accept: application/json' -X GET http://cbur-master-cbur.$NAMESPACE.svc.cluster.local:80/v2/task/7b59914d-f521-41552b81b-765a45f85a42
