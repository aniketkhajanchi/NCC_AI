Reversed TBCD string mapping

Some input fields are sent from Charging application in TBCD string format, which is essentially an octet string with values 0-9, *, #, a, b, c, and where the characters in every octet have been swapped. For example, octet string 123456 is 214365 as a TBCD string.

ASN.1 Encoder Node will do the same TBCD mapping automatically based on the ASN.1 schema of those fields, reverting the changes done in Charging application. To get the TBCD strings out correctly, the mapping function needs to revert the mapping done in Charging application and let the ASN.1 Encoder Node redo it.

For example, an IMSI value coming from Charging application as 21436587 is a TBCD formatted octet string. The mapping code must convert it to 12345678 and ASN.1 Encoder Node will then convert it back to 21436587.
