4G/5G data:
  • If this action is NOT present or set to true, then CDR is generated at every CCR-I, CCR-U, and CCR-T.

4G voice:

  • If Enable-Single-CDR is NOT present and (ADD-Initial-CDR is NOT present OR set to true) then CDR is generated at CCR-I, CCR-U, and CCR-T.
  • If Enable-Single-CDR is NOT present and ADD-Initial-CDR is set to false then CDR is generated only at CCR-U and CCR-T.
  • If Enable-Single-CDR is present and (ADD-Initial-CDR is NOT present or set to false) then a single CDR for the complete call is generated.
  • If Enable-Single-CDR is present and ADD-Initial-CDR is set to true then a CDR for CCR-I and a single CDR for the complete call is generated.

Defines how to perform the aggregation on the custom tag for the output CDR. It requires custom tag name and the level at which it is created as input.

Valid values:.

  • AGGREGATE_UNIQUE: Aggregates the unique values in Pipe separated format into the final CDR. If there is only one value in the custom tag, the output has only one value, and same values is shown multiple times with pipe separation.

Copy-CDR-tag

RATING_POST_PROCESSING

RESOURCE_POST_PROCESSING

This attribute copies the existing tag from the CDR and uses it as custom tag in an other level based on the definition.

Format:

CALL_RESULT.Copy-CDR-tag ( <CUSTOM_TAG_NAME> , CUSTOM_TAG_LEVEL_ENUM , <Existing_Tag_Name> , <Existing_Tag_Name_Level>, AggregationType_ENUM )

Where,
  • <CUSTOM_TAG_NAME> : Name of the custom tag. It is a mandatory parameter.
  • CUSTOM_TAG_LEVEL_ENUM for RATING_POST_PROCESSING: It is a mandatory parameter. It can take the following values:
    • AVPLevel_COMMMAND_LEVEL
    • AVPLevel_MSCC_LEVEL
  • CUSTOM_TAG_LEVEL_ENUM for RESOURCE_POST_PROCESSING: It is a mandatory parameter. It can take the following values:
    • AVPLevel_COMMMAND_LEVEL
    • AVPLevel_MSCC_LEVEL
    • AVPLevel_CHARGING_SERVICE_LEVEL
    • AVPLevel_RESOURCE_LEVEL
  • Existing_Tag_Name: Name of the existing tag in CDR. This name should match with the existing tag name. Otherwise, it is ignored. It is a mandatory parameter.
  • Existing_Tag_Name_Level: It represents the level of the existing tag that is to be copied. It is a mandatory parameter. It can take the following values:
    • TAGLevel_deviceInfo
    • TAGLevel_groupInfo
    • TAGLevel_subscriptionInfo
    • TAGLevel_chargingServiceInfo
    • TAGLevel_counterInfo
    • TAGLevel_accountInfo
    • TAGLevel_bucketInfo
    • TAGLevel_noChargeInfo
    • TAGLevel_MSCCRootElements
    • TAGLevel_CDRRoot
    • TAGLevel_stepInfo
    • TAGLevel_discountInfo
    • TAGLevel_additionalBalanceInfo
  • AggregationType_ENUM: It is an optional parameter. It can take the following values:
    • AGGREGATE_UNIQUE : It is a pipe separated value. For example, 0|2|4.
    • FIRST_VALUE : First value. For example, 0.
    • LAST_VALUE : Last value. For example, 4.
    • AGGREGATE_SUM : Summed value. For example, 6.
Example: CALL_RESULT.Copy-CDR-tag ( "usageRateId" , AVPLevel_COMMMAND_LEVEL , "rateId" , TAGLevel_ENUM_accountInfo , AggregationTyp)

Post-Call-Notification-Handling GY_PRE_PROCESSING This attribute is used to determine whether to send a post call notification.
  • If it is null or Not_Sent, then no post call notification is needed.
  • For PS call, if it is Send_At_CCRu_CCRt, then the post call notification is sent per CCRu/CCRt message
  • If it is Send_At_CCRt_Per_Session, then post call notification is only sent when CCRt message is received.
For other call types, the valid value is Not_Sent and Send_At_CCRt_Per_Session. If it is set to Send_At_CCRu_CCRt, then post call notification is not triggered.
Example: Enum
  • Not_Sent
  • Send_At_CCRu_CCRt
  • Send_At_CCRt_Per_Session
