Enabling MR aggregation

For the MR aggregation to take place, set the following node parameter values:

  • For CG Kafka Collector Node (Kafka input stream)
    • PassThrogh: no (hidden)
    • SequenceNumberFields: recordSequenceNumber
    • MRSessionTimeout: To keep a session ongoing in Redis, set this node parameter to a value greater than the validity time in the NCC side.
    • SessionIdFields: sessionId
    • ChargingRecordTypes: OCSChargingRecord
  • For CG Mapper Node (Business logic stream)
    • RecordSequenceNumberValidation: yes
    • CustomPartialCheckerClass: com.nokia.mc.node.cg.sps.SPSPartialChecker (hidden)
    • GeneratePartialFromStart: yes (hidden)
  • Logic for aggregation tags can be controlled using the following parameters:
    • MRAccumulatedFields: This parameter defines the list of fields that are accumulated during aggregation for MR requests.
    • MRFirstOccurrenceFields: This parameter defines the list of fields that are generated from the first occurrence for MR requests.
    Note:
    • Fields should be added to either MRAccumulatedFields or MRFirstOccurrenceFields. If any field is added to both, it is generated from the last occurrence and a warning is raised on the mapper stream.
    • Any field that is not part of those configurations, is generated from the last occurrence.

Rounding up is disabled by default. It can be enabled by setting CG Mapper Node node parameter MRRoundingEnabled to yes. If rounding is enabled, the number of digits to be rounded can be configured with node parameter MRRoundingDigits, which has a default value of 5. MRRoundingFields defines the list of fields that have to be rounded.
