configure-manual-oos

Description

This command is used to manually declare a subsystem that is in a geo-redundant deployment out of service (OOS). It can set and clear the manual out of service flag for a subsystem (stored in SDC).

When a subsystem is set to OOS by using configure-manual-oos:
  • other subsystems in the geo-redundant configuration see this subsystem as OOS until it is returned to service by using configure-manual-oos -e <subsystem name>.
  • there is a system-wide graceful shutdown of all Diameter links (Client or NCC Initiated).
  • provisioning operations cannot be executed on the OOS subsystem, until it is returned to service.
Note:
On an IG system, this command must indicate whether the request is for the SM or ME subsystem.

Syntax

configure-manual-oos [-d <subsystem name>] [-e <subsystem name>] [-h] [-s]
Option Description
-d <subsystem name> Declares the subsystem manually out of service.
-e <subsystem name> Clear the subsystem from being declared manually out of service.
-h or -? Displays usage/help information.
-s Print each subsystem name and whether or not it is manually out of service.
configure-manual-oos-d ME Example to illustrate that the subsystem name is case sensitive and must be entered as upper case ME or SM.
Note:
Even though the command is always run against the local system, it is still required to specify the subsystem name.

Examples

Configure-manual-oos only produces output when the -s option is used. This output example is expected when running configure-manual-oos -s on an IG that is running with all services UP in a geo-redundant deployment. 
configure-manual-oos -s
------------------
 COMPONENT | MOOS 
------------------
ME: FALSE
SM: FALSE
Note:
The -s prints each subsystem name and whether or not it is out of service.
Table: Output Explanation
Line Description
COMPONENT | MOOS

Header for configure-manual-oos tool results.
ME: TRUE

The corresponding service state, printed in the this format.

<subsystem>: <status>
Where:
  • <subsystem> is the subsystem type. Possible values are ME or SM.
  • <status> indicates whether or not the subsystem is manually out of service. Possible values are TRUE and FALSE.

For IG systems, configure-manual-oos produces entries for both ME and SM subsystems.

This is an example of the output from the configure-manual-oos -d ME command. 
configure-manual-oos -d ME
configure-manual-oos -s
