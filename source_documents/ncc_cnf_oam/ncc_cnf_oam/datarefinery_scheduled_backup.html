CGF (DR) scheduled backup

The scheduled backups for CGF (DR) are not enabled by default. After installation, scheduled backups may be enabled. For more information, see Scheduled backups.

The default backup schedule is as follows.

0 0 * * *

This indicates backups will begin daily at 00:00. The name of the NCC Backup and Recovery policy is cdr-streams-cbur-client.

Update schedules for scheduled backups

To update the cronJob schedules defined at install time for either of the NCC Backup and restore policy, a helm upgrade must be performed to provide the new value for the cronJob schedules. Old values are reused for all other properties. Then, the scheduled backups must be enabled. For more information, see Scheduled backups.

As an example, the following helm upgrade will change the schedules in the policy to run the backup for CSDC daily at 3am.

helm upgrade cdr-streams cdr-streams-chart-*.tgz -n $NAMESPACE --set cbur.cronSpec="0 3 * * *" --reuse-values --no-hooks
