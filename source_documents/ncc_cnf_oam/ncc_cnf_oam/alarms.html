Alarms

Overview

NCC alarms are written to stdout and are processed/directed by Fluentd. They can be forwarded to northbound systems and stored in a database to be indexed, viewed, and searched with standard tools such as OpenSearch Dashboard.

NCC can be deployed with an optional NCC Telemetry package which includes an alarm manager.

The NCC Telemetry package offers a VES (Vendor Event Stream) option for streaming alarms, logs, and metrics offboard into a single event stream per the VES specification (supporting ONAP, Open Network Automation Platform), to be consumed by the operator's VES collector. To enable this option, see the CVEA section of the Input Generator spreadsheet.

Note:
Detailed alarm information is available in the Alarms Spreadsheet.
Note:
For integrating alarms with an external NMS, refer to the Installation and Upgrade Guide.
