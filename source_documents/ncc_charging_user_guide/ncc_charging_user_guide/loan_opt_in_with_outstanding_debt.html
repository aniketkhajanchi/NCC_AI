Loan opt in with outstanding debt

This feature provides the functionality to allow the subscriber to receive a loan (opt in) even when they already have outstanding remaining debt after repayment (opt out). This functionality can be configured using the attribute LOAN_RESULT.Allow-OptIn-With-Debt in the LOAN_PRE_PROCESSING trigger.

Configuration

To enable this functionality, Allow-OptIn-With-Debt attribute in the LOAN_RESULT result context should be set to true. If this attribute is set to true, then only the account loan state is updated from opt out to opt in and the main balance remains the same.

Note the following points about this feature:
  • This feature is applicable only when it is configured.
  • This feature is applicable only for recurrent type loan and single loan Id.
  • Notification is sent when the loan service is activated.
    Note:
    The subscriber is not granted extra loan, and the main balance remains the same. However, the subscriber is notified that the loan has been received upon opt in.
  • If loan Id is not provided in the loan request, rules are configured to opt in to a different loan when the account has outstanding debt, and LOAN_RESULT.Allow-OptIn-With-Debt is set to true, then the account opts in to the same loan from which it previously opted out.
