Preparing to add independent pairs of 1+1 ME subsystems

This procedure provides instructions on how to prepare for the addition of one independent pair of 1+1 ME subsystems to the existing deployment.

  1. Ensure the following:
    • Your system satisfies the installation prerequisites including Python requirements and tool dependencies.
    • The repository charts are downloaded on the system
  2. Update the Inputs.xlsx file as follows:
    • In the Inputs.xlsx file, add two more ME tabs and ensure that the values in the new tabs are configured as needed.
    • In the Common tab, go to the SPS section and update the Partition ID and Partition ID Site Names parameters so that the values are configured in accordance with the resulting deployment.
    • Update the other parameters in theInputs.xlsx file so that the new topology with two additional MEs is reflected in the Inputs.xlsx file. For information on how to prepare the Inputs.xlsx file, see the procedure Preparing Helm chart input values.
  3. Generate the output files using the input generator and transfer the output file to the Kubernetes cluster controller node. For instructions, see the procedure Generating Helm charts input.
    To backup your existing output directory, enter:
    cd <</path/of/input-generator/>>
mv output output_$(date +'%d%b%Y')_$(date +'%H%M%S')
    To upload the Spreadsheet enter:
    scp Inputs.xlsx <user-name@1.2.3.4>>:/path/to/input-generator/
    To generate the installation YAML files, enter:
    ./generate.py --inputs Inputs.xlsx
    To make generated scripts executable, enter:
    find . -type f \( -name '*.py' -o -name '*.sh' \) -exec chmod +x {} \;

    The results are created in the output directory. Each element from your spreadsheet will have a separate sub-directory containing a site.Conf file and all of the YAML overrides needed to perform a deployment.

    The generated output files will contain the new topology that includes new MEs. The new topology has to be added to both the SMs and MEs via helm upgrade. The output file has to be transferred on all the sites where you add new MEs. The commands sftp /scp can be used.
    Note:
    The output files must not contain either any empty line or extra chars.
