Provisioning REST API logs

REST API logs capture requests and responses coming from and going to the upstream provisioning system. In the case of an error, REST API logs log the error from the ME to the SM and from the SM to the provisioning system. Provisioning logs may be enabled in the management console using the REST APIs Tracing feature. Once enabled, REST APIs Tracing leverages a dedicated index provision-tracing to store all provisioning requests and responses coming from and going to the upstream provisioning system.

Under Features in the SM Application Preference panel, REST APIs Tracing enables or disables tracing and logging. These key attributes are shown in these OpenSearch Dashboard log examples.

  • correlational handler (<httpmethod>-<entityname>-<UUID>)
  • eventName
  • operatorName
  • userAgent
  • httpMethod
  • entityName
  • requestData
  • responseBody
  • path/uri
  • exceptionCode
  • exceptionDetailMsg
