Reprocessor stream

Reprocessor stream handles records that have been rejected by the Business logic stream. The stream transfers the rejected records to the SFTP server and brings these records back to the database after the reason for rejection has been identified and fixed. The stream contains the following two nodes:

  • CG Rejection Distributor Node, which transfers rejected records from the database to the SFTP server. The records are transferred automatically whenever the node is running
  • CG Rejection Collector Node, which brings rejected records back to the database from the SFTP server. The node starts transferring records only after its execution has been scheduled. The records stored by CG Rejection Collector Node are processed by Business logic stream in the same way as the records received from Kafka input stream.
