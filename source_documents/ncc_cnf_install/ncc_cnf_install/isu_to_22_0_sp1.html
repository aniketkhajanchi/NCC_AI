NCC in-service upgrade to Release 23.8

Upgrading NCC from the previous release to the current release should be completed via an in-service upgrade (ISU) while traffic is running on the system. There are minimal traffic impacts during service upgrade and rollback, but traffic should stabilize once the procedure is completed.

For deployments with SM or ME; all SM systems must be upgraded first followed by the ME subsystems. For a rollback, the reverse order can be followed.

For deployments with IG and NCC Telemetry; upgrade all IG systems first followed by NCC Telemetry. For a rollback, follow the reverse order.

Before you begin the upgrade, you must ensure all of the ME subsystems are set to the version of the previous release. Perform these steps to verify the correct version.
  1. Log in to the SM GUI.
  2. Navigate to the Platform section.
  3. Verify the version value for each system is the previous release.
If any of the systems do not have the version value set to the previous release, perform these steps to change the value to the previous release for each system.
  1. Select the system you want to edit.
  2. Click the three vertical dots at the end of the row.
  3. Select Change Version.
  4. Click on the New Software Version drop-down field.
  5. Select the previous version.
  6. Click on the Update button.

Strategy to upgrade NCC and Nokia-supplied Istio

  • Upgrade the NCC charts to the latest/target release keeping the API Compatibility version the same as the currently installed Istio version.
    Note:
    Verify the NCC upgrade is successful and tested properly before you upgrade Istio. NCC rollback is not supported after you upgrade Istio.
  • Upgrade the Istio to the target version.
  • Restart NCC on latest release with target Istio version as API Compatibility.
