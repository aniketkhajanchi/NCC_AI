Tiered Rates

NCC provides the functionality to rate and charge the utilities such as water, electricity, and gas. A smart meter collects the usage from each system and sends it to the Nokia data refinery (DR). DR processes the usage information and builds diameter request which is sent to NCC using Immediate Event Charging (IEC) request.

The smart meter can be prepaid or postpaid. For postpaid meters, one reading is received by the end of the month whereas for prepaid meters, a reading is received every 6 hours.

Tiered Rate defines tiered or step-based tariff for smart meter charging.

Configuration

Creating a Charging Logic with Tiered Rate
The following configuration takes effect while creating a Charging Logic with Tiered Rate:
When a tiered rate is referred in Charging Logic in the NCC GUI, then the following rules are automatically added in tariff pass of the Charging Logic:
  • RATING.Tiered-Rate ( "Tiered-Rate-Name", "Target")
  • Rating.Counter-Selection = Counter-Name
Note the following points about Tiered Rate:
  • Only one Tier should be referred during creating/updating Charging Logic.
    Note:
    Referring multiple tiered rates in a Charging Logic can result in validation failure or unexpected call behavior.
  • Tiered rate is also supported for secondary balances.
  • All the rules are mandatory for tiered rate to function as expected.
  • This feature becomes non-functional if both Rate and Tiered Rate are present in rules.
  • At least a single Tier is mandatory in a Tiered Rate.
  • Versioning is not supported in Tiered Rates. Changes to the tiered rates are effective immediately.
  • Name, Type, and Unit parameters should not be updated. Validation rejects any such request.
  • Monthly fee is treated as bundle fee or renewal fee.
  • Total fees should be defined in incremental order.
