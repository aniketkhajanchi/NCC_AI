Table: EDR_PROCESSING source context
Attribute name Trigger Description
Get-Entity EDR_PROCESSING This attribute is used to call entities to be able to fetch details about the entity and populate the information.
Exanple: EDR_PROCESSING.Get-Entity-Type ( EntityType_SPR_DEVICE ) = BooleanEnum_TRUE / EDR_PROCESSING.Get-Entity-Type ( EntityType_SPR_ACCOUNT ) = BooleanEnum_TRUE
Note:
This rule attribute is used to control execution of SPR_ACCOUNT and SPR_DEVICE service context. It is not executed by default for all provisioning and lifecycle event. When any of this attributes are used then either SPR_DEVICE or SPR_ACCOUNT related information are fetched and when both attributes are used then both SPR_DEVICE and SPR_ACCOUNT related information are fetched. If not defined, then the account and device information is not fetched and populated in the EDRs.

For provisioning scenario related to account, EntityType_SPR_DEVICE fetches device information from the first subscription assigned to device and associated to account and for provisioning scenario related to device, EntityType_SPR_ACCOUNT fetches account information associated to the first subscription of the device.

Example

A profile is created with Device, Postpaid Account, user and subscription. Configure Add-EDR rule into EDR-Processing trigger to include the tags for SPR_ACCOUNT as follows:
if
EDR_PROCESSING.Get-Entity-Type ( EntityType_SPR_DEVICE ) = BooleanEnum_TRUE and EDR_PROCESSING.Get-Entity-Type ( EntityType_SPR_ACCOUNT ) = BooleanEnum_TRUE and
EDR_PROCESSING.RECORD_TYPE ( "PROVISIONING_RECORD" ) = true and
EDR_PROCESSING.invokedMethod ( "update_device" ) = true
then
CALL_RESULT.Add-EDR ( "accountBalanceBefore", SPR_ACCOUNT.Available-Balance ) and CALL_RESULT.Add-EDR ( "accountBalanceAfter", SPR_ACCOUNT.Available-Balance ) and
CALL_RESULT.Add-EDR ( "accountStateBefore", SPR_ACCOUNT.Previous-Entity-LifeCycle_State and CALL_RESULT.Add-EDR ( "accountPLCStateBefore", SPR_ACCOUNT.Previous-Period-Lifecycle_State ) and CALL_RESULT.Add-EDR ( "entityLifecycleState", SPR_ACCOUNT.Entity-LifeCycle_State ) and CALL_RESULT.Add-EDR ( "periodLifecycleState", SPR_ACCOUNT.Period-Lifecycle_State )
