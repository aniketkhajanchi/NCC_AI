NCC Telemetry scheduled backup

The scheduled backups for the optional NCC Telemetry components MariaDB, Prometheus, and gen3gppxml are not enabled by default. After installing NCC Telemetry they can be enabled, if desired. Follow the instructions here: Scheduled backups.

The default schedules for the cron jobs that performs scheduled backups for all NCC Telemetry BR policies (cmdb, cpro, gen3gppxml) are set to the same value: "0 2 * * *" which means daily at 2am.

Update schedules for scheduled backups

To update the cronJob schedules defined at install time for any of the BR policies created by NCC Telemetry, an helm upgrade has to be performed, providing the new value for the cronJob schedules, and reusing old values for all other properties.

Then the scheduled backups have to be enabled by following the instructions at: Scheduled backups.

# For example, the following helm upgrade will change the schedules to run the backup for Prometheus daily at 5am
helm upgrade btel-ig-cpro cpro-chart-*.tgz -n $NAMESPACE --set server.cbur.cronJob="0 5 * * *" --reuse-values --wait --timeout=1200s
 
# To make the change effective, the scheduled backups have to be enabled.
# If the scheduled backups were already enabled before doing the upgrade, then they need to be disabled and re-enabled for the changes to the policy to have effect.
