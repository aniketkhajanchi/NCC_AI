Changing and updating the cipher suites

In NCC, the TLS cipher suites are configurable with the parameter global.aerospike.mtls.cipherSuites. The default value of this parameter, which is configured in the values.yaml file, is DEFAULT. This procedure shows how to configure the cipher suites.

  1. Set the parameters in the values.yaml file as follows.

    global.aerospike.mtls.cipherSuites to <VALUE>

    Where <VALUE> is the desired cipher suite value. For example, this example uses AES128-GCM-SHA256 as <VALUE>.

    global.aerospike.mtls.cipherSuites to AES128-GCM-SHA256
  2. Run helm upgrade to apply the changes.

  3. Restart the Aerospike server pods to upgrade the servers.

    Note:
    Aerospike communication uses a certificate for the aerospike-credential secret. When the current certificate expires or is rotated, the ongoing client-to-server connections are not affected. However, new connections require the updated certificate. To apply the updated certificate, the application and Aerospike pods must restart.
