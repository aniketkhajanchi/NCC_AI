Status overview

The Status Overview page enables you to view details of peers for a given ME, and the number of peers that are up and down. It is the landing page for Connectivity app. You can also open this by navigating to Connectivity → ☰ → DIAMETER PEERS → Status Overview. All the available MEs are displayed as tiles.

In any ME tile, click ME OVERVIEW to view information about all the diameter peers for that ME. The following table describes the parameters displayed for peer connection status for the ME:

Table: ME overview
Parameter Description
The following information is displayed at the top of the page.
Active Peers Number of active peers on Managed Elements.
Up Number of peers that are up.
In maintenance Number pf peers that are in maintenance.
Down Number of peers that are down.
The following information is displayed in the main page.
Peer status Specifies whether is diameter peer is UP, DOWN, Active, In maintenance.
Peer name Host name of the diameter peer.
Protocol Protocol being used.
Application Unique identifier for the diameter peer.
Node name DNS host name of the blade/pod that created this object. This is used to clean up the objects when the diameter stack is stopped.
Connection type Direction of the connection. It is either inbound or outbound.
Note:
When the diameter peers are deleted, the peer down status reflects immediately, but the diameter peer entry is not removed immediately. This depends on the scanner which runs in 3600 sec, controlled by peerScanIntr from DiameterConfig. The scanner jobs run every hour (3600 secs) to check the disabled connection and removes the entry from DiamPeerStatus which reflects on the SM GUI peer status.
