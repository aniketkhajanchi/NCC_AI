Aerospike compression parameters

Aerospike supports an option to compress data in its persistent/disk storage. Compression results in less IOPS and disk space. Compression is only applied to records that are written to persistent/disk storage (for example: SSDs or persistent memory (pmem)). Record data in memory (RAM) is not compressed. Changing the compression settings does not affect existing records on storage. It only affects records that are newly inserted or updated after the compression settings are changed. The Inputs.xlsx spreadsheet by default has Aerospike persistent/disk compression enabled. The value is set with the Aerospike Data Compression parameter in the Common > SPS section. The options are either:
  • disk = compression enabled (default)
  • none = no compression
