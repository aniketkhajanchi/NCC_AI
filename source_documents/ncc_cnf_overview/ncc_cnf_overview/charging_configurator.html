Charging Configurator

This section provides the overview of the Charging Configurator in NCC.

Converged charging adds a more intuitive graphical interface for configuring charging, referred to as the Charging Configurator. The charging engineer uses the SM GUI to create the charging service entities, which are provisioned to the MEs. The user creates a charging logic which supports the new user experience for creating charging configurations, and that charging logic information is used to internally generate the charging service which is provisioned to the MEs. This is transparent to the user. The Charging Logic is translated and provisioned when a bundle that references it is provisioned.

A Charging Logic entity defines an assembly of charging components and data to populate the associated component parameters. Components are of two types: conditions and actions.

The Charging Logic GUI supports two authoring modes for users: design mode and advanced (code) mode. The latter is for backward compatibility and allows the user to simply enter rules as done in the legacy GUI.

For more information regarding these components, see the Charging User Guide.
