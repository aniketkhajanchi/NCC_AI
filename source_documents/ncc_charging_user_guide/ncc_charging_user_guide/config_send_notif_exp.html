Configuring Send-Notification action for expiry and pre-expiry notification

Operators can configure the Send-Notification action for sending notifications only for last instance of queue at two different points of time:

  • at Pre-Expiry
  • at Expiry

On the basis of the value provided in condition attributes Subscription.is-Queued-Instance and Subscription.is-Last-in-Queue, expiry and pre-expiry notifications are sent only for the last subscription in the queue.

Note:
If the condition (in rule) fails, System.Do-Nothing must be configured so that notification is not sent.
