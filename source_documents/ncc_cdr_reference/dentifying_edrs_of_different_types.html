Identifying EDRs of different types

Different EDR types can be identified by checking the values in the following input fields:

  • recordType
  • invokedBy
  • invokedMethod
  • triggeredEntityType

If all the input field values (i.e. values found in the EDRs in the NCC generic CDR format) match the values listed in Identifying EDRs of different types, excluding the columns marked with dash (-) which can have any value, the EDR type is as defined in the first column of the table.

EDRs with input field values not matching the ones described in the table are not mapped and stay in NCC generic CDR format.

Table: Identifying EDRs of different types

renewSubscriptionRecord LIFECYCLE_RECORD RenewSubscriptionAction - Subscription
