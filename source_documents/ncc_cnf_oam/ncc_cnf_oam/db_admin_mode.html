db-admin-mode

Description

The db-admin-mode tool can be used to set the Aerospike database cluster in an administratively disabled state (also referred to as admin down state).

All NCC database nodes are organized in two racks, each rack containing half the number of total nodes. The NCC monitoring service monitors the state of all database nodes in a rack and determines the state of a rack as a whole. A rack is considered UP if not more than K nodes are unavailable, where K is configurable. The monitoring service also determines the overall state of the database cluster. If the state of any rack is DOWN, then the overall database state is considered as DOWN.

The db-admin-mode tool can be used to override the default behavior of the NCC database monitoring service. When the database cluster is in a administratively disabled state, the NCC monitoring service does not complain even if one complete rack is unavailable.

A rack-wise database upgrade is an example where the database is moved into admin down state so that a complete rack can be pulled down and upgraded at a time.

Notice:
Only a single copy of each data record may be available to the application in admin down state. The decision to move the database cluster to a admin down state should be taken with utmost caution and only after consultation with Nokia support.

Location

This tool can only be run from the admincli pod and is located in the /opt/tpa/bin/ directory.

Syntax

db-admin-mode -s true
Option Description
none Display the current admin mode (enabled or disabled).
-s {true|false} Set the admin mode.
  • true: Enable admin mode, that is, move the database to administratively down state.
  • false: Disable admin mode, that is, remove the administratively down state of the database.
-h Display the help message.

Examples

Display help information

[tpaadmin@growthme-admincli-67bf879957-ll6q9 tmp]$ /opt/tpa/bin/db-admin-mode -h
usage: db-admin-mode [-h] [-s {true,false}]
 
A tool to set admin state of the database cluster. If admin state is set to
'true', the monitoring application considers the overall DB cluster state to
be UP even if a complete rack of DB nodes is not available.
 
optional arguments:
-h, --help show this help message and exit
-s {true,false} Set the database admin state to either true or false.

Set admin mode to enabled:

[tpaadmin@growthme-admincli-67bf879957-ll6q9 tmp]$ /opt/tpa/bin/db-admin-mode -s true
--------------------------------------
Database admin mode is now set to enabled
--------------------------------------
 
[tpaadmin@growthme-admincli-67bf879957-ll6q9 tmp]$ /opt/tpa/bin/db-admin-mode
--------------------------------------
Current database admin mode is enabled
--------------------------------------

Set admin mode to disabled:

[tpaadmin@growthme-admincli-67bf879957-ll6q9 tmp]$ /opt/tpa/bin/db-admin-mode -s false
--------------------------------------
Database admin mode is now set to disabled
--------------------------------------
 
[tpaadmin@growthme-admincli-67bf879957-ll6q9 tmp]$ /opt/tpa/bin/db-admin-mode
--------------------------------------
Current database admin mode is disabled
--------------------------------------
