db-check-rewind-status

Description

db-check-rewind-status is used to check the status of the database rewind in progress for the specified remote system by checking the number of recoveries_pending. When the number goes down to 0, the rewind is complete and the script ends. The check is done in a loop every 30 seconds, with a default of max 100 times. The script also ends when the max time of retries is exceeded with the last status.

Syntax

/opt/tpa/bin/  db-check-rewind-status-s <remote system> [-r <retries>]
Note:
-s is mandatory
Option Description Option arguments
-h, ?, or no arguments Display the help information.  
-s Identify the target system for the rewind process from the local system as the source, as it appears in the topology data. <remote system> is the name of the target system.
-r Max number of times the check is tried before the script exits. Defaults to 100. <retries>: some positive integer value. Default is 100.
