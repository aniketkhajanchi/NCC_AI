CGF (DR) and SPS-stream backup and restore

The CGF (DR) chart does not need a backup and restore since configuration is provided as values and overrides during installation, which do not change during runtime. Re-installation of the CGF (DR) chart with the same values is sufficient for a restore.

The SPS-stream chart provides backup and restore, and it is enabled by default. Backup and restore may be disabled by overriding these values in the NCC-stream chart.
cbur:
    enabled: true

For both backup and restore operations, set these variables.

CBUR_MASTER_NAMESPACE=<cbur_master_namespace>
NAMESPACE=<namespace>
RELEASENAME=<releasename>
Where:
  • CBUR_MASTER_NAMESPACE is the NCC Backup and Recovery master namespace. For OpenShift deployments, the app namespace NAMESPACE must be the same as CBUR_MASTER_NAMESPACE or backup and restore will fail.
  • NAMESPACE is the app namespace of the application to be backed up or restored.
  • RELEASENAME is the name of the release.

For example:

CBUR_MASTER_NAMESPACE=sps-btel-ig   #For Openshift, the app namspace MUST be the same as cbur-master namespace, or backup and restore will fail
NAMESPACE=sps-btel-ig
RELEASENAME=btel

To backup SPS-stream

SPS-streams use auto-injection of NCC Backup and recovery sidecar, which requires restart of cbur-client pod.

curl -v --header 'Accept: application/json' -X POST http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/backup/$NAMESPACE/$RELEASENAME?helm_version=3 2>&1 | tee backup_sps_streams.log

To restore SPS-stream

Run the following command to restore SPS-stream.

curl -vkL --header 'Content-Type: application/x-www-form-urlencoded' --header 'Accept:application/json' -X POST -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/restore/$NAMESPACE/$RELEASENAME?helm_version=3 2>&1 | tee restore_sps_streams.log

Use the following command to view the status of the restore command.

curl -v --header 'Accept: application/json' -X GET http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/task/a536dfae-ae26-42bb-8c8f-25a860b29ae6

To apply the changes, see Publishing lookup configuration to database.
