Vital vs non-vital services

Services provide a view on how the system is performing for debugging and monitoring purposes. Noting that a service is DOWN can help with debugging. Each service is listed as UP or DOWN. Each service defines how many pods need to be in the READY state for the service to be considered UP. For most pod types except Aerospike, having at least one pod in the ready state keeps the service UP.

A non-vital service just gives information about the status of the service.

A vital service not being UP would mean that key primary use cases like Diameter, 5G transaction processing, Bill Cycle Reset, or Provisioning are not available.

An NCC subsystem with a vital service DOWN causes the subsystem to be out of service. This triggers the geographic subsystem level redundancy.
