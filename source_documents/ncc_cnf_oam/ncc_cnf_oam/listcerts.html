listCerts

Description

This operation provides a way to easily monitor the certificates stored in secrets in different

namespaces, ensuring that they remain up-to-date.

Syntax

adminctl [-h] -o listCerts [-o {table,json,yaml}] [-j JMESEXPRESSION, --jmesExpression JMESEXPRESSION]
Option Description
No arguments Displays list of certificates in table format.
-h or -? Displays usage/help information.
-o <type> Determines the output type:
  • table (default)
  • json
  • yaml
-j JMESEXPRESSION, --jmesExpression JMESEXPRESSION Performs the JMESPath filter on the output data.

Apply a JMESPath filter to the returned value (for example: [?namespace=='ig1']. You can concatenate multiple JMESPath filters like [?namespace == 'ig1'] | [?daysToExpiry.to_number(@) <= \`700\`]).
Note:
When using the ! operator, the error bash: !: event not found may occur. Run the command set +H to avoid this.

Examples

To display certificate list in YAML format:

Command: adminctl -o listCerts -o yaml
                
Output: 
- daysToExpiry: '344'
  expiryDate: '2024-04-25'
  issuer: CN=noop-cert.nokia.com
  namespace: ig1
  secretKey: keycloakcert.pem
  secretName: dr-keycloak-client-secrets
  serialNumber: <serial_number>
- daysToExpiry: '410'
  expiryDate: '2024-06-29'
  issuer: L=currentcluster, CN=Intermediate CA, O=Istio
  namespace: istio-system
  secretKey: ca-cert.pem
  secretName: istio-istio-discovery-cacerts
  serialNumber: <serial_number>

To display data in JSON format and apply filter on secretName:

Command: adminctl -o listCerts -o json -j "[?secretName == 'sps-ldap-credential']"
                
Output:
[
   {
      "namespace": "ig1",
      "secretName": "sps-ldap-credential",
      "secretKey": "cacert",
      "serialNumber": <serial_number>,
      "issuer": "L=Finland, CN=NCC Server Intermediate CA, O=Nokia",
      "expiryDate": "2025-01-30",
      "daysToExpiry": "625"
   }

]

These examples demonstrate how to apply filters using JMESPath to extract specific data based on different criteria:

Filter data by 'secretName' with the value "sps-ldap-credential":
JMESPath: "[?secretName == 'sps-ldap-credential']"

Filter data by 'daysToExpiry' less than or equal to 700 days:
JMESPath: "[?daysToExpiry.to_number(@) <= \`700\`]"

Multiple filters can be used together using pipe ( | ) operator.
JMESPath: "[?secretName == 'sps-ldap-credential'] | [?daysToExpiry.to_number(@) <= \`700\`]"

Filter data by 'secretName' containing substring 'maria':
JMESPath: "[?secretName.contains(@, 'maria')]"

Filter data where 'secretName' does not contain 'maria' or 'ldap':
JMESPath: "[?!(secretName.contains(@, 'maria') || secretName.contains(@, 'ldap'))]"

Filter data by 'secretName' matching 'sps-ldap-credential' or mtls-client-credential:
JMESPath: "[?secretName == 'sps-ldap-credential' || secretName == 'mtls-client-credential']

Sort data by 'secretName':
JMESPath: "sort_by(@, &secretName)"

Reverse sort by 'secretName':
"sort_by(@, &secretName) | reverse(@)"
