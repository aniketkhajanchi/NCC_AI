Executing tests

The overall status of the X-button test package is displayed under the defined test group buttons, indicating:

  • Ready to Run - Any of the buttons can be clicked to trigger a new execution job.
  • Running <test suite> - A test execution job is in progress.

An execution job can be triggered if the status is 'Ready to Run'. The following figure presents an example of it.

Figure: 'Ready to Run' status

Only one execution job is allowed at a time for an X-button test package. When the user clicks any of the blue buttons, the status changes to 'Running <test suite>' and all buttons are greyed out. The following figure presents an example of it with a 'Running Regression' status.

Figure: 'Running' status with the buttons greyed out

When the running job completes, all buttons turn to normal. The user can now check the report and log of the latest run for each button by clicking the associated REPORT button. For more information on checking reports and logs, see Checking reports and logs.

The Execution Results view lists the history of all execution jobs, for example, the overall status, buttons they belong to, and the time when they have been executed. For more information, see Execution job history.
