SM subsystem overview

The Service Manager (SM) subsystem provides services in an NCC network deployment to manage the network of ME subsystems that provide the charging and other call-processing functions. The SM subsystem provides the entry point for external interfaces for GUI access and API gateway access for various sorts of operator IT systems and for mobile applications.

The SM subsystem is built on the same software infrastructure as the ME subsystem. The primary differences are in the business logic in the servicemanager pod on the SM subsystem, compared to the charging pod on the ME subsystem, and any additional services to support each of those two applications. This shared foundation allows the SM subsystem to benefit from the same cloud-native architecture, and provides horizontal scalability and high performance in the as in the ME subsystem.

All provisioning goes through the SM subsystem, whether through its GUI or its APIs. The SM subsystem manages and maintains a mapping of which subscribers/devices are homed on which ME subsystems in the network as primary and secondary. The actual subscriber/device records are stored only on the ME subsystems. For the service (configuration) data, a primary copy is maintained on the SM subsystem and distributed to the ME subsystems.

The SM GUI can be accessed using Google Chrome or Mozilla Firefox browsers. For compatibility information, see the Release Notes.
