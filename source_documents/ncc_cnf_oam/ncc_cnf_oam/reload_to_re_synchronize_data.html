Reload to resynchronize data

If required, perform a reload to resynchronize data on the recovering site. This is required to provision leftover data on the ME, which includes some local data on the ME that resides in the dsclocal namespace of the ME database. Also, there may be some data that is updated/added/deleted after a scheduled backup of the ME, and that data will not be recovered by restore and XDR synchronization from the mate site. Instead, that data will be recovered on the ME by reloading the ME from the healthy SM.

Perform these steps to reload and resynchronize data on the recovering site.
  1. In the NCC GUI, choose Global Configurations, then Managed Elements.
  2. Hover the cursor over the recovering ME in the Managed Element List and click the Reload ME icon. A dialog box with the following message should appear: Select the status to reload the ME.
  3. In the aforementioned dialog box, select the All radio button. This action forces updates from the SM to an ME for the dsclocal namespace and corrects any failed transactions
Note:
In case of failure and partial status, retry the above three points.
