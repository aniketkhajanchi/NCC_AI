GUI elements

Log in and log out of the User and Role Management GUI

To log into the User and Role Management GUI as an administrator, open the web browser and enter https://$SM_FQDN/auth in the address bar. Then, enter the username and password to access the User and Role Management server GUI.

Note:
The username and password credentials are stored in system parameters, so they need to be in sync if SM login is to interact correctly with the User and Role Management server. This command can be used to view and change the credentials used by SM to communicate with the User and Role Management server:
tpaadmin@naperil95vzwcchf-y-nk-ig-000-ig-admincli-5dbd9884bd-crqn9 /]$ ./opt/tpa/sbin/TPA_systemProp | grep CKEY
77 CKEY Admin Password: Admin@123
78 CKEY Admin
Note:
Generally, a user who is not an administrator only needs to log in as usual and User and Role Management redirects the user as required.

Launch pad

Once a user logs into the User and Role Management server GUI, the Launch Pad is displayed that provides various options. The desired category under which all the entities belonging to that particular category are available may be selected.
Table: Categories available on the User and Role Management server GUI
Category Description
Administration Console Allows the admin to centrally manage all aspects of the User and Role Management server by redirecting to the Admin Console of that server.
Documentation Allows the admin to access the documentation related to User and Role Management server information, such as server installation and configuration guide, server administration guide, and Javadocs amongst many others.
User and Role Management Project Allows the admin to learn User and Role Management server basics, such as adding authentication to User and Role Management server applications and securing services.
Mailing List Allows the admin to manage or subscribe to the User and Role Management user mailing list.
Report an issue Allows the admin to resolve any known User and Role Management server issues by redirecting to the known issues list and their related workarounds.
