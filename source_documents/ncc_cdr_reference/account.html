EDR tags

Tag Description Example
accountID This attribute indicates the account ID associated with the entity. If the application preference AutoGen SPR Ids is set to true then this tag contains the value of accountName. AccountTest1
accountDbID The account identifier stored in database. This tag appears only if the application preference AutoGen SPR Ids is set to true. -
accountName The account name stored in the database. AccountTest1
hourOfDay This parameter indicates the hour of day to which the entity is applicable. A value of 24 indicates EXACT. This indicates that the time in end date and time is same as subscription start time. A value of 25 indicates start of new day. The end date and time are aligned to 00:00:00 of the next day, in accordance with the lifecycle periodicity. 1
dayOfWeek This parameter indicates the day of week to which the entity is applicable. A value of 7 indicates EXACT, that is, the day of week is same as the day of week of the subscription purchase date. SUNDAY
dayOfMonth This parameter indicates the day of month to which the entity is applicable. A value of 32 indicates EXACT, that is, the day of month is same as the subscription purchase date. 2
timeZoneId This parameter specifies the account time zone. Asia/Kolkata
accountcreationTime This parameter specifies the time at which the Account was created. 03/12/2022 12:40:02
accountlastUpdateTime This parameter specifies the time at which Account was last modified. 03/12/2022 12:40:15 a
accountOwner The owner of the account. Device121189
accountOveragelimit This parameter specifies the overage limit as defined in the Account entity. -9.999999999E9
accountType

This parameter specifies the type of Account, which is the type configured by the operator in the Account entity.

Possible values: PRE_PAID, POST_PAID

 POST_PAID
thresholdProfileGroupIds This parameter specifies the threshold profile groups attached to the account. [TPG175931]
renewStatus This parameter indicates if account is renewed via lifecycle action. false
billCyclecalendar The parameter indicates the bill cycle calendar applied for the reset action. BCC1
taxationID This parameter specifies the identifier for the taxation profile. This tag appears only when taxation profile is configured in tarifff or main balance. tax1
customDataAdded This parameter indicates the new custom data attributes added in Create/Update Account request. [Custom_Account1]
customDataDeleted This parameter indicates the custom data attributes removed in Update/Delete Account request. [Custom_Account1]
CustomDataName_ExpiryTime Indicates the expiry time of the custom data in the format dd/mm/yyyy hh:mm:ss. 12/09/2026 02:54:39
customDataExpiryTimeUpdated The list of custom data names for which the update operation has been executed for expiry time. [CD2_Account, CD1_Account]
old_CustomDataName_ExpiryTime

Indicates the old expiry time of the custom data when expiry time is updated.

 12/09/2026 02:54:39
userId This parameter contains the userId and if an unknown user is renamed, then it contains the updated name. usr1
balance This parameter indicates the monetary balance of the account, which supports a precision of five decimal digits. If more than five decimal digits are present, then half round-up is applied at 5th decimal places. It can be a positive or negative number and the default value is 0. 10
availableBalance This parameter indicates the monetary balance of the account with reserved values, which supports a precision of five decimal digits. If more than five decimal digits are present, then half round-up is applied at the 5th decimal places. It can be a positive or negative number and the default value is 0. 10
homeSite This parameter indicates the ID of the home site on which the entity is homed. 6
partition This parameter indicates the partition in which the entity is created. 6
localSite This parameter indicates the ID of the local site from which the entity is created. 6
globalEntity Indicates whether the account is global or local.
  • true: global
  • false: local
 true
accountBalanceBefore Indicates the account balance before the operation. 50.75
loanStateBefore Specifies the loan state before the operation. OPT_IN
totalLoanRemainingDebtBefore Specifies the remaining debt of the loan before the operation. 50.50
result This parameter indicates the result, that is, SUCCESS or FAILURE. SUCCESS
generationTimestamp This parameter indicates the timestamp at which the EDR was generated. 03/12/2022 12:40:02
RecordTimestamp This parameter specifies the date and time of the CDR/EDR generation. This value is taken from the CDR module timestamp. By default, it is presented in standard DD/MM/YYYY HH:MM:SS format. 03/12/2022 07:10:02
loanInfo Record property tag for loan information. RecordProperty
↘loanId The identifier of the loan. ETS_1267
↘loanAmountBefore Specifies the loan amount remaining before the operation. 50.00
origOverageLimit The attribute carries the value of the original value of overage limit in case of temporary overage limit updates.​ It also indicates that update on overage limit is temporary, or not, depending on whether it has a value. -150.0
taxIncluded This parameter indicates whether applied tax is included to the fee/cost or not.

If true, tax is included to the cost/fee.

If false, tax is applied over and above cost/fee.

Default Value: false
