Requirements

Cluster

NCS must be installed with Cinder, GlusterFS/rook or localStorage enabled on the Worker nodes, given CMDB and CCTF both require Persistent Volumes (PV).

For NCS installation instructions, see NCS documentation.

For example, for vLab (OpenStack), you can choose OneKey, CLI - OpenStack or the Web Portal - OpenStack Guide. OneKey is the easiest one.

  • Software recommendation: NCS version 20 or later
  • Resource requirement: Minimum 2 VMs:
    • 1 Deployment node
    • 1 Control/Worker/Edge combined node
Table: Resource requirements
Node role CPUs Memory (GB)
Deployer 2 4
Control 4 8
Worker 2 4
Edge 2 4
Storage 2 4
Note:

If you have a Control/Worker/Edge combined VM, the requirement for the combined VM should be 8 cores and 16G RAM.

If NCS 20 is used, the Worker node should have at least 4 cores and 8G RAM.

Ingress controller

To access the CCTF UI, an ingress controller needs to be installed. The selection of ingress controller depends on the cluster administrator.

It is recommended to use citm-ingress as the ingress controller. For more information, refer to CITM guide.

User provided Docker image

User provided Docker images will be started as a container to run test cases.

The default user in user provided Docker images can be root or non-root (UID:10000, GID:10000). CCTF will not change the default user defined in user provided docker images.

Note:
If a non-root user is used, create a user with UID:10000 and GID:10000 and ensure that all test scripts can be executed via this non-root user.
