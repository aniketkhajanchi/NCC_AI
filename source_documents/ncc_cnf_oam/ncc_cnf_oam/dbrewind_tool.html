db-rewind

Description

The db-rewind tool enables forwarder and starts the XDR rewind, using the local system as the source. Before running this tool, make sure the tool db-prepare-rewind has already run for the destination system. When run, this tool fetches the primary partition namespace of the local system if the namespace is not provided in the command. The tool only proceeds if the remote system exists in the topology file /opt/tpa/conf/topology/SpsTopologyConfiguration.yaml, and it must also share a common namespace.

This tool may be used in all NCC systems.

Syntax

/opt/tpa/bin/db-rewind -s <remote system> [-n <namespace>] [-t <seconds>]
Option Description Option arguments
No arguments Displays usage or help information.  
-h or -? Displays usage or help information.  
-n <namespace> Namespace to be rewound. Values can be:
  • dsc
  • dscglobal
  • spspd<N>, where <N> is a number
When input for -n is not provided, the primary partitioned namespace spspd<N> will be fetched on the local system by default.
-s <remote system> Identify the target system for the rewind process from the local system as the source, as it appears in the topology data. System ID or name
-v Display the script version  
-t <seconds> Number of seconds in the past from where to start the rewind. Default value is set to rewind site to earliest possible value.
