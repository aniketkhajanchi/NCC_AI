Kafka redelivery stream

The Kafka redelivery stream handles failed distributed records by the Kafka output stream, after being written by Log distributor stream to an SFTP location configured by the SFTPRemoteDirectory node parameter. It is important that the node parameter SFTPRemoteDirectory is set to the same value in both the CGF (DR) SFTP Kafka Failure Collector Node and CG Log Distributor Node.

As part of this stream, a Kafka health check process was introduced to observe the Kafka brokers status before trying to redeliver messages to Kafka.

Kafka redelivery stream contains these nodes:
  • CGF (DR) SFTP Kafka Failure Collector Node: collects failed files from a remote server by using SFTP and writes them securely into the database.
  • CG Kafka Distributor Node: distributes the encoded records from the database to Apache Kafka topics.
