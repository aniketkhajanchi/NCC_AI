4G diameter connectivity rules

The Diameter/HTTP Rule Engine application plugin is supported to enable the handling of generic message manipulation scenarios. It implements an event-driven, forward-chaining rule evaluation algorithm to facilitate the routing functions of NCC, which are relaying, modifying, and Diameter message answering.

After the Diameter Rules Engine plugin has been sequenced, the application plugin is evaluated and invoked by the system to process Diameter messages as shown in the following figure.

Rule profile (Diameter rule system version)

A rule system version captures a series of triggers and associated rule groups. The SM supports a graphical drag-and-drop interface with an embedded text editor for creating rules.

To configure 4G Diameter Connectivity Rules, navigate to Connectivity → ☰ → MESSAGE MANIPULATION and select 4G DIAMETER tab. Click + and configure a 4G diameter connectivity rule as described in the following table:
Table: 4G diameter connectivity rule parameters
Parameter Description
Name Enter a name for the 4G diameter connectivity rule.
Description Specify a description for the connectivity rule.
Click APPLY, and configure the rules and variables.

Rule profile parameters

In the Rules and Variables window, enter information as shown in the following table.

Parameter

Description
Name

Enter a name for the Rule System Version.

Maximum characters 50 only.
Description Specify a description of the profile.
Select Trigger Select a trigger to define one or more associations between triggers and rule groups.
Rule Group Select a rule group or add a rule group to define one or more associations between triggers and rule groups. The selection opens an Rule Configuration panel that allows you to provide a name for the rule group, state if the rule group is a top level rule group, and define the rules in the rule group.
