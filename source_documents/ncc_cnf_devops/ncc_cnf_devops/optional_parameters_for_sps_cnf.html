Common Airflow variables

In addition to the variables mandatory for viewing the default Airflow test suite DAGs in Airflow UI (see Enabling DAG execution), there are also common optional variables that can be configured for different NCC setups depending on the user's requirements.

Table: Optional variables for NCC configuration
Variable Description
local-ip

IP of the Maestro setup. If not passed, Maestro tries to determine the IP. If there are multiple network interfaces available on Maestro (for example, a dual stack IPv4/IPv6 installation or two IPv4 networks), it is recommended to configure this variable with the IP that is expected to communicate with NCC.
email-from-address User can receive a test execution summary by configuring e-mail variables. This variable defines the address, from which the user receives e-mails.
email-relay-server

User can receive a test execution summary by configuring e-mail variables. This variable defines the SMTP server, to which e-mails are sent. For Nokia internal purposes, set this parameter to mailrelay.int.nokia.com.
email-subject-prefix User can receive a test execution summary by configuring e-mail variables. This variable defines a prefix to be used before an e-mail subject.
email-to-address User can receive a test execution summary by configuring e-mail variables. This variable defines the address, to which the user is expecting e-mails. For multiple e-mail addresses, provide the e-mail addresses separated by comma.
max-reports-count-per-suite Airflow has an option to view old reports. This variable defines the number of old reports (per test suite) to be kept in the system. The default value is 60.
