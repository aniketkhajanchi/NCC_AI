Sequencing of CDRs per Record Type

It is possible to allow CDRs to be ordered sequentially in output files based on a unique number within

records coming out of the collector. This number is mapper to localRecordsequencenumber field which keeps on incrementing per record type. All records within the same file have this incremented field. Every time a new file is created, it is continue from the previous counter. This counter is added to ASN1 chargingRecord and genericRecord CDRs. Each type start counting based on the LocalRecordSequenceNumberInitial parameter till LocalRecordSequenceNumberLimit parameter. In the case of ISU, LRSN start from the initial value instead of continuing from the previous number.

Configuring LRSN node parameters in CDR streams

For aggregation to take place, set the following node parameter values:

  • for CG Mapper Node (Business logic stream)
    • LocalRecordSequenceNumberOrderEnabled: yes
  • for CG File Collector Node (File output stream)
    • CustomLRSNClass: com.nokia.mc.node.cg.file.collector.NCCCustomLRSNImpl (hidden)
