Set recovering system to in service

Set the recovering system to in service by running these commands.

kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/configure-manual-oos -e ME

# Verify MOOS status should change to "FALSE"
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/configure-manual-oos -s ME

# Output:
# COMPONENT | MOOS
# ME: FALSE

# Verify all pod should be up and running
sleep 20
kubectl get pods -n <namespace> | egrep 'charging|central|provi|notification|iohd'
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
