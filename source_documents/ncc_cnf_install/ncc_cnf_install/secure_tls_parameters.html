Secure mTLS parameters

The default option for the TLS mode is SIMPLE. Edit/add these values in the Input Generator spreadsheet to enable mTLS mode.
  • Set the Aerospike Intersite TLS Mode and Intersite TLS Mode values to MUTUAL in the SPS section of the Common worksheet.
  • Enter the pre-provisioned server secret names in the HTTP Secret Name value field for each namespace worksheet in the Istio section. The default value is sps-credential.
  • Enter your specific mTLS FQDN Client Secret Name value in the Shared section of each namespace worksheet. The mTLS OAM FQDN Client Secret Name and mTLS Provisioning FQDN Client Secret Name are present in the Shared section of each namespace worksheet. These are to be used when you need to set different certificates for OAM and Provisioning traffic. These are optional fields. By default, the mTLS FQDN Client Secret Name value is used, if the parameters are not set. The format must be in this order: secret-name, private-key, public-cert, and CA-cert.

Third-party egress communication

You must create a service entry and destination rule for external communication with a third party. You must provide the FQDN along with the protocol, port, tls-mode, secret-name, private-key, public-cert, and CA certification of the external system in the Input Generator spreadsheet under the Service Entries section for each namespace. The Ports value must be in this format: protocol/sourcePort>upgradePort/ipAddress#tlsMode(secretName:privateKey:publicCert:caCert).
