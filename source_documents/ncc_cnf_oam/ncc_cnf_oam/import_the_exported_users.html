Import the exported users

After the migration to SSO/LDAP is completed, these steps can be performed to import the users that you exported prior to the migration.
  1. Login to the User and Role Management GUI.
  2. Select the sps-auth realm from the drop-down list in the left navigation.
  3. Click the Realm settings option under the Configure section.
  4. Click the Action drop-down button on top right corner and select Partial import.
  5. Click the Browse button to find the exported JSON file.
  6. Select the sps-auth-users-0.json file that was exported for the Partial Import.
    1. Select Users in the Choose the resources you want to import section.
    2. Choose the appropriate option. The possible values are: Skip, Overwrite, and Fail import.
  7. Click Import.
  8. Repeat the same partial import for the btel-auth realm with the btel-auth-users-0.json file that was exported.
  9. You must create the grafanaeditor and a grafanaadmin roles for the Metrics Viewer in the btel-auth realm, if they do not exist.
    1. Click on Realm Roles in the left navigation.
    2. Click on the Create Role button.
    3. Enter grafanaeditor in the Role Name field.
    4. Click on Save.
    5. Repeat the same process to create the grafanaadmin role.
  10. You must remove any existing Metrics Viewer group memberships (GrafanaAdmin and GrafanaEditor) from the imported btel-auth realm local Metrics Viewer users and assign the newly created Metrics Viewer roles to them. This step only applies to imported users that are members of GrafanaAdmin and/or GrafanaEditor.
    1. Click on Groups in the left navigation.
    2. Select the GrafanaEditor group.
    3. Click the three-dot menu at the end of the selected row and click the Delete button.
    4. Click the Delete button in the confirmation pop-up window.
    5. Repeat the same process for the GrafanaAdmin group.
    6. Click on Users in the left navigation.
    7. Click on the user that you want to edit.
    8. Click on the Groups tab.
    9. Select the GrafanaEditor group.
    10. Click the Leave button and click the Leave button in the confirmation pop-up window.
    11. Select the GrafanaAdmin group.
    12. Click the Leave button and click the Leave button in the confirmation pop-up window.
    13. Click on the Role Mappings tab.
    14. Click on the Assign Role button.
    15. Select the grafanaeditor role and the grafanaadmin role.
    16. Click on the Assign button.
    17. Repeat this process for any other local Metrics Viewer users.
  11. For LDAP: Select Users from the left navigation of the realm, then List All Users and verify that the list is only those users that you want to remain local. LDAP-resident users appear after the first time they authenticate.
