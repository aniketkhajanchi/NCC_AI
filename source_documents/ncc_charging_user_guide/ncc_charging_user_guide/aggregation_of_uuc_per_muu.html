Aggregation of UUC per MUU

In 5G/N40 calls, under MUU array there can be multiple MUUs. Also, there can be multiple UUC (array) in RG. If all the UUCs of one MUU are processed sequentially, it may cause higher latency for update and terminate messages.

The application preference Aggregation of UUC per MUU provides the flexibility to club the UCC into one, so that it is processed as one UUC, and as a result, latency decreases. To enable it, set its value to ENABLE_AGGREGATION.

Note:
  • This feature is not applicable for OFFLINE charging.
  • This feature is not applicable if the application preference MAX Number of UUCs per request = 1.
  • Aggregation is not applicable for APR and duplicate requests.
  • Since only one UUC is picked and the quota of other UUCs is aggregated into it, other UUC parameter logic is ignored, and rating logic is called only for the aggregated UUC.
  • This feature has no impact on CDRs.
  • If this parameter is enabled, then clubbing of values is done only for the following units:
    • CcTotalOctets
    • CcTotalInputOctets
    • CcTotalOutputOctets
    • CcTime
    • CcServiceSpecificUnits
