Prepare the NCC operator custom resource

Configure the NCC operator with details on how you want the canary deployment to operate.

Set these variables:

export DOCKER_REPO=sps-docker-candidates.repo.cci.nokia.net/
export CURRENT_ABSOLUTE_DIRECTORY=$(realpath .)

Run this command to upgrade the spreadsheet to match the new release:

#copy your existing Inputs.xlsx spreadsheet into the current directory.  Fix the filename in the command.
docker run -u="${UID}:$(id -g)" -v ${CURRENT_ABSOLUTE_DIRECTORY}:/work --rm ${DOCKER_REPO}chf/oneclick:${TARGET_CHF_SOLUTION_VERSION} oneclick -i InputsXXX.xlsx

Run this command to create the custom resource:

mkdir output
docker run -u="${UID}" -v ${CURRENT_ABSOLUTE_DIRECTORY}:/work --rm ${DOCKER_REPO}chf/chf-operator:${TARGET_CHF_SOLUTION_VERSION} python3 /python/customResourceFromYaml.py --inputYamlFile /work/input-generator/inputs.yaml --overridesDirectory /work/input-generator/overrides --elementsToInstall ig1 --mode canary --outputDirectory /work/output

Run this command to deploy the custom resource in your cluster:

kubectl -n ncc-operators create -f output/chargingfunction-ig1.yaml
