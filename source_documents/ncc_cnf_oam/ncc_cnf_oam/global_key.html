Updating the Pod Restart Counter value

There is a global key/value pair in the NCC Input Generator spreadsheet in the Pod Restart section for each namespace.

The default value is restart0. When NCC is first installed on the network, all of the pods for that system have the custom annotation value set to restart0.

Open the deployment spreadsheet and update the default value of the pod Restart Counter from restart0 to restart1, or any value which is different/unique from the previous value for each NCC system where a pod restart is required.

Generate the override files by using the Input Generator. The Input Generator prepares the values for all Helm charts across all systems in your network deployment. For more information about the Input Generator, see the Installation and Upgrade Guide.

Run the helm upgrade command with the newly generated override files. Since there is a new value for the pod Restart Counter custom annotation, Helm will interpret this as an update and will start performing a rolling update for all the pods.

For example, if the current value of Restart Counter is restart1, and you need to restart all the pods in that system, then change the value to restart2. The next time you restart all of the pods, then change from restart2 to restart3.

Note:
For this feature to restart all the pods, it is important that the pod Restart Counter custom annotation value is always unique/different from the previous value used.
