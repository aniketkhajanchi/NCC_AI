CDRs for Gy session

CDR per request and per MSCC

For Gy sessions, NCC provides an option to either generate CDRs per request or per Multiple Services Credit Control (MSCC).

  • CDR per request: A single CDR is generated for each CCR request. Even if there are multiple MSCCs in a single CCR, a single CDR is generated. Information of each MSCC is presented as shown in Example CDR structure per request vs. per MSCC. In this example, there are three MSCCs in a single CCR.
  • CDR per MSCC: A CDR is generated for each MSCC in the request. In the example shown in Example CDR structure per request vs. per MSCC, there is a single CCR with three MSCCs. As a result, there are three corresponding CDRs, one for each MSCC.

The application preference CDR Generated per MSCC is used to enable or disable CDR generation per MSCC. Refer to the Global Configuration Guide for details.
