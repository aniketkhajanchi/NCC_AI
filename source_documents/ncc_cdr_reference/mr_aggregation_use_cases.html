MR aggregation use cases

  • Complete MR session which starts from recordSequenceNumber = 0 to recordSequenceNumber = numberOfRecipients – 1. The tag multipleRecipientAddress contains all the values of recepientAddress from input requests.
  • Single failed MR request identified by resultCode = 4012 and the existence of multipleRecipientAddress tag is populated by CGF (DR) as it is.
  • Incomplete session generates consolidated and partial outputs after passing MRAggregationTimeout. A warning is raised on the GUI showing the incomplete session ID with the missing request (start, update, or last).
    Note:
    • Partial outputs is populated directly once a request is processed by CGF (DR).
    • Consolidated output is populated after receiving all requests with no gap in regard to recordSequenceNumber.
    • Missing request leads to a delay in partial and consolidated outputs generation.
    • Partial and consolidated outputs are populated into the same directory for SMS/MMS requests.
    • recordSequenceNumber, sessionSequenceNumber, and recipientAddress are removed from the consolidated output.
