Bundle

EDR tags

EDR Tag Description Example
bundleName Specifies the Bundle name. Bun1
bundleVersion This attribute indicates the bundle version, if versioning is enabled. 1
bundleVerTime Specifies start time of version. 23/03/2021 14:37:10
bundleVerState Specifies the state of entity, that is, ACTIVE or TEST_ACTIVE. ACTIVE
bundleFee Fee deducted when bundle is purchased. 10
chargeMissedRecurrentPeriods Specifies if missed periods are charged during Subscription Renewal on Account Recharge.

Possible values: True, False

Default value: False

 False
customData Specifies custom data names for the bundle. [RZ, HZ]
customDataAdded Specifies the newly added custom data names for the bundle when the bundle is updated. [RZ]
customDataDeleted Specifies the removed custom data names for the bundle when the bundle is updated. [CD_Zone1, CD_Zone2]
maxRenewals

It signifies the maximum number of times the subscription can be renewed. If a value is not specified, then it implies that unlimited renewals can take place.

 5
maxBundleInstances Indicates maximum instances of bundle that can be subscribed per Device/Group. 1
chargingServiceName Specifies the name of Charging Service associated with the bundle. CS1
usageControlService Does not belong to charging (deprecated). -
bundleCreationTime Specifies the time at which the entity was created. 23/03/2021 14:37:10
bundleLastUpdateTime Specifies the time at which entity was last modified. 23/03/2021 14:37:10
disableTTC

This signifies if TTC calculation should occur. If this flag is set to true, then TTC calculation is not done and VT is aligned to the expiration date and time of the subscription.

Possible values: true, false

 true
rateAllowanceOffer It indicates that this bundle will pick rate from other bundles based on priority.

Possible values: true, false

 false
ucServiceName The name of the UC service. UC1234
hourOfDay

This attribute indicates the hour of day.

A value of 24 indicates EXACT. This indicates that the time in end date and time is same as subscription start time.

A value of 25 indicates start of new day. The end date and time are aligned to 00:00:00 of next day, as per the lifecycle periodicity.

 1
dayOfWeek

This attribute indicates the day of week.

A value of 7 indicates EXACT. This indicates the day of week is same as the day of week of the subscription purchase date.

 SUNDAY
dayOfMonth

This attribute indicates the day of month.

A value of 32 indicates EXACT. This indicates the day of month is same as the subscription purchase date.

 12
taxationID Specifies the identifier for the taxation profile. tax1
alignBundleRef Base bundle ID with which end time is aligned.  
renewalPriority This tag specifies the priority used at the time of subscription renewal through account BCR. 0
processingType This tag specifies whether the bundle is auto-manageable.
ProcessingType has the following values:
  • AUTO_MANAGE: It indicates that the bundle is auto-manageable.
  • NONE: It indicates that the bundle is not auto-manageable
 AUTO_MANAGE
campaignStartDate This parameter indicates a start date, after which the bundle is available for use. 23/03/2023 14:37:10
campaignEndDate This parameter indicates an end date, after which the bundle is not available for use. 23/03/2023 14:37:10
subscriptionEndDate This parameter indicates a subscription end date, after which a bundle cannot be subscribed by any subscriber. 23/03/2023 14:37:10
renewalEndDate This parameter indicates a renewal end date, after which a subscription corresponding to a bundle cannot be renewed. 23/03/2023
taxIncluded This parameter indicates whether applied tax is included to the fee/cost or not.

If true, tax is included to the cost/fee.

If false, tax is applied over and above cost/fee.

Default Value: false

 true
chargeFeeDiscountType The available discount activation fee types are available for selection.
  • None
  • Percentage : Indicates that the charge fee discount value is in percentage.
  • Absolute : Indicates that the charge fee discount value is in Big decimal format.
  • Override : Enables the operator to configure a discounted fee which overrides the charge fee for bundles attached to the discount.
 PERCENTAGE
chargeFeeDiscount It is value of bundle fee discount. 50
