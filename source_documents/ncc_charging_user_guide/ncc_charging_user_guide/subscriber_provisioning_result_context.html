SUBSCRIBER_PROVISIONING result context

Table: SUBSCRIBER_PROVISIONING result context
Attribute name Trigger Description
Update-Validity SUBSCRIBER_POST_PROVISIONING

SUBSCRIBER_POST_TRANSFER

This action is used to configure the number of days that the account or device or account validity time should be extended.

The number of days is added to the current device or account validity.

The number of days configured should be more than 0.
SetValidity SUBSCRIBER_POST_PROVISIONING

SUBSCRIBER_POST_TRANSFER

This action is used to configure the number of days that should be added to the current validity of the account or device.

The number of days configured should be more than 0.

If the updated validity is less or equal to the current device/account validity, then the current validity of device/account is not updated.
