Common steps for SM/IG 3 and all related systems in the network deployment

  1. In the Inputs.xlsx file, set these properties:
    1. Deployment Type = migrate to sso in the Common tab Global section. This step disables backup cronjobs for MariaDB.
    2. Enable SSO = true in the Common tab Global section. This step helps generate appropriate Helm overrides with SSO related properties and sections for some charts.
    3. MariaDB Replication Enabled = false in SM/IG 3 tab CKEY section. This step disables MariaDB replication between Site SM/IG 3 and rest of the SM/IG systems. At this time, no SM/IG systems are replicating from each other.
  2. Regenerate Helm overrides for chf-resource chart, CKEY, Servicemanager, Metrics Viewer, User and Role Management realm upload jobs, and CGF (DR) chart.
