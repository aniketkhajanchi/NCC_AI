NCC Telemetry backup and restore

These instructions assume that the optional NCC Telemetry package is installed with NCC Backup and Recovery enabled. For more information, see the Deploying Backup and Recovery Helm chart section of the Installation and Upgrade Guide.
Note:
NCC Backup and Recovery master rest API calls must be called from the admincli container.
Note:
Kubectl and Helm commands must be executed from a control node. Kubectl commands may not be executed from the admincli container.
Note:
When an NCC Telemetry instance is shared, the NCC Backup and Recovery master rest API may be called from the NCC Telemetry pod which provides curl support.
This table shows the components that can be backed up and restored.
Helm Chart Helm Release Name Type of Data Backed up
cpro btel-{SITE_TYPE}-cpro Prometheus metric data
cpro-gen3gppxml cpro-gen3gppxml gen3gppxml data that is waiting to be retrieved by ftp
cmdb btel-{SITE_TYPE}-cmdb Metrics Viewer configuration data and cnot alarms

For both backup and restore operations, set these variables.

CBUR_MASTER_NAMESPACE=<cbur_master_namespace>
NAMESPACE=<namespace>
RELEASENAME=<releasename>
Where:
  • CBUR_MASTER_NAMESPACE is the NCC Backup and Recovery master namespace. For OpenShift deployments, the app namespace NAMESPACE must be the same as the CBUR_MASTER_NAMESPACE or the backup and restore will fail.
  • NAMESPACE is the app namespace of the application to be backed up or restored.
  • RELEASENAME is the name of the release. The release name can be obtained by running a helm3 list command on the control node.

Manual procedure to backup and restore Prometheus data

Perform these steps to backup the data.
  1. Set the required variables. For example: 
    CBUR_MASTER_NAMESPACE=sps-btel-ig
NAMESPACE=sps-btel-ig
RELEASENAME=btel-{SITE_TYPE}-cpro
  2. Backup the data by running this command. 
    curl -v --header 'Accept: application/json' -X POST http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/backup/$NAMESPACE/$RELEASENAME?helm_version=3

    By default, the Backup and Recovery primary is installed with Celery/Redis to support asynchronous functions. The Rest API calls return a taskId which can be used to query the status of the request with another Rest API call:

    curl -v --header 'Accept: application/json' -X GET http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/task/7b59914d-f521-41552b81b-765a45f85a42
Perform this step to restore the latest data for Prometheus.
  1. Restore the data from the SPS admincli container/common btel pod by running this command. 
    curl -vkL  --header 'Accept: application/json' -X POST --header 'Content-Type: application/x-www-form-urlencoded' -d 'backup_id=&volume=&object_type=&object_name=' http://cbur-master-cbur.$CBUR_MASTER_NAMESPACE.svc.cluster.local:80/v2/helm/release/restore/$NAMESPACE/$RELEASENAME?helm_version=3
