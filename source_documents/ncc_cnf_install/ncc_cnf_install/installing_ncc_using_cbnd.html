Installing Helm charts with NCOM

This procedure provides instructions on how to install the Helm charts by using the Nokia Cloud Operations Manager (NCOM).

Note:
Installation, upgrade, terminate, and delete operations can be completed via the NCOM GUI or the deployNS.sh tool. For more information about the deployNS.sh tool, see Using the deployNS.sh tool.

  1. Follow this procedure to generate a network service (NS) package: Generating Network Service Package in input generator.
  2. Prepare NCS using the procedure Preparing NCS.
  3. Upload the NS package of the appropriate system type to NCOM, for each system.

    For information on how to upload a package, see section Managing Packages/Creating a new package in the <NCOM Version x> Operating Guide.

  4. Create and deploy a network service (NS) in NCOM, for all of the systems.
    1. On your local machine where you executed the input generator, collect all of the values.yaml files from the directory input-generator/output/<site type>/charts.
    2. Generate a chartsOverrideInputs.json file by using the tool yaml-to-Json.py and then import the generated file for creation and deployment of NS. For more information on the tool and file, refer to Tool location.
      Note:
      Instead of providing multiple values for charts, after the first deployment of NS, you can download the inputs file and save it for later use. The inputs file can be downloaded by going to the Inputs tab of the Network Service tab (installed Network Service) and then click the Download button.

    For information on how to create and deploy an NS, see section Managing Packages/Creating and deploying an NS in the <NCOM Version x> Operating Guide.

    Note:
    The NS deployment might have issues. Sometimes a chart might be successful but show as unsuccessful because of some issues such as timeouts.
  5. After the NS has been deployed, if needed, you can monitor the NS operations.

    To see the status, you can click the job ID during the deployment of an NS. For information on how to monitor NS operations, see section Monitoring Jobs/ Viewing jobs/<Job id> in <NCOM Version x> Operating Guide.

    Expected outcome

    The NS deploys successfully or fails.

  6. If the deployment of the NS fails or the installation of some Helm charts fails, follow this procedure: Repackaging an NS package.

    This step allows you to remove the charts that caused the failure and redeploy the NS with the successful charts, do some testing, and then deploy the other charts together with the NS.
