Upgrading Istio

Perform this procedure to upgrade the Nokia-supplied Istio version.
Note:
Contact Nokia support prior to performing this procedure to verify the supported Istio versions and steps for your deployment.

  1. Upgrade the NCC charts to the latest/supported target release keeping the API Compatibility version the same as the currently installed Istio version.
    The Helm upgrade can be done by using the new Helm inputs (for example, run the install-charts.sh script) or any Helm orchestrator tool. If any issues occur during this step, perform a rollback.
    Note:
    An NCC rollback cannot be performed after you trigger the ISU database commit. You must thoroughly test the upgrade with your requirements before you perform the database commit.
  2. Update the Istio version to the supported target version and generate the output folder by using the updated spreadsheet which is generated by the Input Generator tool.
  3. Run a Helm upgrade on Istio by using the new Helm inputs (for example, run the install-charts.sh script) or any Helm orchestrator tool.
  4. Upgrade the NCC charts with the new Istio API Compatibility version with the restart value increased for all charts.
    After the Istio upgrade is successful and the Istio pods are UP and running, the application pods must be restarted.
  5. Restart the application pods once they are running normally.
    Application pods with Istio sidecar installed must be restarted to upgrade the sidecar.

    1. Navigate to the Pod Restart section of each namespace worksheet in your Inputs.xlsx file.
    2. Edit the Restart Counter value in each worksheet and increase the value by one digit.
      For example: if the current value is restart0, change the value to restart1.
    3. Update the API Compatibility of each system to the supported target version.
    4. Perform a backup of the state files.
    5. Regenerate the Helm inputs.
    6. Upgrade each Helm chart with the new inputs.
