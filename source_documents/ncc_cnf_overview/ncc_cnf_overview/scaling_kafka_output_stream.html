Output stream scaling

CG Kafka Collector Node and ASN.1 Encoder Node can be scaled by increasing the number of node instances, but it must be done with caution to ensure that the other nodes in the stream can cope with the load. Otherwise, CG Kafka Collector Node stops collecting from time to time to let the other nodes finish processing data already collected. Every time the collection stops, a warning is raised. The number of instances cannot be increased for CG Kafka Distributor Node, but the node can be scaled by adjusting the concurrency used when sending records, defined with node parameter Concurrency.

There can be several Kafka output streams working in parallel. Different CG Kafka Collector Node instances do not interfere with each other when reading the same database.
