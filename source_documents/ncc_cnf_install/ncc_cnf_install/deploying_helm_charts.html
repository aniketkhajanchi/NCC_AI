Installing Helm charts with a custom script

This procedure provides instructions on how to deploy the Helm charts for a system of any type by using a custom script generated by the input generator. This script installs only the Helm charts for that particular system, and matches the Helm charts to the system's generated Helm input file.

If the parameter Enabled in the BTEL section is set to false in your Inputs.xlsx file for any namespace, then the script does not install the corresponding NCC Telemetry and CVEA Helm charts of that namespace. The script installs the corresponding NCC Telemetry and CVEA Helm charts for a namespace, if that parameter is set to true in the Inputs.xlsx file for that namespace.

Note:
There are multiple example scripts provided in each system's examples directory. These scripts are examples that may not work for every environment.

More examples are included in the input generator's staticfiles/examples directory. If you are installing all namespaces in a single cluster and have previously verified that the namespace-specific examples work in your environment, these files can be used to execute all the related scripts once for each namespace. This prevents you from switching to each namespace's directory to run the scripts.

  1. Using a text editor, open the examples/install-charts.sh script, verify the steps in the script, and adjust the script as needed by commenting or removing the lines that do not apply to your deployment.
  2. Execute the install script by entering: 
    cd output/SITENAME
./examples/install-charts.sh
    Note:
    To execute install scripts for all namespaces at once, enter:
    ./staticfiles/examples/install-charts.sh
