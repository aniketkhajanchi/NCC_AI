Preparing Helm chart input values

This procedure provides instructions on how to prepare the Inputs.xlsx file for your network deployment.

Note:
The Inputs.xlsx file must be edited with a currently supported version of LibreOffice or Excel since both applications provide the best support for input validation. Versions of LibreOffice prior to 6.2 do not support custom validation.
Note:
If you are running the Docker container on the machine that does not have a spreadsheet editor, transfer the Inputs${CHART_VERSION}.xlsx from the server to your local machine, edit it, and then transfer it back to the server using, for example, the scp tool:

Example to transfer the file to your local machine:

scp cloud-user@111.111.111:/path/to/input-generator/Inputs${CHART_VERSION}.xlsx

Example to transfer the file back to the server:

scp Inputs${CHART_VERSION}.xlsx cloud-user@111.111.111:/path/to/input-generator/
The Inputs.xlsx file allows you to specify customer values for the Helm charts. To modify the values that are not exposed in the Inputs.xlsx file, you create a custom YAML file, which is specific to each Helm chart. For example, if you need to modify the values in the User and Role Management and NCC Telemetry Helm charts, you need to use two custom YAML files: one file for the User and Role Management Helm chart and the other file for the NCC Telemetry Helm chart. For more information about the contents of the Helm charts and Inputs.xlsx file, see the following:
  • For the descriptions of the values in the Helm charts, see the charts' README files or run a queryhelm show values [CHART].
  • For more information on the Aerospike parameters, see section Aerospike parameters.
  • For more information on the topology parameters, see section Topology parameters.

  1. Ensure that Excel or LibreOffice version 6.2 or later is available on your workstation where you are going to edit the Inputs.xlsx file.
    For more information on how to install LibreOffice, see LibreOffice documentation.
    Note:
    The following commands might help you determine what LibreOffice version your system has.
    To check the version, enter: 
    yum info libreoffice
    If your version is older than 6.2 and cannot upgrade using yum, download a newer RPM version, remove the existing version, and install the newer version by entering:
    sudo yum remove openoffice* libreoffice*
cd /path/where/you/saved/the/file/
tar -xvf LibreOffice*.tar.gz
cd LibreOffice*/RPMS/
sudo yum localinstall *.rpm
  2. Ensure that the structure of the Inputs.xlsx file corresponds to your deployment as follows:
    • Remove the tabs that represent the namespaces that are not in your deployment. For example, if IG is not in your deployment, remove the IG tab.
    • If your deployment contains more than one system of a certain type, duplicate these tabs. For example, if your deployment has two MEs, duplicate the ME namespace tab.
    • If the NCC Telemetry is shared across multiple NCC systems in the same Kubernetes cluster, ensure that each of those NCC systems has a corresponding worksheet for Telemetry.
    Note:
    All worksheets in the Inputs.xlsx file are taken into account for the network topology. For example, if your deployment has one SM and two MEs that belong to the same cluster, and you install shared NCC Telemetry, your Inputs.xlsx file must have the Common worksheet, one worksheet for SM, two worksheets for ME, and one Telemetry worksheet.
  3. For each namespace's Ingress node, create an FQDN.
    Note:
    This FQDN is used for handling the internal NCC traffic.
  4. Fill in the Inputs.xlsx file as needed and required.
