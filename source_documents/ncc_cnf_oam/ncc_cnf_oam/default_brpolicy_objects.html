Default NCC Backup and Recovery objects

The table below shows a list of NCC Backup and Recovery policies in an NCC deployment when NCC Backup and Recovery is enabled.

The names of the NCC Backup and Recovery objects follow the pattern <ReleaseName>-<ChartName> unless a fullnameOverride has been specified for the chart at install time in the override values YAML file. In that case, the NCC Backup and Recovery policy name will be the value assigned to the fullnameOverride property.

One exception is the name of the NCC Backup and Recovery policy object for MariaDB, which also has the suffix -mariadb.

Component Chart name Release name example BrPolicy name example Data backed up
admincli admincli *-admincli *-admincli Aerospike data
csdc csdc *-csdc *-csdc etcd configuration data
ckey ckey ckey *ckey-ckey secret
cmdb cmdb ckey-cmdb *ckey-cmdb-mariadb User and Role Management authentication configuration
btel cmdb cmdb *-cmdb *cmdb-*-mariadb Metrics Viewer configuration data and cnot alarms
ss7 cmdb cmdb *- ss7cmdb *-ss7cmdb-mariadb SS7 data
cpro cpro *-cpro *-cpro-server Prometheus metric data
gen3gppxml cpro-gen3gppxml cpro-gen3gppxml *cpro-gen3gppxml gen3gppxml data that is waiting to be retrieved by ftp
When looking for details of an NCC Backup and Recovery policy, or history of past backups, or restore operations for a specific component or release, use these commands on a control node. 
# Set the following to the appropriate value, for example:
NAMESPACE=<namespace>
                    
# Find the desired BrPolicy name:
kubectl get brpolices.cbur.csf.nokia.com -n $NAMESPACE

#Example output:
# NAME                    AGE
# ckey-ckey               23h
# ckey-cmdb-mariadb       23h
# btel-gen3gppxml         162m
# cmdb-mariadb            162m
# cpro-server             162m
# grafana                 162m
# sps-btel-sps-admincli   11m
# sps-csdc                11m

# Use the BrPolicy name to see details
# For example:
BR_POLICY_NAME=ckey-cmdb-mariadb
                    
# This command gives information about the current cronJob specs, if scheduled backups are enbled or disabled, and Backup History data.
kubectl describe brpolices.cbur.csf.nokia.com $BR_POLICY_NAME $NAMESPACE
