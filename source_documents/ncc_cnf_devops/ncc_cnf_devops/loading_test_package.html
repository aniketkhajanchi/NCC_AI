Loading test package

The following procedure describes how to load a test package in the CCTF UI.

  1. Select and expand the target product in the product list, and then click Load a test package, for example:

    A Load a test package pop-up window opens:

  2. Fill in the necessary fields.

    Mandatory fields (when uploading a remote test package):

    • Upload Method - Users could select to load a test package via a URL or from the local machine. Below information are also needed.
    • Test Package Name - The name of the test package, which will be shown in the CCTF UI.
    • Test package URL - The URL through which the test package is downloaded.
    • Docker Image - The URL of the Docker image, which is used to bring up a container as the run time environment for the test package.

    Optional fields (when uploading a remote test package):

    • Docker Registry Server - The Docker registry server for the Docker image. Together with 'Docker Registry User Name' and 'Docker Registry Password', it is used to log on the Docker registry before a Docker image can be pulled. These three data items are optional.
    • Docker Registry User Name - The user name to log on the Docker registry server.
    • Dokcer Registry Password - The password for the user to log on the Docker registry server.
  3. Click ADD.
