SSO/LDAP migration introduction

Note:
SSO and LDAP are mutually exclusive configurations. The procedure described here is similar for both and the differences are noted where applicable.

This section describes the steps used to upgrade NCC from a non-SSO/LDAP configuration to a SSO/LDAP configuration in a multi-system network deployment with geo-replication enabled for the MariaDB for User and Role Management in the IG systems. The focus is on the steps needed to upgrade User and Role Management, MariaDB, Servicemanager, Metrics Viewer, chf-resource, and CGF (DR) Helm releases that span multiple sites.

Due to differences in the configurations, you must disable the MariaDB data replication between IG systems that are not migrated to SSO/LDAP configuration or replication issues will occur.

This process for upgrade can be different depending on which IG system is being upgraded in the deployment and whether or not it uses SSO/LDAP configuration along with the other IG systems in the deployment at that time.
