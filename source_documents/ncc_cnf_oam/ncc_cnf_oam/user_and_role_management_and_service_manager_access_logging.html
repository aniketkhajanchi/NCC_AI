User and Role Management and Service Manager access logging

NCC has various security events that can be monitored via logging.

User and Role Management events

User and Role Management can be configured to record events via the User and Role Management GUI. You can enable events for LOGIN and LOGIN_ERROR under Realm > sps-auth > Events > Config. These events are then visible in the Login Events tab.

These events can also be viewed in the OpenSearch log viewer by searching for LOGIN and LOGIN_ERROR in the log entries.
Note:
Other events can be enabled in the User and Role Management GUI, but some are logged at a lower level and do not appear in OpenSearch by default, even though they are visible on User and Role Management GUI events report (for example: LOGOUT).

User and Role Management does not support a method for exporting the events directly.

Service Manager events

Service Manager logs these events for authentication and authorization issues which are also visible in OpenSearch.
  • SM_UNAUTHORIZED_ERROR - occurs when the user requests an operation that their permission settings do not allow for.
  • Unauthorized - occurs when a request is received with an invalid or expired bearer token or if the request has a basic auth header, bad username, or password.

Service Manager API logging

Service Manager supports logging of input REST API requests and their corresponding responses, along with meta data such as the requesting user and timestamp. This is not enabled by default. To enable these, select the REST APIs Tracing system parameter check box for the SM GUI, on the SM Platform > Preferences page. API requests and responses are then logged by Service Manager.

Saving log results

Logs are collected in OpenSearch and can be indexed and queried by using the provided OpenSearch tools.
