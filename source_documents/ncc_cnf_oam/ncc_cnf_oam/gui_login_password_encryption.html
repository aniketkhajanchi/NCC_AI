GUI login password encryption

The login GUI page, as with all GUI interaction with the NCC system, is protected by the encryption of a TLS session. If an extra level of encryption is desired, specifically for the password field of the submitted login form, this is supported by an extension of User and Role Management which is made available as an alternate Keycloak image you can install and enable by selecting an optional theme to use for the login page.

Helm chart customization

A helm chart override is required for deploying this customized image rather than the default Keycloak image that is delivered by NCC. The overrides to use the updated image are provided as documentation and are applied by the customer team.

  1. Place this file in the input-generator/overrides directory.
  2. To manually set the imageTag value in the spreadsheet, run this command from the Input Generator root and edit the value: 
    ls charts/ckey-chart-*.tgz | grep -oE '[1-9]+[0-9.-]+\.tgz' | sed 's/\.tgz$//'
  3. To automatically set the imageTag value in the spreadsheet, run these two commands from the Input Generator root and edit the value: 
    CKEY_VERSION=$(ls charts/ckey-chart-*.tgz | grep -oE '[1-9]+[0-9.-]+\.tgz' | sed 's/\.tgz$//')
sed -i "s/^imageTag: .*/imageTag: ${CKEY_VERSION}/g" overrides/ckey-login-encryption.yaml
  4. Reference this file in the Inputs.xlsx on each IG tab under: CKEY > Overrides 
    imageRepo: chf/ckey
imageTag: CKEY_VERSION

Activation

Under Realm sps-auth > Client spssm > Settings, select the nokia-pswd-enc value from the Login Theme drop-down list.

A similar setting can be changed under the btel-auth realm for clients: sps-calm, sps-grafana, and sps-kibana. This theme is identical to the nokia-csf theme except it encrypts the password on submission of the login form.
