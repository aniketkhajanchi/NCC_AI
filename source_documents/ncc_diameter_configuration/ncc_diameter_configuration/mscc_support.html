Multiple-Services-Credit-Control AVP support in the NCC Gy interface

Multiple-Services-Credit-Control AVP is of type Grouped and contains the AVPs related to the independent credit-control of multiple services feature. Each instance of this AVP carries units related to one or more services or a single rating group or single service identifier. NCC supports multiple MSCC.

Note:
The following applies to multiple MSCC:

  • If both rating group and service identifier values are received in a Diameter request, then rating group takes precedence over service identifier. Also, if multiple service identifiers are received in a Diameter request, then the first service-identifier value is considered.

  • Unit type is concluded at CCR level and not at MSCC level.

  • Currently, if USU is received with some value in CCR-I, then set it to Null and value of USU is ignored without throwing any error and same is followed.

  • Multiple MSCC support for IEC is for selective cases only.

Handling of asynchronous MSCC

Consider the following points when handling the asynchronous MSCC:

  • If RSU AVP is not received in the CCR-U for a specific MSCC, then that MSCC is considered to be closed.

  • A Diameter Gy session is not terminated even if all rating sessions in a CCR-U are terminated. The Diameter Gy session is cleared by audit or CCR-T.

  • If all the rating sessions in a CCR-T are terminated, then only the Diameter Gy session is terminated.

  • If a mandatory AVP is missing, then a global message in CCR-T is considered as an RFC-4006 error.

  • If a mandatory AVP is mentioned, then all rating sessions in CCR-T are deleted followed by the deletion of Diameter Gy session.

  • If any new MSCC is received in a CCR-U, then a new rating session is opened for that specific MSCC, rating group, or service identifier.
