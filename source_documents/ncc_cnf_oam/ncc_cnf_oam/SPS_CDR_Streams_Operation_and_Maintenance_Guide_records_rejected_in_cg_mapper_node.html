Records rejected in CG Mapper Node

If a record cannot be processed in CG Mapper Node, a warning is raised and the record is rejected. Rejected records are first stored in the database, and the keys of the rejected record are appended to the "{rejected}sourceQueue" list. After that, rejected records are transferred to a remote server, as configured with the node parameters of CG Rejection Distributor Node in the Reprocessor stream. The trasfer to the remote server starts after the limit configured with node parameter RecordsPerBatch or SecondsPerBatch of the CG Rejection Distributor Node has been reached.
