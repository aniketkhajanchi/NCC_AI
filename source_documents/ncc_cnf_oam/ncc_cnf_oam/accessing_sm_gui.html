Accessing the NCC GUI

Setting up access to the NCC GUI from the User and Role Management admin console

As the administrator, select Sps-auth from Realm Selection in the upper left of the screen then choose Clients > Settings. Configure these items to allow access to the NCC GUI:
  • Root URL
  • Base URL
  • Valid Redirect URIs: This is the URL that is used to access the SM GUI. If the user accesses the SM GUI via another alias, an error message will pop up, and a redirect will occur to the configured URL, so this should be set to the most commonly used URL. For example: 
    https://$SM_FQDN/servicemanager/*
  • Admin URL
    This should be the same as the root URL except port 8666 and path /auth are present in the URL. For example: 
    https://$SM_FQDN/auth
  • Web Origins:

    * is the most permissive value. To view the other values that are available, see the online help.
