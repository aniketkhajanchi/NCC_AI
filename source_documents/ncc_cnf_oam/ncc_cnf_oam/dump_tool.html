RMS dump tool

The dump tool is used to export voucher information to a file. This tool supports these scenarios:

  • The tool is executed on the standby RMS.
  • The tool generates a compressed file.
  • RESTful APIs for add/stop dump jobs are supported.
  • The tool is executed automatically (after every configured time interval).
  • The tool defines disk space threshold (for example, 80%), exceeding that threshold stops the dump.
  • The tool removes files older than x days. This value should be configured with a small value to save disk space.
  • The tool does not execute if a concurrent execution is in progress.
  • After every successful or failed dump, the service sends one email and/or short message notification to indicate the end of the dump operation, if NCC Telemetry is properly setup and related email/smpp addresses are correctly configured.

Directory Description

Dump operation RESTful interface commands

  • Add job

    This command is used to enable the dump operation. After this command is triggered successfully, the dump task is executed periodically according to the configured frequency.

    Attribute Value
    HTTPS Operation POST
    URL https://<RMS IP>:<port>/RMSAPIServer/restApi/v1/dump/add
    Response Return a string. If succeeds, will return “Process dump add operation successfully, dump is enabled”.
  • Stop job

    This command is used to disable the dump operation. After this command is triggered successfully, the dump task is not scheduled again. Any current tasks are not impacted.

    Attribute Value
    HTTPS Operation GET
    URL https://<RMS IP>:<port>/RMSAPIServer/restApi/v1/dump/stop
    Response Return a string. If succeeds, will return “Process dump stop operation successfully, dump is disabled”.

Dump operation sequence

  • If you want to enable a dump task, send a RESTful add command to the standby RMS. If successful, a return response of “Process dump add operation successfully, dump is enabled” is generated. If a dump task has been enabled, a return response of “Add operation has been sent before” is generated.
  • A dump task is executed periodically according to the configured frequency.
  • After the dump is completed, you can collect the dumped files from the /rms/files/dump/output directory by using SFTP.
  • If you want to disable a dump task, send a RESTful stop command to the standby RMS. If successful, a return response of “Process dump add operation successfully, dump is enabled” is generated. If a dump task has been disabled, a return response of “Stop operation has been sent before” is generated.

Dump operation output files

After the dump operation, the tool generates two compressed files in the /rms/files/dump/output directory. One for the generated cards information named generated_cards_dump_YYYY_MM_DD_HH_MM_SS.csv.gz and another for the imported cards information named imported_cards_dump_YYYY_MM_DD_HH_MM_SS.csv.gz.

This table lists all of the dumped fields of generated cards and their descriptions.

Table: Dumped fields of generated cards
Field Description Type Length
voucher_serial_no This field indicates the serial number of the card. It is from serial_no in the ucms_cards table. NUMBER 38
status This field indicates the state of the card. It is from card_status in the ucms_cards table. VARCHAR2 20
recharge_amount This field indicates the recharge value that is associated with the card type. It is from recharge_amount in the ucms_card_types table. NUMBER 38
face_value This field indicates the monetary value that is associated with the card type. It is from denomination in the ucms_card_types table. NUMBER 38
generation_date This field indicates the generated time of the card. It is from generated_time in the ucms_cards table if rcmsserver.db.time.extension is enabled, else from the date_placed in the ucms_manufacturer_orders table. DATE N/A
activation_date This field indicates the activated time of the card. It is from activated_time in the ucms_cards table if rcmsserver.db.time.extension is enabled, else is null. DATE N/A
used_time This field indicates the used time of the card. It is from used_time in the ucms_cards table. DATE N/A
expiry_date

This field indicates the expiry date for the batch of cards. It is from the ucms_cards table (has higher priority) or the ucms_batches table if the rcmsserver.card.level.expiry.date is enabled, else from the ucms_batches table.
manufacturer This field indicates the name of the manufacturer. It is from the manufacturer_name in the ucms_manufacturers table. VARCHAR2 20
distributer This field indicates the official name of the distributor. It is from the distributor_name in the ucms_distributors table. VARCHAR 50
