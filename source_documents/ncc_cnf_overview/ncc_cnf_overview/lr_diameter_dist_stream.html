Late rating Diameter distributor stream

Late rating Diameter distributor stream consists of only one node, Late Rating Diameter Distributor Node, which collects records from the database and converts them into Diameter messages, then sends them out as Diameter requests.

After the records have been processed and mapped by Late rating business logic stream, Late Rating Diameter Distributor Node constructs a Diameter message for every serialized event record written to the database, where the serialized event record is converted into a Diameter message by adding the extra fields needed to meet the Diameter standard, which is represented is configurable dictionary.xml.

Late Rating Diameter Distributor Node saves the OutputRecord object in a list and establishes a connection to the Diameter server based on the node parameters configuration of Late Rating Diameter Distributor Node and a configurable stack.xml file, and passes the list of output records for sending the list as Diameter requests. If the connection to the Diameter server fails, all the output objects lists are saved to a failed list. Otherwise, the stream keeps listening to the responses of Diameter requests, and if it receives any unsuccessful responses, the corresponding original records are stored to the failed list as well. After that, all the records in the failed list is written to the database log queue to be handled by Late Rating Log Distributor Node. The log queue is defined with node parameter LogQueueName.

Scaling

Late Rating Diameter Distributor Node can be scaled by adding multiple instances of the node into the stream. There can also be multiple Late rating input streams working in parallel.
