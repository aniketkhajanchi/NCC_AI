Aggregation use cases

  • Complete fully successful session, which includes initial, updates (optional) and terminate requests, which are aggregated into multiple CDRs per rating group.

    Aggregated CDRs are dumped into CDR/GY/FINAL file directory and cdr-gy-final Kafka topic, while partial CDRs are dumped into CDR/GY/PARTIAL file directory and cdr-gy-partial Kafka topic.

  • Partially successful CDRs, including multiple rating groups with both success and failure results.

    Successful requests for the same rating group are aggregated into the final destinations mentioned above, and failure requests are dumped as failed partial CDRs into CDR/GY/FAILED_PARTIAL file directory and cdr-gy-failed-partial Kafka topic.

  • In case of any missing requests, a warning appears on the CGF (DR) UI and no aggregated outputs are generated.
  • In case of missing termination with a report reason = Final in the last occurrence per MSCC, the successful requests per MSCC are aggregated and dumped into the final directory and Kafka topic mentioned above.
  • In case of missing termination with a report reason = Final in the last failed occurrence per MSCC, no aggregation is done and partial failure CDRs are dumped into the failed partial directory and Kafka topic.

All the cases listed have partial CDR outputs dumped into the partial directory and Kafka topic.
