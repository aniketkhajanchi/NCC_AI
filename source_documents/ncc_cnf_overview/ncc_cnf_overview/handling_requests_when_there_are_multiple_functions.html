Handling requests when there are multiple subsystems

4G/5G network traffic can arrive at any NCC ME subsystem and be handled. If a subsystem does not own the data associated with the device, the request is internally routed to the subsystem that does own the device through a mechanism called site routing.

API traffic (TMF, native API, and eCommerce) can arrive at any NCC IG subsystem and be processed locally (catalog and configuration data) or sent over SOAP to the primary NCC ME subsystem for the subscriber.
