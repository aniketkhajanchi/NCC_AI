Recovery from catastrophic disaster (complete re-install)

On the recovering system SM4, you must perform a Helm install by using the original release names and the original overrides file.

Since this installation is done for disaster recovery, you must generate the overrides again by using the same Inputs.xlsx file that was used for the original install and edit the values that are listed in this table.

Tab Section Property Value
Common Global Deployment Type Disaster Recovery
Common Global Enable XDR Traffic false
SITE CKEY MariaDB Replication Enabled false

This disables replication for both Web SSO data and Aerospike data, by omitting the creation of the Kubernetes object needed for networking. This keeps the recovering system isolated until it is safe to add it back to replicate data with other systems.

The deployment type Disaster Recovery also disables automatic Backup and Restore cronjobs which at this stage could overwrite meaningful data with empty data.
