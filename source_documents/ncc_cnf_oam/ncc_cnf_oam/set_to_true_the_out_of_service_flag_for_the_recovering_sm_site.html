Set to true the out of service flag for the recovering SM system

Before you start the recovery procedure, you must set the recovering SM system as out of service.

## On SM4, run this command:
kubectl -n $NAMESPACE -it exec <admincli-pod> -- /opt/tpa/bin/configure-manual-oos -d SM
 
## On IG also need to run this command:
kubectl -n $NAMESPACE -it exec <admincli-pod> -- /opt/tpa/bin/configure-manual-oos -d ME
