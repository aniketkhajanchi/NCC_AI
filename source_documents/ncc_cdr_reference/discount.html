This parameter is the Record property, dumped when "Add 5G Device Identities in EDR/CDR" is set to 1.

This parameter takes precedence over Include Final Usage In BCR EDRs flag. When both the flags are enabled, Device IDs are saved in "deviceIdentityInfo” domain instead of “listOfSubscriptionID” domain. Valid values: 0 (disabled), 1

This is present if discount is attached with Device.

 RecordProperty
deviceIdentityType

This tag indicates the Device ID type.

Note: There can be multiple deviceIdentityType and deviceIdentityValue tag sets. This indicates multiple Subscriber IDs in the created Device.

Valid values: 0 (e164), 1 (iMSI), 2 (sIP-URI), 3 (nAI), 4 (uSER-PRIVATE), 100 (eXTERNAL-ID), 101 (dEVICE-ID), 102 (sUPI), 103 (gPSI).

 0
deviceIdentityValue This parameter is the value of the device identifier. Device012172
discountedPeriods Indicates the maximum number of times a discount can be applied. If a value is not specified, then it denotes that unlimited renewals can take place. 10, 12...
discRemPeriods Indicates the current value of the remaining discounted periods. 1, 2, 3, 4...
discPriority It is priority of discount which is applied when multiple discounts are applicable. 101
discountStartTime If a start date and time is defined, then the discount does not apply until the current time is on or after the start date and time. If no start date and time are defined, then the discount is immediately available when it is applied to a device. 10/06/2020 02:01:34
discountEndTime If an end date and time is defined, then the discount does not apply after the current time is after the end date and time. if no end date and time is defined, then the discount remains available and never expires. 13/06/2020 02:01:34
deletedDiscount This parameter indicates the discount which is deleted. D1
