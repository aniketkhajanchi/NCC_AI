NCC service

NCC identifies core functionality with different pod types as a service. Each service is configured with a different formula type to indicate the system knows it is healthy. This concept is proprietary to NCC. Examples include the lock and transaction manager, service manager, provisioning, or Aerospike.

Each service has a state of either up or down.
