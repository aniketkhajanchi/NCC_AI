Record event results for Gy and IMS sessions

Whenever a session/event request has occurred, it is either successful or unsuccessful depending upon the logic processing in NCC. The RecordEventResult tag contains the numeric number created by NCC explaining the cause of session/event closure.

Example: recordEventResult = 21

Following are the record event result ranges with their descriptions:

  • 00-20: Related to successful call
  • 21-40: Call unsuccessful due to insufficient balance
  • 41-80: Error due to object not found in database
  • 81-100: Call unsuccessful due to some problem in rule engine output
  • 100-130: Call unsuccessful due to some error, for example Device Barred, Subscription Barred, Group Barred
  • 131-140: Call unsuccessful due to some error in charging service
  • 141-150: Call unsuccessful due to some error in subscription or group
  • 151-200: Call unsuccessful due to some problem in rate configuration
  • 200 onwards: Call unsuccessful due to some configuration problem at call control
  • 300-320: Session audit result codes for successful/unsuccessful result due to different events, like RAR. These are listed in Record event results for session audit CDRs
