Parameter file

The CCTF test package is portable and flexible to be used to verify different SUTs with minor or no changes. To aim for keeping the CCTF test package SUT independent, the configuration information of SUT is taken out from the test cases and put into one parameter file. This parameter file is then filled with correct data for a specific SUT or lab.

The parameter file is named as parameters.yaml and saved under the top directory of the test package. NCC supports only YAML format (while some other test products and CCTF might support also JSON format). The default parameters.yaml file shipped with the spstests test package is as follows:
