User administration

Overview

The maintenance of GUI users, passwords, and role mapping are performed by using the User and Role Management GUI. These tasks are performed by the administrator. The user administration on the NCC GUI is replaced by the User and Role Management GUI, wherein the user logs in again as a User and Role Management administrator to perform actions other than changing their own password.

If NCC is integrated into the your SSO environment, or is configured to use LDAP to a remote authentication server, then users are NOT configured into NCC. The only users configured into NCC would be a few special users for emergency access in case the your authentication servers are inaccessible.

Creating a new or onboarding a new user

The User and Role Management administrator creates new users using the these methods:
  • The administrator creates a new user using the given attributes:
    • Blank email address:
      • In the case that the email client is not configured, mark the "Email Verified" field as ON and make sure the "Required User Actions" is empty to enable user to log in.
    • Temporary password, which is further communicated to the user separately.
    • Required User Actions, set as Verify Email (Only if email client is configured).
    • Assigned a single NCC user role.
  • When a user logs in for the first time to the GUI:
    • Upon successful login, the user is forced to change their password.
    • If the Administrator did not enter the email ID when creating the new user, the user is redirected to the account settings screen where adding the email address is a mandatory entry.
    • The user logs in again to the GUI and after entering their credentials, the user is informed that email address is required to be verified and an email has been sent to the user email address with a link to validate.
    • The user receives the email, clicks the link, and logs in directly without entering any username or password.
    Note:
    This flow can be bypassed if the User and Role Management administrator creates the new user with a profile with the email verified as ON and the Required User Actions field is empty.
    Note:
    The correct "email host" and a "from email id" should be configured to enable User and Role Management to actually send an email to the user's email id. That configuration is done from: Spsauth → Realm Settings → Email in the Admin Console. The email host that is given should have an email client running on it.

Forgot or change user password

When the initial GUI is loaded and redirected to the User and Role Management login screen, a forgot password link is provided to invoke the User and Role Management features to administer the individual password of a user and for changing the password. A link is added to the NCC GUI, which in turn invokes the User and Role Management features.

Note:
The default password expiration policy is 90 days. You can change the password before it expires or edit/delete the password policy value by using the GUI.
Follow these steps to edit or disable the password expiration policy.
  1. Log in to the User and Role Management GUI as the administrator.
  2. Click on Manage > Users in the left navigation panel.
  3. Use the search field to select the user you wish to edit.
  4. Remove the Update Password value from the Required User Actions field on the Details tab.
  5. Click on Configure > Authentication in the left navigation panel.
  6. Select the Password Policy tab.
  7. Edit the Expire Password > Policy Value field to change the value or click Delete from the Actions column to remove the policy. You can re-add the policy by using the drop-down menu, if necessary.
