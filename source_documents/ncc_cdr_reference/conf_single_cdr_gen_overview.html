Configuring single CDR generation

NCC supports per-request and per-session CDR generation. In case of per-session CDR, the CDR is not generated until there is an active session. The CDR is generated only when a trigger to generate a CDR is received. The trigger can be a terminate request or audit.

Single CDR can be configured in IMS, SMS, and MMS calls using rules. If this is configured, then a single CDR is generated for MOC, MTC, and FWD calls at the time of call termination or at audit.

Note:
The single CDR feature does not take in to account or process the following operations that can happen during the call:
  • provisioning operation like account/bucket balance adjustment, reset/renewal of resources (for example, Bucket, Counter, Account)
  • time tariff switch
  • resource versioning (reserved for future use)

In such cases, CDR tags showing Before and After values of resource are not accurate.

Single CDR for IMS calls is not generated if the session still remains in the database, for example, when Update/Terminate requests fail with error code 5012. However, it is generated for Initial requests that fail with 5012, as the session is not created (therefore, it is not present in the database) when the call fails. It is generated if the request fails with error code 4012.
