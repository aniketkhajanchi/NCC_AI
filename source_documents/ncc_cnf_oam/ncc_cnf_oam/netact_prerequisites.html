NetAct prerequisites

This section provides information about the parameters that must be configured prior to NetAct integration. Most of these parameters are configured as part of the NCC installation. For more information about each parameter that is listed, refer to the Installation and Upgrade Guide. If the configuration of any of these parameters was missed during the NCC installation, contact Nokia Support.

These fault management configurations must be set for NCC and NetAct integration:
  • Alarm Service Admin Password
  • Alarm Service Admin Username
  • Alarm Service database Password Encrypted
    • Enabled
    • Engine ID
    • Heartbeat Interval
    • Notification Targets Configuration
    • SNMPv2 User Configuration
    • SNMPv3 User Configuration
These performance measurement (metrics) configurations must be set for NCC and NetAct integration:
  • Gen3GPPXML SFTP Password
  • Gen3GPPXML SFTP Password Enabled
  • Gen3GPPXML SFTP Password Private Key
  • Gen3GPPXML SFTP SSH Key Enabled
  • Gen3GPPXML SFTP SSH Public Key

SFTP configuration

SFTP is used to integrate performance measurements (metrics) that are on the Gen3GPPXML pod with NetAct. The MEAS mechanism requires that SFTP is configured.
  1. The default SFTP user name is sftp. The minimum NetAct username length is six digits. You must install cpro-gen3gppxml with at-least a six-digit username. Update the user information in the cpro-gen3gppxml.yaml override file. 
    ...
  sftp:
    user: sftpuser
...
  2. The cpro-gen3gppxml file must be installed with the hideNonExistentMetrics value set to true. If this file is modified after installation, existing Gen3GPPXML files are ignored by NetAct. Only new files are processed. Verify the parameter is set correctly in the cpro-gen3gppxml.yaml override file.
  3. Any XML files that are generated by the Gen3GPPXML pod are placed in the /var/3gppxml directory.

NetAct integration for alarms (FM)

If a load balancer is installed on Istio, the istio-ingress-gateway must be bypassed since does not support UDP. For the ingress SNMP request to function, the load balancer must route the SNMP request directly to the Alarm Service service.

  1. Edit the citm-udp configmap and add an entry for the Alarm Service port (such as 1161) along with the Alarm Service namespace and service. 
    ...
                            
DATA
====
1161:
----
btel-ig2/calm-calm-snmp:1161
...
    Note:
    The load balancer must be installed with the scope value set to false, so that CITM can reach another namespace service directly. Edit the citm-ingress.yaml file and set the scope value to enabled: false.
  2. Since NetAct is configured with FQDN during setup, SNMP requests must contain the load balancer external interface IP. For SNMP requests to flow from the NCC Alarm Management to NetAct via the load balancer, external IP routes must be configured on the NCC Alarm Management worker nodes. Contact Nokia Support to configure external IP routing ports.
