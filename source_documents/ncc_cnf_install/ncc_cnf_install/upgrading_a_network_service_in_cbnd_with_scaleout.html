Upgrading and rolling back a network service in NCOM

You can perform an upgrade of the deployed charts by using NCOM. There are two different ways to create the NSD required for the upgrade.

  1. (Option 1) Create a new directory in examples/NCOM and copy the NetworkServiceDescriptor(rendered).zip into the new directory.
    1. Unzip the NSD package and make changes to the main.yaml file.
    2. Re-create the NSD package with the updated main.yaml file.
  2. (Option 2) Edit the MANIFEST file that is present in the NCOM directory and run the generate-network-service-descriptor.sh script. The script generates a new NSD. Since the name of the two NSDs are the same, the previous one is overwritten by the newly-created NSD.
  3. Once the new NSD is created, use the NCOM GUI to create a new NS package with the updated NSD. The name of new NS package must be same as the one which is to be upgraded but version should be different.
  4. Once the NS package is created, select the NS instance which is to be upgraded and select the update option. You will be redirected to the page where you have to select the NSD package version that you want to update. This updates the selected charts.

To perform a rollback, select the NS instance and select the roll back option to initiate the rollback.
