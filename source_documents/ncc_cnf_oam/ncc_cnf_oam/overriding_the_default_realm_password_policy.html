Overriding the default realm password policy

User and Role Management only supports one password expiration policy per realm. You must add a Group Policy to alter the realm password expiration behavior for certain users, such as machine-to-machine (M2M) users. Perform these steps to create the new policy.

  1. Log in to the User and Role Management GUI.
  2. Select the realm you want to add the new policy to.
  3. Navigate to Configure > Authentication.
  4. Select the Password Policy tab.
  5. Open the Add policy... drop-down list and select Group Policy.
  6. Provide a name in the Policy Value field for the new policy type.
    For example, groupPasswordPolicy.
  7. Click Save.
  8. Select the Required Actions tab.
  9. Click on the Register button.
  10. Select Group-based Expired Password from the drop-down list.
  11. Use the arrow buttons to move this entry to just below the Update Password entry.
  12. Navigate to Manage > Groups.
  13. Click on the New button.
  14. Enter a name for the new group.
    For example, m2mgroup.
  15. Click Save.
  16. Select the Attributes tab.
  17. Enter the policy name you previously created in the Key field.
  18. Enter forceExpiredPasswordChange(number_of_days) in the Value field.
    The valid values for forceExpiredPasswordChange() are:
    • 1 - n = positive integer for the number of days until expiration. For example, a value of 30 means the password expires in 30 days.
    • -1 = never. The password never expires.
    • 0 = always. The user must change the password on every login. This option is only used for testing.
  19. Click the Add button.
  20. Click Save.
  21. Navigate to Manage > Users.
  22. Click on the ID or the Edit button for the user you want to add to the new group.
  23. Select the Groups tab.
  24. Select the new group from the Available Groups section.
  25. Click the Join button.

    Expected outcome

    The new group appears in the Group Membership section.
  26. Repeat the steps to add a user to a group for any additional users you want to use the new policy.
