Node operations

Note:

This procedure must be repeated for every worker node of the system one by one, not together.

Broadly speaking, the node operations follows the below sequence.

  1. Find the pods on current worker node to be taken down for the maintenance activity.

    kubectl get pods --all-namespaces -o wide --field-selector spec.nodeName=<worker node name>

  2. Drain the code to evict the pods from the current worker node so that they can be scheduled on other worker node. Note the results of the drain operation

    kubectl drain <nodename> --delete-local-data=true

  3. Bring the node back into the operation by uncordoning the node by using the following command

    kubectl uncordon $NODENAME
