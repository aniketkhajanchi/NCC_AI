Configuring Vendor Specific Resource Definition (VSRD) values by using a custom YAML file

This procedure provides new instructions on how to configure a Vendor Specific Resource Definition (VSRD) by using a custom YAML file and the Vsrd Configuration file parameter in the Inputs.xlsx file. The previous method of configuring a VSRD through the Overrides parameter of Inputs.xlsx is deprecated. This new approach is recommended, but both ways to configure VSRD are still supported.

  1. Create the vsrd-configuration.yaml file in the input-generator/overrides directory.
  2. Add the custom data dictionary definitions to the vsrd-configuration.yaml file: 
    vsrd:
  localizationConfig: |
    CustomVendorDefinition=/opt/tpa/conf/sac/diameter/rule/CustomVendorDefinition.xml
    AttributeDefinitionVerizon=/opt/tpa/conf/sac/diameter/rule/AttributeDefinitionVerizon.xml
  CustomVendorDefinition: |
    <?xml version="1.0" encoding="UTF-8" ?>
    <diameterDefinition xmlns="diameterDefinition">
    <!--
      <vendor id="1" name="test"/>
    -->
    </diameterDefinition>
  AttributeDefinitionVerizon: |
    <!--
      Used to define the VSAs that have to be included to application contexts by the rule engine.
      -->

    VSRD need not be targeted to any specific component and is applied to all applicable containers. The file can just have the VSRD contents, starting with VSRD key.

  3. For each namespace for which you are overriding the VSRD, enter the vsrd-configuration.yaml file name as the value in the Inputs.xlsx file:
    1. In the Inputs.xlsx file, click the namespace's worksheet.
    2. Go to the SPS section.
    3. Go to the Vsrd Configuration file row and enter vsrd-configuration.yaml as the value.
    4. Remove any VSRD values from the Overrides property.
