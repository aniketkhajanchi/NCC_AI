Uploading container images to ECR

In case the deployment uses the Elastic Container Registry (ECR), the input-generator tool includes a script that can help with uploading the images.

Before running the script to upload the container images, perform these steps.

  1. Retrieve an input-generator for your release and preparing the spreadsheet inputs.
  2. Run this script to generate a manifest of the required images. 
    ./staticfiles/generate-manifests.py
  3. Create an ECR instance in AWS. You need to record the ECR registry ID as input to the script.
  4. All containers should be loaded into your local container image list.
    • For example, for each container tar image, you could run this command: 
      docker load -i XXX.tar
The script uses these arguments.
  • -d | –debug - Enable debugging.
  • -h | –help - Show help.
  • -a | –accountid - AWS account id. This assumes that the AWS command line tool is set up with authentication tokens.
  • -n | –reponame - Name of the ECR repository within the AWS account. If not passed it is created automatically.
  • -r | –region - AWS region.
  • -m | –manifest - The path to the input-generator manifest file created with the generate-manifests.py utility.
This is an example of executing the script: 
./staticfiles/upload_images_aws.py -d -a 2148-1263-6870 -r us-west-1 -m output/manifests/manifest-all.yaml
