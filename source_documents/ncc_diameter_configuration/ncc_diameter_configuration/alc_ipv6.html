ALC-IPv6-Address VSA manipulation under GXCCR context

Message manipulation for ALC-IPv6-Address VSA

The VSA ALC-IPv6-Address can be manipulated under the GXCCR context when you enable the application preference “Support Alc-IPv6-Address on Gx” to allow the Gx session to bind with the ALC-IPv6-Address. This application preference decides whether to bind Gx Session with ALC-IPv6-Address (VSA) present in CCR initial request. If this application preference's value is true and only ALC-IPv6-Address is present in the request all other IP-addresses (that is, IPv4, Framed-IPv6, and Delegated-IPv6-Prefix) are not present, then GX Session binds with ALC-IPv6-Address. Using message manipulation a rule can be authored that will add the Framed-IPv6-Prefix in the CCR that assigns the value from the “ALC-IPv6-Address” VSA. The following are rules that can be created for this type of message manipulation:

Rule to add ALC-IPv6-Address (VSA) in CCR that assigns the value from Framed-IPv6-Prefix

GxCCR.ALC-IPv6-Address.Add = GxCCR.Framed-IPv6-Prefix

Rule to add Framed-IPv6-Prefix in CCR that assigns the value from ALC-IPv6-Address (VSA)

GxCCR.Framed-IPv6-Prefix.Add = GxCCR.ALC-IPv6-Address

The following are other operations for ALC-IPv6-Address under message manipulation:

  • Rule to add the ALC-IPv6-Address AVP in the message:

    ALC-IPv6-Address.Add

  • Rule to set the ALC-IPv6-Address value:

    ALC-IPv6-Address.Set

  • Rule to set the ALC-IPv6-Address M-Bit in the message:

    ALC-IPv6-Address.M-Bit

  • Rule to set the ALC-IPv6-Address P-Bit in the message:

    ALC-IPv6-Address.Add.P-Bit

  • Rule to set the ALC-IPv6-Address V-Bit in the message:

    ALC-IPv6-Address.Add.V-Bit

  • Rule to remove the ALC-IPv6-Address AVP in the message:

    ALC-IPv6-Address.Add.Remove

  • Rule to remove the value of the ALC-IPv6-Address AVP in the message:

    ALC-IPv6-Address.Add.Remove-Value
