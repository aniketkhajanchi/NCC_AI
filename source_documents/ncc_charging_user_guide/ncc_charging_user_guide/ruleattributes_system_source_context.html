Generate-UUID All Triggers If this attribute is used, a time-based UUID string (with five parts separated with "-") is created.

Example of type 1 UUID: a89e6d7b-f2ec-11e3-bcfb-5c514fe65f28

Where, timestamp is formed from 1e3 f2ec a89e6d7b; 5c514fe65f28 represents the MAC address (for example, 5c:51:4f:e6:5f:28). The date 15/10/1582 is, in general, used as a staring point for UUID generation. This is the day when the Gregorian Calendar started.
  • NCC uses the first three parts of UUID as described in the calculation algorithm above.
  • The third part starts with 1 (that is, type 1), which indicates that it is a time-based string.
  • The fourth part can be any random UInt16.
  • The fifth part is the first six characters of the ME Node ID that triggered the notification. NCC uses ME host name in hex.
