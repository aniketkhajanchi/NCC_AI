updateSecret

Description

This operation updates the certificate stored in a secret associated with a Kubernetes namespace, provided the admincli pod has the appropriate cluster role to access the secret.

Note:
If you intend to update a certificate in a certificate chain, it is necessary to provide the complete chain along with the updated certificate. Merely providing a single certificate in a chained certificate does not suffice. You must copy the certificate file to the admincli pod and run the tool within that container.

Syntax

adminctl [-h] -o updateSecret -n NAMESPACE -s SECRET -f FROM_FILE
Option Description
-h or -? Displays usage/help information.
-n <namespace> Namespace where secret exists.
-s <secret_name> Secret name which contains the certificate.
-f <key:certificate_file_path> Secret key and certificate file path.

Multiple keys can be updated recursively (-f <key1:certificatePath> -f <key2:certificatePath2>).
Note:
When using the ! operator, the error bash: !: event not found may occur. Run the command set +H to avoid this.

Examples

To update a single key in a secret:

Command: adminctl -o updateSecret -n ig1 -s test-secret -f cacert:/opt/tpa/osgi/pandora/scriptlet/test_cert.pem
                
Output: 
cacert : Certificate updated successfully

To update multiple key in a secret:

Command: adminctl -o updateSecret -n ig1 -s test-expiry-secret -f cert:/opt/tpa/osgi/pandora/scriptlet/test_cert.pem 
   -f cacert:/opt/tpa/osgi/pandora/scriptlet/test_cert.pem
                
Do you want to continue? (y/n): y
cert : Certificate updated successfully
Do you want to continue? (y/n): y
cacert : Certificate updated successfully
