Get list of relevant sites

The disaster-recovery-helper.py tool may be used to return the list of which have any common namespace (partition or dscglobal or dsc) and where the commands should be run for recovery. Use the following command from a terminal on the recovering site.
Note:
Exclude unavailable or MOOS sites except for the recovering site.
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -w
Where:
  • <admincli> is the name of the admincli pod on the recovering site.
  • <container> is the name of the container on the recovering site.
  • <namespace> is the namespace of the recovering site.

As an example, the following shows expected output with growthme-admincli-b6496b469-hlmwb as <admincli>, growthme-admincli as <container>, and sps-btel-me5 as <namespace>.

[root@bcmtvzrm3-control-01 me5]# kubectl -it exec growthme-admincli-b6496b469-hlmwb -c growthme-admincli -n sps-btel-me5 -- /opt/tpa/bin/disaster-recovery-helper.py -w
2021-06-13T16:12:23,445 INFO: This tool must be run on following list of sites : ['me7', 'me6', 'me8', 'me5']
2021-06-13T16:12:23,475 INFO: Exiting...
INFO: Logs can be seen in log file '/tmp/disaster-recovery-helper.py.log'.
