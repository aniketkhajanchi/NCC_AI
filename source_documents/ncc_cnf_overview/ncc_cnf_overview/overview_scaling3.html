Scaling

CG Collector Node and ASN.1 Encoder Node can be scaled by increasing the number of node instances, but it must be done with caution to ensure that the other nodes in the stream can cope with the load. Otherwise, CG Collector Node stops collecting from time to time to let the other nodes finish processing the data already collected. Every time the collection stops, a warning is raised. Because CG Distributor Node cannot be scaled, the optimal scaling of the other nodes is three CG Collector Node instances and two ASN.1 Encoder Node instances.

There can also be several File output streams working in parallel. Different CG Collector Node instances do not interfere with each other when reading the same database.
