Diameter routes

NCC route configuration specifies which peer to use to reach a given host or realm. You can also specify which peer to use for a list of applications, for example, Gx, Gy, and Rx, to reach a specified host or realm. By default all applications are valid for a route. 

The routing tables that NCC uses to forward messages to the proper destination contain a large number of potential destinations and can therefore become difficult to manage. The tables are also frequently updated with new Diameter destinations and the removal of older ones. Routing based on patterns in destination names can be used to help manage the large number of destinations.

Pattern-based routing

The pattern-based server naming is used to associate multiple destination servers, for example by server type or server region. This use of pattern can be used in routing tables to shrink the size of routing tables.

NCC supports the following operations:

  • Uses patterns for Destination-Realm and/or Destination-Host values in the route specification
  • Performs pattern matching on all the routes
  • Uses the highest priority available route for routing the message

Pattern matching is supported for both realm and host routes.

Patterns are configured using the asterisk character as a wildcard in the destination host or destination realm fields in the SM GUI.

For example, for a realm route, any of the following would match the MyRealm route:

  • *
  • M*
  • M*lm
  • *m
  • *Rea*

For example, for host route, any of the following would match the homepcrf1.myrealm route:

  • *
  • home*
  • home*lm
  • *m
  • *pcrf1*
