Restore the database on the recovering system

Run these commands to copy the previous local backup files of the recovering system that are located in the /opt/tpa/data/db-backup/ directory of the of admincli pod.

Note:
Use latest backup. If a local backup is not available, use last Backup and Recovery file to restore the local database with the standard backup and restore procedure.
kubectl -n <namespcae> -c <container> cp db-backup/ <admincli>:/opt/tpa/data/db-backup 
# This example assumes that you have copied your db-backup to <local-dir>/db-backup (/root/db-backup/)
 
# Confirm the backup is copied to the admincli pod of the recovering system.
kubectl -it exec <admicli> -c <container> -n <namespace> --   ls -lRh /opt/tpa/data/db-backup/db-backup
