Recharge Management System (RMS)

The Recharge Management System (RMS) is a containerized solution based on a micro-service architecture. It runs as active-standby across two sites, assuming to be deployed in two NCS systems in two sites. The MariaDB enables HA across systems. RMS uses MariaDB in a primary-replica-replica configuration with 1+1 Maxscale. RMS MariaDB and Maxscale is a new instance (not shared with other NCC MariaDB instances). Even though the database replication is two ways as the RMS application runs as active-standby, the actual data replication at one time must be one way only. Therefore, there is no need to consider the data replication conflict scenario.
