Procedure to upgrade to SSO/LDAP configuration

Note:

Review these prerequisites before you proceed with the upgrades.

  • In deployments with integrated (IG) systems, the ME subsystem in that IG must be modified in the same maintenance window as the SM subsystem in that IG. In deployments that have IG systems and additional ME systems, one or more of the additional ME systems must be modified in the same maintenance window as the IG.
  • The CGF (DR) chart (Helm releasename=dr) must be upgraded.
  • The Metrics Viewer must be upgraded.
  • Do not make updates to authentication data before the migration procedure is completed for all systems. Else, some data might get lost in case of a rollback.

This procedure upgrades four SM/IG systems (1, 2, 3 and 4) as the use case. In the case of only two SM/IG systems, certain steps can be omitted.

Before you start the migration procedure, verify the MariaDB replication status on all SM/IG systems and ensure all pods are in sync and replicating with each other. It can take a few minutes for the replication status to display updated results. If the verification commands to check replication status indicate problems, see SSO/LDAP migration troubleshooting.
