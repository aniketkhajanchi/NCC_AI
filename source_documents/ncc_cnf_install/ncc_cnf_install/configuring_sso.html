Configuring SSO

Optionally, NCC can be configured to integrate with an operator’s existing Single Sign-On (SSO) environment, including SAML and OIDC integration with the operator’s Identity Provider. If NCC is to be integrated in this way during installation, these SSO properties must be set in the Inputs.xlsx file when you prepare the spreadsheet for a deployment.

To migrate an existing NCC deployment to utilize SSO, see the SSO migration section in the Operations, Administration, and Maintenance Guide.

  1. Set the Deployment Type value to installation in the Global section of the Common tab.
  2. Set the Enable SSO value to true in the Global section of the Common tab.
  3. Configure all of the properties in the SSO for GUI - SAML IDP section of the Common tab.
  4. Configure all of the properties in the SSO for SM REST APIs - OIDC IDP section of the Common tab.
