EDRs for rehome API

For rehoming, generic EDRs (for create, update, delete) are not generated at source and target MEs. The source and target MEs generate single consolidated EDRs.

Source ME

This section describes the EDR tags for rehoming generated at source ME. Following tags are generated from the data available in the rehome API request.

Table: EDR tags at source ME
Tag Description Example
entityId The Id of the Device based on which the Device is to be found. Device1142311
entityType This attribute stores the type of entity being rehomed. Currently only entity type "Device" is supported. Device
key The custom data key. IMSGROUPID
targetMEName Defines the target ME name to which subscriber is being rehomed. Mumbai
includeSessionData Indicates whether session data is included for rehoming. true
isRehomeRequired Indicates whether rehoming is required.

true: When rehoming decision is made.

false: When rehoming decision is not made. It is the case of only local update of change Identities or custom data.

 true
rehomeStatus Indicates whether rehoming was successful. This tag appears only if isRehomeRequired is true. SUCCESS
accountIds The account IDs rehomed. This tag appears only if rehoming was successful. [Account1142311, Account2142311]
userIds The user IDs rehomed. This tag appears only if rehoming was successful. [User1142311]
subscriptionIds The subscription IDs rehomed. This tag appears only if rehoming was successful. [0af5ec3d-da80-4bc3-969d-928b26e5eb12, a695b5cd-7485-4172-8e44-9011126a603b]
RecordProperty Group tag for deviceIdentityInfo. Multiple blocks may be present. deviceIdentityInfo
deviceIdentityType This tag indicates the Device ID type. 0
deviceIdentityValue Value of the device identifier. 9811423111
RecordProperty Group tag for changeIdentityInfo. Multiple blocks may be present. changeIdentityInfo
changeIdentitiesType The change identity type. 1
oldValue Old value of the identity. 40411142311001
newValue New value of the identity. 40411142311002
action The action to be performed on the identity: ADD, REMOVE, or UPDATE. UPDATE

Target ME

This section describes the EDR tags for rehoming generated at target ME. For target ME, a consolidated EDR (of Device, Account, and User) is generated for rehoming. For the description of tags, refer to the relevant sections for these entities.

Table: Additional EDR tags at target ME
Tag Description Example
sourceMEName ME name of device where it is originally created. GGN
targetMEName The ME where rehome request is migrating data. MUM
errorMessage The error message displayed for failure. ReHome Status: REMOTE_FAILED on targetME :...
rehomeResult Status of the rehoming. failed
result Result of the rehoming. ENTITY_00001
