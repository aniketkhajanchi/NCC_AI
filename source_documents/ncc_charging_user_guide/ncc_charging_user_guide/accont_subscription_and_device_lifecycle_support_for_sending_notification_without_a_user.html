Sending notification for a device without user

You can send notification through a lifecycle event using SendNotification action, which is available for account, subscription, device, and group. Using this action you can also send notification through a lifecycle event using the SendNotification action without a USER configuration.

When configured in account and subscription, the USER defines the association between the funded account and devices. Therefore, using the USER entity, devices are identified to send notification, and thus user configuration required to send notification. However, if an operator does not want to define any user with device or any association in an account entity, then this feature provides the flexibility to send notification without user configuration.

Configuration

If the action Notification.Disable_User_Configuration is configured in a lifecycle rule, then user is not used during notification processing. If the application preference No User Notification is enabled, then this attribute is ignored.

Note:
The No User Configuration is invalid if notification server is configured for SMS or emails.
