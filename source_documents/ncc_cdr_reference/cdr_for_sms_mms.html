CDRs for SMS/MMS

CDR tags for SMS

The CDR tags for SMS calls are same as that of Gy call (see CDR tags for Gy session) along with additional tags (or tags that have different meaning) described in the following table:

Table: Additional CDR tags for SMS

Tag

Description

Example
isOnNet

If the called and calling party are part of the same network.

Valid values: true/false

 true
recordEventType Represents call type. Its value is SMS for SMS calls.

For multi-recipients SMS calls, its value is SMS_MR.

 SMS
subRecordEventType

Represents call sub type for SMS.

Valid values: SMS_MO, SMS_MT.

 SMS_MO
cugGroups This tag is specific to the CUG feature. It dumps the list of Group IDs of all the common groups of called and calling party. [group1, group2]
originatorAddress Address-Data from Originator-Received-Address AVP of SMS-Information AVP. 1234556
numberOfRecipients Total number of recipients in SMS call.
Note:
This tag is available only when there are multiple recipients.
 5
recipientAddress Address-Data from Recipient-Info of Recipient-Address AVP of SMS-Information AVP. 1234557
multipleRecipientAddress A list of all the recipient addresses.
Note:
In case of MO MR-SMS call, multipleRecipientAddress tag replaces the recipientAdress tag.
