vFQDN resolution for NRF

With this feature it is possible to configure a vFQDN for a given NRF server. The vFQDN is resolved into FQDNs using an external DNS. Out of the resolved FQDNs, NCC uses the highest priority In-Service FQDN for Nnrf_NFManagement functions.

Note:
As a prerequisite, for this feature to work, in a given vFQDN all the associated FQDNs-NRF SERVERs are syncing/replicating with each other for NFProfile data.

The NF probe is executed for all the FQDNs resolved against the vFQDN of a NRF server if:

  • vFQDN is of primary NRF server
  • vFQDN of secondary NRF server only if it is in-use

For a given FQDN, based on the consecutive successes/failures of NF probe, its status is marked as In-Service/Out of Service, respectively.
