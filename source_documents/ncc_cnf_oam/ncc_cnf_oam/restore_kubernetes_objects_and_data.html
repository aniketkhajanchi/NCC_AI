Installation

On the recovering system, edit the values that are listed in this table in the Inputs.xlsx file.

Tab Section Property Value
Common Global Deployment Mode Disaster Recovery
Common Global Enable XDR Traffic true

Install the Helm overrides file to restore the Kubernetes objects. The data is restored later in this procedure. Do not re-enable scheduled backups that contain partition, dscglobal, dsclocal, or dsc data at this point to prevent overwriting the existing backups.

Due to enabling XDR traffic, incoming XDR starts but the mate systems could remain OUT_OF_SERVICE in the xdr-status tool results.

Web SSO

To restore a Web SSO instance which has experienced a catastrophic failure, see Web SSO data recovery.

CVEA

Reinstall CVEA by using the installation instructions with the original release name and overrides file.

NCC Telemetry

To restore an NCC Telemetry instance which has experienced a catastrophic failure, reinstall BTEL by using the installation instructions with the original release name and overrides file. To avoid overwriting the existing backups, do not re-enable scheduled backups at this point.

Once reinstalled with the original release names and overrides file, recover the NCC Telemetry data. The NCC admincli pod is used to accomplish this, so if NCC has also experienced a catastrophic failure, proceed with the NCC recovery instructions and return to these instructions to complete data recovery.

To recover the data, see NCC Telemetry backup and restore. Once the data has been restored, re-enable scheduled backups by using the installation instructions.
