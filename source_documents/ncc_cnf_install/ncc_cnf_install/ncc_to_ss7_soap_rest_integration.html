NCC to SS7: SOAP/REST integration

The SS7 service can be configured to act as the server and NCC the client to send notification messages to HLR. For successful calls, you must complete these basic configurations for the USSDGW and APIGW services. The USSDGW service is used to receive notification message from APIGW and send the message to HLR. All of the HLR configuration is done at the USSDGW service. You can edit the configurations via the OSP UI or the SS7 Manager GUI. Details are found in the NCC SS7 User Guide.

APIGW USSD response code object

This object is used to relay the HTTP code towards NCC. For the Notification feature, select the Request Name from drop down list. Configure messages for each HTTP response code that is sent to NCC.

USSDGW USSD notification configuration object

Configure all of the information about HLR based on the specific prefix (for example, HLR Global Title and HLR Primary Point Code). This instance is picked up while sending the messages.

USSDGW MAP response mapping object

Configure the HTTP codes for all responses that are received from HLR and relayed to NCC.
