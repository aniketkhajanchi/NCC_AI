NRF Event EDRs

This chapter provides information about the EDRs generated for NRF event.

EDR tags

Table: EDR tags for NRF event
Tag Description Example
RecordType Indicates the type of EDR generated. For NRF, its value is set as NRFEvent. NRFEvent
recordId

A unique ID for each CDR having format <DateTime>_UUID where <DateTime> is in YYYYMMDDHHMMSS format and UTC designation.

20220324110447_70df8c5d-16ce-4f71-a108-4f9980082b1a
entityId The entity ID for which the EDR is generated. 22aab74d-438b-4ad5-9fdd-dd7557ca2daa
nrfServerSetId Identifier for the NRF server set. NSet1
nrfFqdn Identifier for the NRF FQDN server. nrf.sl.nokia.local
nrfVersion The NRF server version. Possible values are 15_3_0 and 16_6_0. 15_3_0
nrfServerId Identifier for the NRF server. NRF_206
startTime The event start time. 1621492164918
endTime The event end time. 1621492165236
operationType The type NRF operation for which the CDR was generate. REGISTER
result The result of the call. SUCCESS
meHostName Host name of the pod that processed the request and generated the EDR/CDR. This tag is present in all CDRs/EDRs. naperil95vzwcchf-y-nk-ig-000-ig-charging-7cbf4fdd78-j6ccr
nccVersion The version of NCC. This tag is present in all CDRs/EDRs. SPS_21_6_R1
RecordTimestamp Signifies the date and time of the CDR/EDR generation. This value is taken from the CDR module timestamp. By default, it is presented in standard DD/MM/YYYY HH:MM:SS format, as shown in the first example.

However, the format can be changed to include milliseconds as well, as shown in the second example. This value is in millisecond format with timezoneoffset. Since RecordTimestamp tag is generated in GMT time, timezone offset is +0000. For this, the ETCD key timestampInMS must be set to true.

CAUTION:
Contact Nokia Support to change any ETCD key.
