Truncate recovering site dsclocal, dscglobal, and partitions data

dsclocal, dcglobal, and partition data should be truncated on the recovering site. To do so, run the following command on only the recovering site.

# For Catastrophic scenario we have to recover dsclocal also so need to Truncate dsclocal
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -t all # -t all will also include dsclocal
# Type y or yes and press enter when it ask for confirmation “Do you want to continue?: y|yes|n|no”
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
