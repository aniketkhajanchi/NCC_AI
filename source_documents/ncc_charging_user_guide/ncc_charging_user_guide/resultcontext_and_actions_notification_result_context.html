Notification-Continue THRESHOLD This action is used to send a notification and continue using the resource.
Selected-Device THRESHOLD This action is used to send notification to associated entities.

Format:

Notification.Selected-Device = <AssociatedEntityId>, <AssociatedEntityTemplateId>, <RecipientFlag>

Where,

  • AssociatedEntityId: Id of the entity to which the notification is to be sent.

  • AssociatedEntityTemplateId: It is an optional parameter. It indicates the notification template to be used for sending notifications to entities.

  • RecipientFlag: It is an optional parameter. It can take the following values:
    • ASSOCIATED_ENTITY_ONLY: If this value is configured, then notification is sent only to the associated entity using the notification template specified in AssociatedEntityTemplateId.
    • ENTITY_AND_ASSOCIATED_ENTITY: If this value is configured, then notification is sent to both the entities (calling recharge party and recharged party) using the notification template specified in AssociatedEntityTemplateId.
    Note:
    If RecipientFlag is not configured, then AssociatedEntityTemplateId is ignored, and notification is sent only to the associated entity specified in AssociatedEntityId.

Notification-Variable IMS_CALL_ESTABLISHMENT

This attribute is disabled by default but can be enabled using a rule. For any notification variable in a template, reports the notification content as variables and value pairs.

You can use the notification variables available and custom notification variables that you add, to return specific information in your message. For example, you can use the Notification Variables: Subscriber-User-Id and 3GPP-User-Location-Info along with a custom Notification Variable that you have defined (in this example, Customized-Date(2)) to return the Subscriber-User-Id and location information with the current system time in the format YYYY-MM-DD.

Custom Notification Variables must be added to the charging Rule Attribute Dictionary before you can use them.

The following lists default notification variables used:

  • 3GPP-RAT-Type

  • 3GPP-User-Location-Info

  • Current-Date-Time-String

  • Policy-Counter-Identifier

  • Policy-Counter-Status

  • Service-Parameter-Name

  • Service-Parameter-Value
