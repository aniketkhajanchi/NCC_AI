User and Role Management scheduled backup

The scheduled backups for User and Role Management and its MariaDB are enabled by default. After installing User and Role Management, scheduled backups may be disabled. For more information, see Scheduled backups.

The default schedules for the cron jobs that performs scheduled backups for ckey-ckey and ckey-cmdb-mariadb are set to the same value: "0 0 * * *" which means daily at midnight.

Update schedules for scheduled backups

To update the cronJob schedules defined at install time for either one of the NCC Backup and Recovery policies created by User and Role Management, a helm upgrade must be performed, providing the new value for the cronJob schedules, and reusing old values for all other properties.

Once updated, the scheduled backups must be enabled. For more information, see Scheduled backups .

# For example, this Helm upgrade will change the schedules to run the backup for ckey-ckey daily at 4am.
# As autoEnableCron and autoUpdateCron are set to true, the changes are immediately effective.
helm upgrade ckey ckey-chart-*.tgz -n $NAMESPACE --set cbur.cronSpec="0 4 * * *",cbur.autoEnableCron=true,cbur.autoUpdateCron=true --reuse-values --no-hooks --wait --timeout=1200
            
# As another example, this Helm upgrade changes the schedules in the policy to run the backup for ckey-cmdb daily at 3am.
helm upgrade ckey-cmdb cmdb-chart-*.tgz -n $NAMESPACE --set cbur.cronSpec="0 3 * * *",cbur.autoEnableCron=true,cbur.autoUpdateCron=true --reuse-values --no-hooks --wait --timeout=1200
Note:
The restore of User and Role Management MariaDB data by using backup and restore should only be performed in deployments that do not enable geo_redundancy. If geo_redundancy is enabled, the data on one site can be restored from a replicated site. For more information, see Web SSO data recovery.
