Mapping Assume Positive records

If the incoming record received by Business logic stream contains a field with name apDispositionRecord with value true, the record is marked as an Assume Positive record and will be mapped as generic.

To map disposed AP record as generic, we need a method for converting it back from internal format to varName-varValue like format (reversed functionality to current MappingUtils.convertGenericRecord()) to be consistent with Generic record in description.asn file.

GenericRecord ::= SET
{
recordType [0] UTF8String,
recordElements [1] SEQUENCE OF RecordElement OPTIONAL,
recordExtensions [2] SEQUENCE OF GenericRecordExtension OPTIONAL
}

GenericRecordExtension ::= SEQUENCE
{
recordProperty [0] UTF8String OPTIONAL,
recordElements [1] SEQUENCE OF RecordElement OPTIONAL,
recordSubExtensions [2] SEQUENCE OF GenericRecordExtension OPTIONAL
}

RecordElement ::= SEQUENCE
{
varName [0] UTF8String OPTIONAL,
varValue [1] UTF8String OPTIONAL
}
