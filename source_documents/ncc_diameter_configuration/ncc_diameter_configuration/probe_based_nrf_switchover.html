Configuration

By default, NRF server switchover is based on NF heartbeat failures. To configure this switchover to be based on probe failure, the following configurations must be done:

Helm parameters

The following parameters must be enabled (set to true). These are not available in the GUI and can be updated in the Helm chart:

  • charging.enableVfqdnResolution
  • centralmanagement.enableVfqdnResolution
  • provisioning.enableVfqdnResolution

These parameters are described within the Helm chart in the Readme file. To enable the DNS-based resolution of vFQDNs for NRF, configure the parameters in the values.yaml. You can configure the parameters during installation when you prepare the input for the Helm charts or after you installed NCC using helm upgrade command. By default, these parameters are disabled (false).

Note:
Helm upgrade requires a server restart. Contact Nokia support before planning for it.

Application preference

ME application preference NRF: Switchover on Probe Failure must be set to true. For more details, see Charging application preferences in Global Configuration Guide.

NRF server configuration

The following parameters are available for configuring the probe in NRF server:

  • Timer(s)
  • Success Count
  • Failure Count
  • FQDN-Probe NF Instance ID
  • Additional Success Codes for Switchback to Primary

Refer to NRF server attributes for details about these parameters.
