Realm based routes

To provision realm based routes from SM for last used Drouter and alternate Drouters for routing Diameter outbound messages (RAR, ASR, SNR), in SM GUI click Connectivity → ☰ → DIAMETER ROUTES → Realm Based Routes. Following figure has sample values: Click + to create a new realm-based route.

Figure: Realm based route

Here, the Next Hop field value is the FQDN of Drouter. Destination Realm is the realm of p-gateway.

All routes for alternate Drouters must be of the same priority as that of primary route (route for last used Drouter saved in the corresponding session).

The realm based route shown in the preceding figure is applicable for RAR and ASR over Gy interface. It is possible to use the same route for SNR over Sy interface or you can have a different route for SNR messages when Drouter used is different for Sy session.

Configure the Diameter peer for Drouters to use Diameter routes for primary and alternate Drouters. Diameter peers can be static or dynamic.

Use the Realm Based Routes menu item in the Routing tab to provision a list of Diameter destination routes.

All Diameter-related configuration areas can also be configured using APIs.

For more information, see the Realm Based Routing parameters.

The following table describes the realm-based routing parameters:

Table: Realm Based Routing parameters

Parameter

Description

Mandatory / Optional
Route Id

Specify a unique identifier for the route that can be used to distinguish between multiple routes with the same destination identity name but different priorities.

NCC can create this id automatically if no id is provided. However, if multiple MEs are registered at the SM, the SM sends this data with id created at the SM side to track the provisioning status on each ME.

If the route name is not specified, it is auto-generated as a concatenation of the values in Next Hop, Realm Based Routes, and Application.

Mandatory
Description Specify a description for the diameter route. Optional
Next Hop

Specify the hostname of the server to which NCC has to forward the incoming message.

The server has to be present in the realm configured in the Realm Based Routes.

Mandatory
Destination Realm

For Diameter Destination Routes, specify the destination realm which has the network entity where NCC has to forward the incoming requests from the particular peer origin host.

The asterisk * can be used as a wildcard to use for matching multiple routes for patterns. NCC uses the highest priority route that matches the pattern.

 Mandatory
Application

Specify a list of one or more application types to reach a specified host or realm. Select an application type from the drop-down to add it to the list.

By default all applications are valid for a route. Keep the field blank to enable all applications.

 Mandatory
Priority Specify the diameter route priority. Mandatory
