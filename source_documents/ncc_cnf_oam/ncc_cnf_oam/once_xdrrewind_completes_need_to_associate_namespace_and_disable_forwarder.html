Associate namespace and disable forwarder

Once xdr-rewind completes, the namespace must be associated and the forwarder disabled. To do so, run the following commands on all ME of pair/quad, except for the recovering site.

Need to run on all me of pair/quad except recovering site

kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -a add -s <recovering_sitename> # (This will associate the namespace and disable forwarder)
# Type y or yes and press enter when it ask for confirmation “Do you want to continue?: y|yes|n|no”
# Verify result should be 'ok' for every DB pod, if you got 'error' retry command again and check all DB pods are running or not.

# verify current association, enabled should be true
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -g -s <recovering_site_name>
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
  • <recovering_site_name> is the name of the recovering site.
