ME systems connected to SM/IG 3

  1. NCC is at Helm revision 2.
  2. For CGF (DR) chart (Helm releasename=dr), update the Inputs.xlsx file with these values and regenerate Helm overrides:
    1. Deployment Type = in-service upgrade in the Common tab's Global section.
    2. Reimport DR realm = true in ME 3 tab's DR section.
    3. Enable SSO = true in the Common tab's Global section.
  3. Helm upgrade NCC:
    1. CGF (DR) chart (Helm releasename=dr) will reimport User and Role Management realm with SSO changes. When User and Role Management is down, the CGF (DR) dr-ui pods try to reach User and Role Management and crashloopback after seven failed attempts. If that happens, delete and recreate those dr-ui pods.
  4. NCC is at Helm revision 3.
Note:
  • These items are impacted when the Servicemanager is inaccessible.
    • All provisioning traffic.
    • Servicemanager related REST API calls.
    • Servicemanager GUI access.
    • SS7 USSD requests are impacted because of its dependence on Servicemanager's service name.
    • SS7 communicates with Servicemanagers through its service name since performance degradation happens when primary and secondary Servicemanager FQDNs are used.
  • When the User and Role Management GUI is down, authentication requests and all GUIs are inaccessible.
  • When User and Role Management is down, CGF (DR)'s dr-ui pods try to reach User and Role Management and crashloopback after seven failed attempts. If that happens, delete and recreate those dr-ui pods.
  • When Reimport DR realm = true and Deployment Type = in-service upgrade in the Inputs.xlsx file, CGF (DR) chart (release name=dr) reupload DR realm to User and Role Management when an in-service upgrade is done. To avoid reimports in future in-service upgrades, set Reimport DR realm = false.
  • When Deployment Type = in-service upgrade in the Inputs.xlsx file, backup cronjobs are enabled for the MariaDB for User and Role Management.

The process to upgrade all sites to SSO is now complete. To optionally import the roles that were exported previously, see Import the exported roles.

If you need to rollback now, skip to Rollback after maintenance window 7.

Verify MariaDB replication status on all SM/IG systems. All of the SM/IG systems should be replicating with each other and have SSO authentication data.

It might take a few minutes for the replication status to show updated results.
