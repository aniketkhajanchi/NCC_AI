Backup and Recovery

The Backup and Recovery feature is introduced in the Performance suite. Backup is executed only once after a fresh installation during a Performance suite execution, and Restore is an on-demand feature. Restore can be run by enabling the ondemand_restore flag in the parameters.yaml file. For more information on Backup and Recovery parameters, see Test execution variables.

The following procedures describe how to first back up the system, and then restore it with on-demand restore. The max_reruns parameter must be set to 0, and it is assumed that Backup and Recovery parameters have been set in parameters.yaml according your system.

To back up the system

  1. Ensure that ondemand_restore is set to false (the default value).
  2. Run the Performance suite.
  3. After the test suite completes, a backup has been created on the SM/ME admincli pod at /opt/tpa/users/tpaadmin.

To restore the system

  1. Before running the Performance suite, set ondemand_restore to true, keeping the rest of the configuration as is.

  2. Run the Performance suite.

    Restore is executed as the first test case in the suite, thereby restoring the system. Restore is done from the backup generated in To back up the system.

  3. After the test suite completes, reset ondemand_restore to false to avoid running restore in the following test suite runs.
