IP addresses in OCSChargingRecords

In the OCSChargingRecords, the IP address input is always surrounded by square brackets. The value inside can be either a single IP address or a list of IP addresses separated by a commas. Every IP address entry has a prefix indicating whether the address is of type IPv4 or IPv6. For example,

  • [IPV4: 192.10.2.2]
  • [IPV6: 2001:db8:a0b:12f0::1]
  • [IPV4: 10.2.158.234, IPV4: 10.2.158.235, IPV4: 10.2.158.236]

When the input is an IPv4 address, convert the IP address to octet string and store it in output field <parent>.iPBinV4Address.

When the input is an IPv6 address, convert the IP address to octet string and store it in output field: <parent>.iPBinV6Address.
