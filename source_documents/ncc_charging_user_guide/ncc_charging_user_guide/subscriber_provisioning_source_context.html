SUBSCRIBER_PROVISIONING source context

Table: SUBSCRIBER_PROVISIONING source context
Attribute name Trigger Description
Method-Name SUBSCRIBER_PRE_PROVISIONING This attribute reads the invoke method of provisioning request and stores in it.

Example: Method name for create Account API is create/Account and for create Profile API is create/Profile.

Examples of method names used with gifting microservice are giftsubscription, gifting/adjustbalance and gifting/Subscription.
SUBSCRIBER_POST_PROVISIONING

Failed-Result SUBSCRIBER_FAILED_PROVISIONING This attribute is used to specify the reason for failure based on the identified error code.
Examples of enum values mapped in case of an error identified during a create subscription request or the gifting MS execution:
  • Beneficiary_Loan: This is identified by gifting MS when querying the beneficiary information. If there is an active loan with remaining debt by the beneficiary, gifting MS raises the error.
  • Beneficiary_Barred: This is identified by gifting MS when querying the beneficiary information, preventing the flow to continue before the subscription API fails. Gifting MS raises the error.
  • Bundle_Not_Exists: Identified by gifting MS when querying bundle information.
  • Beneficiary_Not_Exists: Identified by gifting MS when querying the beneficiary information.
  • Internal_Error: Any other type of error is mapped to this enum enabling a generic notification about the error in the service.
  • Account_Not_Enough_Funds: Identified when Adjustment API used to deduct for the bundle fee.
  • Adjustment_Error: Generic error in Adjustment API for non-mapped events.
  • Account_Barred: Not applicable in case of Gifting MS given that the validation of beneficiary Account status is done before the Adjustment API.
  • Device_Barred: Subscription is attached, though.
  • Device_Final_State: Subscription is not attached.
  • Group_Barred: Subscription is attached, though.
  • Group_Final_State: Subscription is not attached.
  • Subscription_Error: Generic error in create/Subscription API for non-mapped events.
