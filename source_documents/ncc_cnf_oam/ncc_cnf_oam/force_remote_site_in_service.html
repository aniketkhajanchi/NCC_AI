force-remote-site-in-service

Description

This command tool can be used to treat another member in a heartbeat community as in service so that processing on behalf of that member will stop. The tool overrides the heartbeat decision for the member that the tool is run on, which continues until a direct heartbeat is received from the errant member.

This tool should only be used in network deployments that have at least four NCC systems across at least two sites. From the member that sees the remote member as OOS, log in to the admincli pod and run the command. The system_ID in the topology of the network is used as the key to turn on and off the forced in-service state.
Note:
The tool must be run on all NCC systems.

Syntax

force-remote-site-in-service [-f <system_ID>] [-i] [-c <system_ID>] [-h]
Option Description
-h or -? Displays usage/help information.
-f <system_ID>[:<subsystem>]

Sets the system to in-service from the other system. Applies to the specified subsystem (sm or me) or to all subsystems in the system.

For example, it sets the ME subsystem inside IG 2 to in-service, from IG 1.

force-remote-site-in-service -f IG2:me

The ME subsystem in IG2 is forced in service. Further, to the example,

-f IG2 - without the :<subsystem> all subsystems in IG2

or

-f IG2:sm - with the :sm impacts just the SM subsystem of IG2

or

-f IG2:me - with the :me impacts just the ME subsystem of IG2
-i

Displays a list of members that have been forced to an in-service state. For example:

force-remote-site-in-service -i

These subsystems are forced in service:

2 ME

2 SM
-c <system_ID>[:<subsystem>]

Manually clears the forced in-service state for a member’s subsystem. Applies to the specified subsystem (SM or ME) or to all subsystems in that system. For example:

> force-remote-site-in-service -c IG2:sm

The ME subsystem in IG2 is no longer forced in-service.

Further, to the example,

-c IG2 - without the :<subsystem> impacts all subsystems in IG2

or

-c IG2:sm - with the :sm impacts just the SM subsystem in IG2

or

-c IG2:me - with the :me impacts just the ME subsystem in IG2
