Resolution

The NCC site must be restored including Kubernetes objects and data. This includes NCC with other dependent components. The installation needs to be done again and database should be restored. For the reinstall, original release names and overrides should be used. Then, data can be restored from NCC Backup and Recovery.

In the case of a complete recovery with all components, the Kubernetes cluster should be at a point where a clean NCC installation may be completed. In the case of an individual component recovery, it is good enough to follow the disaster recovery procedure specific to that component. For more information regarding the installation of components, see the Installation and Upgrade Guide.
