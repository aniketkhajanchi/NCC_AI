Realm settings

Purpose

Realm Selection on the upper-left drop-down menu allows the selection of the realm to be managed. The sps-auth realm is the only realm used in NCC, and it is created with SSL automatically configured on it by the installation scripts.

Realm

A predefined realm is created when the User and Role Management server is booted for the first time. This initial realm is referred to as the master realm or the head of all realms. The administrators of the master realm have the permissions to view and manage any other realm created on the server instance. When the initial administrator account is defined, an account must be created in the master realm. The initial login to the User and Role Management GUI is initiated through the master realm.

The realm has a dedicated admin console that can be accessed from the URL: /auth/admin/<realm-name>/console. The users within that realm can be granted realm management permissions by assigning specific user role mappings.
Note:
: Do not use the master realm to manage the users and applications in an organization. The master realm should only be preserved for super administrator tasks such as creating or managing realms in the system.

The NCC installation scripts create a realm called sps-authalong with associated client configurations for communicating with the local SM application.

SSL mode

The realm has an SSL mode associated with it, which defines the SSL or HTTPS requirements for interacting with the realm. The web browsers and applications that interact with the realm must comply with the SSL or HTTPS requirements defined by the SSL mode. SSL is configured by default during installation.
