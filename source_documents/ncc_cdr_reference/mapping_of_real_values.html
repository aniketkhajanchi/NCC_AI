Mapping of Real values

Output fields of type Real in the ASN.1 schema must be converted in the following way for them to appear correctly in the binary ASN.1 file:

First, the value must be a number in decimal format, with either a dot or a comma as separator. The value can also have a plus or minus sign in front of it. The value is then treated as a string value and converted to octet string. For example, value -123.56 becomes octet string 2d3132332e3536.

Valid string characters are:

  • numbers 0-9 (as octet: 30-39)
  • dot “.” (as octet: 2e)
  • comma “,” (as octet: 2c)
  • plus “+” (as octet: 2b)
  • minus “-“ (as octet: 2d)

After encoding the value as octet string, an extra octet with fixed value 02 must be added in front of the value. This indicates that the Real value is a base 10 encoded value that uses the NR2 formatting described in ISO 6093.
