Creating Jenkins slave service, virtual service, gateway and destination rule

This procedure describes how to create YAML files for Jenkins slave service, virtual service, gateway and destination rule on OpenShift with custom SCC and service account.

  1. Create (and navigate to) a jenkins-slave directory: 
    mkdir jenkins-slave
cd jenkins-slave
  2. Create a jenkins-slave.yaml file for Jenkins slave service, with the given content: 
    cat <<EOF > jenkins-slave.yaml
apiVersion: v1
kind: Service
metadata:
  labels:
    jenkins: slave
  name: cctf-jenkins-slave
  namespace: <NAMESPACE>
spec:
  type: ClusterIP
  ports:
  - name: slavelistener
    port: 11200
    protocol: TCP
    targetPort: 11200
  selector:
    jenkins: slave
EOF
  3. Create a jenkins-slave-gw.yaml file for gateway, with the given content: 
    cat <<EOF > jenkins-slave-gw.yaml
apiVersion: networking.istio.io/v1beta1
kind: Gateway
metadata:
  name: jenkins-slave-gw
  namespace: <NAMESPACE>
spec:
  selector:
    istio: cctf
  servers:
  - hosts:
    - '*'
    port:
      name: tcp-slave
      number: 11200
      protocol: TCP
EOF
  4. Create a jenkins-slave-vs.yaml file for virtual service, with the given content: 
    cat <<EOF > jenkins-slave-vs.yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: jenkins-slave
  namespace: <NAMESPACE>
spec:
  gateways:
  - jenkins-slave-gw
  hosts:
  - '*'
  tcp:
  - match:
    - port: 11200
      - '*'
    route:
    - destination:
        host: cctf-jenkins-slave
        port:
          number: 11200
  5. Create a jenkins-slave-dr.yaml file for destination rule, with the given content: 
    cat <<EOF > jenkins-slave-dr.yaml
apiVersion: networking.istio.io/v1beta1
kind: DestinationRule
metadata:
  name: jenkins-slave
  namespace: <NAMESPACE>
spec:
  host: jenkins-slave.cctf.svc.cluster.local
  trafficPolicy:
    tls:
      mode: DISABLE
EOF
  6. Replace <NAMESPACE> with the real namespace: 
    find . -type f -print | xargs grep -l "<NAMESPACE>" | xargs sed -i "s/<NAMESPACE>/$NAMESPACE/g"
  7. Apply the values defined in the YAML files: 
    oc apply -f jenkins-slave.yaml
oc apply -f jenkins-slave-gw.yaml
oc apply -f jenkins-slave-vs.yaml
oc apply -f jenkins-slave-dr.yaml
  8. Add port 11200 to the istio-ingressgateway service: 
    kubectl edit svc -n cctf istio-ingressgateway

...
  - name: tcp-jenkins
    port: 11200
    protocol: TCP
    targetPort: 11200
...
  9. Create service entries for SM and SE sites:
    Note:
    Replace <NAMESPACE> and <xx-site-FQDN> with the actual values.
    cat << EOF | kubectl apply -f -
apiVersion: networking.istio.io/v1beta1
kind: ServiceEntry
metadata:
  name: ext-me-se-443-3868
  namespace: <NAMESPACE>
spec:
  hosts:
  - <me-site-FQDN>
  ports:
  - name: https-443
    number: 443
    protocol: https
  - name: tcp-3868
    number: 3868
    protocol: tcp
  location: MESH_EXTERNAL
  resolution: DNS
EOF

    where port 3868 is the default port of the 4G test server. If needed, you can define additional ports.

    cat << EOF | kubectl apply -f -
apiVersion: networking.istio.io/v1beta1
kind: ServiceEntry
metadata:
  name: ext-sm-se-443
  namespace: <NAMESPACE>
spec:
  hosts:
  - <sm-site-FQDN>
  ports:
  - name: https-443
    number: 443
    protocol: https
  location: MESH_EXTERNAL
  resolution: DNS
EOF

    cat << EOF | kubectl apply -f -
apiVersion: networking.istio.io/v1beta1
kind: ServiceEntry
metadata:
  name: ext-ssh-se-22
  namespace: <NAMESPACE>
spec:
  addresses:
  - <jumphost_IP-CIDR>
  hosts:
  - "any"
  ports:
  - name: ssh-22
    number: 22
    protocol: tcp
  location: MESH_EXTERNAL
  resolution: NONE
EOF
  10. Check that the istio-ingressgateway service is running: 
    kubectl get svc -n cctf istio-ingressgateway
