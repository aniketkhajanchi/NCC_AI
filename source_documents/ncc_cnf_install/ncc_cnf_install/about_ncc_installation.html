About NCC installation

The NCC installation procedures assume a Kubernetes platform is already present and does not cover setting one up.

The high-level process for doing the NCC installation includes:

  • Obtain the Input Generator tool container image that assists with preparing Helm inputs.
  • Obtain the Dimensioning spreadsheet that is prepared by the Nokia sales team.
  • Plan the network deployment sites and options in the Input Generator tool spreadsheet.
  • Generate the Helm chart inputs with the Input Generator tool container image, using the Input Generator tool spreadsheet and Dimensioning spreadsheet to drive the process.
  • Prepare the Kubernetes namespaces.
  • Deploy the Helm charts. Multiple mechanisms exist to help with executing the necessary Helm commands.
  • Validate the installation.
This guide provides instructions for these parts of installation:
  • Planning tools
  • SM, ME, or IG installation

This guide does not provide instructions on how to install Kubernetes. Before you install NCC, Kubernetes must be installed in a cloud service such as Nokia Container Services (NCS), OpenShift, or Tanzu, or in any other cloud service. For more information on NCS, see the NCS documentation on Nokia Discovery Center (https://doc.networks.nokia.com/default).
