Format of the remote file

The Reprocessor stream aggregates rejected records into batches and stores each batch as a separate file on the SFTP server. The number of records in a batch is defined with CG Rejection Distributor Node parameter RecordsPerBatch. In addition to the count rule, a (potentially smaller) file is generated after a specified timeout since receiving the first rejected record. The timeout is specified with node parameter SecondsPerBatch.

Inside the files, records are stored using the format in which they were received by the Business logic stream. The files can be stored either as they are or in compressed format on the target host. For more information, see the Operation, Administration and Maintenance Guide in the NCC documentation set.
