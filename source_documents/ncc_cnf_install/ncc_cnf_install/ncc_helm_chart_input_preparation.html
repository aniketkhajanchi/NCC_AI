Generating Helm charts input

This procedure provides instructions on how to prepare the values for the Helm charts for your deployment.

Note:
To perform this procedure, your system must have Docker installed.

The deployment is documented in the Inputs.xlsx spreadsheet file, which is included in the input generator chart. For more information about the input generator and the structure of the Inputs.xlsx file, see Input generator.

These are the general usage commands for this procedure:
# Create an instance of the container
docker run
  # Execute as the local user so the generated file permissions match (omit if running as root or using podman)
  -u="${UID}:$(id -g)"
  # Mount a local directory with the input-generator tool
  -v $(realpath .):/input-generator:z
  # Remove the container instance when the command completes
  --rm
  # Location of the docker image
  ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION}
  # Command to run.  If not specified the default is a bash shell.
  # add "-it" after "docker run" and it will be an interactive command line
  /input-generator/generate.py

Additional utilities are available in the /input-generator/staticfiles directory. To use a utility command, append that utility command at the end of the docker run command.

Example:

To run inside the container the command:
./staticfiles/estimate-resources.py -i inputs.yaml
Enter:
docker run -u="${UID}:$(id -g)" -v $(realpath .):/work:z --rm ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION} ./staticfiles/estimate-resources.py -i inputs.yaml

  1. Ensure that your system has Docker installed.
  2. Set up the variables as follows:
    • Set the chf solution version.

      Example:

      CHART_VERSION=21.9.1-55
    • Set the Docker repository.

      Example:

      DOCKER_REPO=sps-docker-candidates.repo.lab.pl.alcatel-lucent.com
 
LOCAL_INPUTS_EXCEL=/path/to/inputs/InputsXXX.xlsx
LOCAL_INPUTS_OVERRIDES=/path/to/inputs/overrides/
  3. Run docker login <registry> and provide username and password.
  4. Extract the Input generator from the Docker Image by entering: 
    # Create a working directory on your local drive and change into it:
mkdir work
cd work
# $(realpath .) is getting the absolute path to the current directory.
docker run -u="${UID}:$(id -g)" -v $(realpath .):/work:z --rm ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION} /bin/sh -c "cp -r /input-generator/ /work"
  5. Migrate an older spreadsheet to the current version by entering: 
    # Run these commands from within the input-generator tool directory:
cd input-generator/
 
# Copy in the input files:
cp ${LOCAL_INPUTS_EXCEL} InputsExternal.xlsx
 
# Copy the unaltered product Excel sheet:
cp Inputs.xlsx Inputs${CHART_VERSION}.xlsx
 
# Run the migration tool:
docker run -u="${UID}:$(id -g)" -v $(realpath .):/input-generator:z --rm ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION} /input-generator/staticfiles/migrate.py -o InputsExternal.xlsx -n /input-generator/Inputs${CHART_VERSION}.xlsx 
 
# Copy the updated Excel into the location of your current Excel:
LOCAL_INPUTS_EXCEL=$(realpath Inputs${CHART_VERSION}.xlsx)
  6. Generate Helm inputs by entering: 
    # Run these commands from within the input-generator tool directory:
cd input-generator/
 
# Copy in the Excel and overrides:
cp ${LOCAL_INPUTS_EXCEL} InputsExternal.xlsx
cp -a ${LOCAL_INPUTS_OVERRIDES}/. overrides/
 
# Generate the Helm inputs:
docker run -u="${UID}:$(id -g)" -v $(realpath .):/input-generator:z --rm ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION} /input-generator/generate.py --inputs InputsExternal.xlsx
# Add "--delete-output" at the end if you want to automatically delete the output directory when it exists (for automation).
# docker run -u="${UID}:$(id -g)" -v $(realpath .):/input-generator:z --rm ${DOCKER_REPO}/chf/input-generator:${CHART_VERSION} /input-generator/generate.py --inputs InputsExternal.xlsx --delete-output
 
# Now you can see the generated files on the host machine:
ls output
    Note:
    The optional argument --inputs FILENAME.xlsx allows you to specify a location other than the input-generator folder for the Inputs${CHART_VERSION}.xlsx file.
    Expected output
    • Created the output directory with subdirectories for each system in the deployment
    • Based on the input provided in the Inputs.xlsx spreadsheet, each subdirectory contains:
      • site.conf file that describes a particular system
      • all the YAML override files needed to deploy that system
      • install scripts to Helm install releases
  7. Run this command to create the required PSMF TLS certificate for the SS7 GUI: 
    kubectl create -n <namespace> secret generic ss7psmf-cert --from-file=cacert=ca-key.pem --from-file=ca-cert.pem=ca-cert.pem

    Where:

    • <namespace> is the location where SS7 charts will be deployed.
    • ss7psmf-cert is the secret name that is used in the PSMF chart. This value is fixed and cannot be changed.
    • --from-file=cacert is the ca-key.pem file input.
    • --from-file=ca-cert.pem is the ca-cert.pem file input.
  8. Run this command to create the required DH TLS certificate for the SS7dh chart: 
    kubectl create secret generic ss7dh-cert --from- file=cacert=ca-key.pem --from-file=ca-cert.pem --from-file=ca-crl.pem --from-file=key.pem --from-file=cert.pem -n <namespace>

    Where:

    • <namespace> is the location where SS7 charts will be deployed.
    • ss7dh-cert is the secret name used in the ss7dh chart. This value is fixed and cannot be changed.
    • --from-file=cacert is the ca-key.pem file input files shared by Nokia.
    • --from-file=ca-cert.pem is the ca-cert.pem file input shared by Nokia.
    • --from-file=ca-crl.pem is the ca-crl.pem file input default file
    • --from-file=key.pem is the key.pem file input default file.
    • --from-file=cert.pem is the cert.pem file input default file.
  9. Run this command to create the required SOAP TLS certificate for the ss7soap chart: 
    kubectl create secret generic ss7soap-cert --from- file=cacert=ca-key.pem --from-file=ca-cert.pem --from-file=ca-crl.pem --from-file=key.pem --from-file=cert.pem --from-file=ca2-cert.pem -n <namespace>

    Where:

    • <namespace> is the location where SS7 charts will be deployed.
    • ss7soap-cert is the secret name used in the ss7soap chart. This value is fixed and cannot be changed.
    • --from-file=cacert is the ca-key.pem file input files shared by Nokia.
    • --from-file=ca-cert.pem is the ca-cert.pem file input shared by Nokia.
    • --from-file=ca-crl.pem is the ca-crl.pem file input default file
    • --from-file=key.pem is the key.pem file input default file.
    • --from-file=cert.pem is the cert.pem file input default file.
    • --from-file=ca2-cert.pem is the ca2-cert.pem file input default file.
  10. Update the license files for EM

    EM requires license files to run and perform its functions. You must copy the license files to a specific location so the ss7em Helm chart can export them. Perform these steps to copy the license files to the EM pod:

    1. Run this command to create a directory: 
      mkdir -p license_dir
    2. Run this command to copy the files: 
      cp lic_file  limits_file license_dir/
    3. Run this command to create the config map: 
      kubectl create configmap sigtran-lic-config -n sps-ig  --from-file=license_dir
      Note:
      The configmap must be named sigtran-lic-config.
