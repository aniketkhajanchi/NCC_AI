Date conversion to compact UTCTime format

Unless differently stated in the mapping rule, the input is a string of format “DD/MM/YYYY hh:mm:ss”, given in local system time.

The timestamp is converted into UTCTime format containing local time plus an offset to universal time: “YYMMDDhhmmssShhmm”. The “local time” in this case is indicated by the offset configured in CG Mapper Node with node parameter TimeZoneDesignator.

The converted value is then converted to octet string, where the digits are stored as binary coded decimals (BCD) to reduce the storage and transmission overhead and the sign for offset converted from ASCII to octet string.

Description Values Encoding
YY Year 00-99 BCD
MM Month 01-12 BCD
DD Day 01-31 BCD
hh Hours 00-23 BCD
mm Minutes 00-59 BCD
ss Seconds 00-59 BCD
S Sign “+” or “-“ ASCII
hh UTC time offset, hours 00-23 BCD
mm UTC time offset, minutes 00-59 BCD
Note:
Time zone is taken from CG Mapper Node's node parameter TimeZoneDesignator, if it has been set. Otherwise it is calculated from the system time zone.

For example, time 25/10/2017 11:30:44 in time zone +02:00 would result in the following octet string if stored with the same offset: 0x1710251130442b0200.

The same input time 25/10/2017 11:30:44 in time zone +02:00 would result in the following octet string if stored directly in UTC time: 0x1710250930442b0000.

Local time includes Daylight Saving Time (DST) depending on the country. DST is applied when system timezone is in use (node parameter TimeZoneDesignator is empty). If any value is defined for TimeZoneDesignator, DST is not supported.
