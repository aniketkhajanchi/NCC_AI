Installing NCC in a Kubernetes environment

This procedure describes the workflow of the installation of NCC in a customer’s Kubernetes environment. This installation applies to the deployments with a single IG, SM, or ME, or pairs of SMs, MEs, or IGs. This procedure supports 1+1 isolated and 1+1 networked deployment options.

If you need to uninstall Helm charts, use the procedure Uninstalling Helm charts.

  1. Retrieve NCC artifacts from NOLS using the procedure Retrieving NCC artifacts from the Support Portal.
  2. Upload every file downloaded from NOLS to your registry using the procedure Pushing images to a customer repository.
  3. Edit the Inputs${CHART_VERSION}.xlsx file using the procedure Preparing Helm chart input values.
  4. Prepare the input for the Helm charts using the procedure Generating Helm charts input.
  5. For each cluster, install an Istio service mesh.
  6. For each Kubernetes cluster, follow this procedure: Configuring a namespace.
  7. For each system, install an Istio Ingress controller.
  8. For each system, install the Helm charts as follows:
    • To install the Helm charts using a custom script generated by the input generator, follow this procedure: Installing Helm charts with a custom script.
    • To install the Helm charts manually, use the procedure Installing Helm charts manually.
    • To install the Helm charts by using Nokia Cloud Operations Manager (NCOM), use the procedure Installing Helm charts with NCOM.
  9. For each system, validate the installation by using the procedure Validating installation of an SM, ME, or IG.
  10. Optionally, if you decide to use the new versioning schema of mapping tables, follow this procedure: Using new complex map versioning.
  11. Integrate SS7 with SM GUI using information in section NCC SS7 integration with SM GUI.
