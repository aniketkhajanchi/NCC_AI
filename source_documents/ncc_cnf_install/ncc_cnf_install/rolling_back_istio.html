Rolling back Istio to the previous version

Note:
During rollback, traffic may be impacted. You may want to divert all of the traffic on the corresponding geographic site to prevent interruption.

  1. Perform the Helm rollback commands or use any orchestrator tool to rollback the Istio charts to the previous version in the reverse order in which the charts were upgraded.
    If you used the install-charts.sh to upgrade Istio, state files of the upgrade were created in the input-generator directory. These files are used for rollback order of charts.
  2. Restart the application pods once they are running normally.
    Application pods with Istio sidecar installed must be restarted to roll back the sidecar Istio version.

    1. Navigate to the Pod Restart section of each namespace worksheet in your Inputs.xlsx file.
    2. Edit the Restart Counter value in each worksheet and increase the value by one digit.
      For example: if the current value is restart1, change the value to restart2.
    3. Update the API Compatibility of each system to the previous version .
    4. Perform a backup of the state files.
    5. Regenerate the Helm inputs.
    6. Upgrade each Helm chart with the new inputs.
