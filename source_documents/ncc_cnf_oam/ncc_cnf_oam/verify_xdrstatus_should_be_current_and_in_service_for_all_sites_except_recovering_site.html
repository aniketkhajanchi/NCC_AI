Verify xdr-status is CURRENT and IN_SERVICE for all systems

Run this command to verify that xdr-status shows CURRENT.

# Both "local site towards remote sites" and "remote sites toward local site" should have sync up state CURRENT.
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/xdr-status
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
