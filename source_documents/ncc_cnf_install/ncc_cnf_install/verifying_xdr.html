Verifying XDR

This procedure provides instructions on what to do if after topology upgrade the XDR status does not show all the destination sites.

  1. Ensure the following:
    • The xdr-status tool shows the status of all remote sites as CURRENT for each CHF.
    • Both local site towards remote sites and remote sites toward local site have sync up state CURRENT, and the tool shows the MEs of the networked pair.
  2. Find the active centralmanagement pod by entering: 
    kubectl get pods -n <namespace> -l haActive=true | grep centralmanagement
  3. From the active centralmanagement pod, execute reloadDestinationSites.
  4. From the application_main_container of the active centralmanagement pod, execute the following: 
    kubectl exec -it <active_centralmanagement_pod> -n <namespace> -c <application_container> -- /opt/tpa/bin/twiddle localhost invoke com.nokia.tpapps.xdr.sourcesite:service=XDRMetricsQueryService reloadDestinationSites
