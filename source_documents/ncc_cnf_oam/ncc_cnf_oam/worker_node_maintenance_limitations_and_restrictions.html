Limitations and restrictions

  1. Daemonsets such as Node Exporter and Fluentd do not get evicted and are not made on other node.
  2. For a node containing cdr-server, expect one percent of the CDR to become lost if calls are running during the maintenance mode.
  3. If a node containing an active centralmanagement pod is taken out for maintenance, expect the sps-status to be down for that system for two to three seconds. If there is routing that is done through this system, the calls may also fail during this time.
  4. If a node contains the cpro-server, then expect to have metrics lost during the time that pod is down.
  5. If a node containing an active IOHD pod is taken out for maintenance, expect the sps-status to be down for that system for two to three seconds, which further results in call failures. If there is routing that is done through this system, the calls may also fail during this time.
