Configuring User and Role Management authorization

A JSON Web Token (JWT) must be generated on the User and Role Management FQDN, in order to use the Rest handler node. This token is used when any of the exposed APIs is called, by passing it in the authorization header.

Run this CURL command to generate the authorization token. 
curl --location --request POST 'https://vbox.nokia.com:8666/auth/realms/nokia/protocol/openid-connect/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'client_id=eventlink_ui' \
--data-urlencode 'grant_type=client_credentials' \
--data-urlencode 'client_secret=852bacc0-2ded-4da7-aca6-f0c374aa3a04'
