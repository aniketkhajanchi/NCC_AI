Enable forwarder and start xdr-rewind

Enable forwarder and start xdr-rewind for all ME of pair/quad other than the recovering site by issuing the following commands. In the case of CHF-R, only run the following commands once from a healthy CHF-R.

# To start xdr-rewind, execute the following command:
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -x start -s <recovering_sitename>          # if xdr-rewind fail, retry again
# Type y or yes and press enter when it ask for confirmation “Do you want to continue?: y|yes|n|no”
 
# Verify xdr-rewind status
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -x status -s <recovering_sitename>         # xdr-rewind status should be complete
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
  • <recovering_site_name> is the name of the recovering site.
Note:
If rewind fails, try one more time. If rewind fails again, restart from Disassociate the namespace from Data Center.
