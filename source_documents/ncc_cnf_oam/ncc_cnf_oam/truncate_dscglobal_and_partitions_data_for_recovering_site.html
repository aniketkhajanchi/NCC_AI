Truncate recovering system dscglobal and partitions data

The dcglobal and partition data should be truncated on the recovering system. To do so, run this command only on the recovering system.

# It will clear all partitions and dscglobal
kubectl -it exec <admincli> -c <container> -n <namespace> -- /opt/tpa/bin/disaster-recovery-helper.py -t 
# Type y or yes and press enter when it ask for confirmation “Do you want to continue?: y|yes|n|no”
Where:
  • <admincli> is the name of the admincli pod.
  • <container> is the name of the container.
  • <namespace> is the name of the namespace.
