Records rejected in ASN.1 Encoder Node

If a record cannot be encoded with the given ASN.1 schema in ASN.1 Encoder Node, it is rejected and stored within the rejected storage of Charging Gateway Function. The rejected storage is located on the pod where the specific output stream is running, under directory /opt/comptel/eventlink/base/status/streams/<streamName>/nodes/VZ_SPS_CFG_ENCODER/rejected/DEFAULT/. The rejected records are stored in sub-directories based on rejection time.

A file may be examined with rejected records by running this command:

r_export <fileName>

The content of the records is shown in Charging Gateway Function internal format, including the reason for rejection at the end of every rejected record.
