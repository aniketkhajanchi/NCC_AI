Charge subscription renewal fee for missed periods

Subscription is renewed when BCR happens. If the account balance is not sufficient to deduct renewal fee, the renewal is failed, and subscription is moved to suspended state (state may change as per lifecycle configuration).
  • If a subscription is suspended for more than one recurrent period due to failed renewal (insufficient funds), then on the next recharge, renewal happens and if chargeMissedPeriod in Bundle is true, missed periods are charged starting from the time of last renewal depending upon the available account balance.
  • The period starts from the date of last renewal, when subscription was suspended. The EDRs are generated for each of the renewal period.
  • EDRs per period are generated, containing charged fee value and Start and End dates of the period.
  • Subscription resets the period cycle, and it is set to Active if all the missing fees are charged.
  • In case the recharge value is not enough to charge all the periods, partial charging is possible, and the End date is set accordingly.
Note:
The above functionality is applicable only to the subscriptions getting renewed by own lifecycle. This functionality is not applicable to subscriptions getting renewed by account periodic lifecycle.
Reset period action:
  • If chargeMissedPeriod is set to true, on account recharge, the start time is set to end time of the entity (instead of NOW). Hence, the renewal starts from the last end time when it failed. If renewal is successful, new start time and end time are set in the entity.
  • If the new end time is in the past, the RepeatCycle transition executes immediately. And it keeps repeating until the end time comes to be a future date on successful renewal of all missed periods + current period (OR) the renewal fails due to insufficient funds and entity moves to barred state.

Number of missing periods in response of REST APIs

If a subscription having missing periods whose corresponding bundle has chargeMissedPeriod as true is queried, then parameter missedPeriodCount containing the number of missing periods is obtained in the response.

Following are the APIs, which can be used for querying the subscription data:

  • getData/Device(with detailedQuery as TRUE/FALSE)
  • getData/Group (with detailedQuery as TRUE/FALSE)
  • getData/Account (with detailedQuery as TRUE)
  • getData/Subscription
  • getData/Profie

Single notification with cumulative fee

In order to have a cumulative fee for all the missing periods charged, whenever a renewal is processed, the fee can be added into totalFeeMissedPeriod. The total fee is available at the end of renewal of all cycles, when chargeMissedPeriod is true in Bundle and when the fee is successfully deducted from the account. This information is stored in subscription context, prepared while sending notification through SendNotificationAction in lifecycles.

The scenarios where single notification is required are:

  1. Subscription in suspended state and successful renewal of all missed periods occurs on account recharge.
  2. Subscription in suspended state and partial renewal of missed periods occurs.
  3. Subscription in frozen state and successful renewal of all missed periods occurs.
  4. Subscription in frozen state and partial renewal of missed periods occurs.
Note:
To get a cumulative fee charged of all the missing periods in a single notification, rules as mentioned in Lifecycle configuration must be configured in Send Notification Action.

Freeze/unfreeze

Frozen period should not be considered as missing period and must not be charged. There is no impact on freeze / unfreeze functionality. Integration with freeze / unfreeze functionality is achieved by Lifecycle configuration.

Note:
  • When ME application preference Subscription Creation On Insufficient Balance is set to ENABLED, subscription is barred at the time of creation due to insufficient funds. With recommended Lifecycle configuration, such subscription is activated for the first time through Renew Subscription Action on account recharge. To avoid charging of missing periods on activating such subscriptions, Reset Period Action must not be configured in Start Cycle transition, else all the missing periods are charged.
  • While charging missing periods, if missed period renewal time falls between the discount start and end time, then discount is valid and applicable.
