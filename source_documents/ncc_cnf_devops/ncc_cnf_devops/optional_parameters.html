Optional parameters

Optional parameters that can be used in any test cases on need basis are located in the tp_parameters section in the parameters.yaml file, including:

  • Flags specific for this test package used by CCTF.
  • SUT configuration/access data.
  • Other parameters that are needed by the test cases.

Users can define whatever parameters per their needs. A sub-section named cctf (reserved for CCTF to read other configuration information) is also located in the tp_parameters section.

health_check

The health check is run in the pre-execution.sh hooks. It pings the target API to ensure a healthy connection. The health check passes if the number of requests specified in the required_success_tries parameter passes in one attempt. If an attempt is unsuccessful, the health check fails and no tests are executed.

Health check parameters:

  • run_health_check defines whether the health check is run (True/False)
  • required_success_tries defines the amount of requests performed in one run
  • sleep_between_tries defines the delay between subsequent requests
  • max_reattempts defines the maximum number of retries before the health check is aborted.

Example:

health_check:
  run_health_check: True
  required_success_tries: 30
  sleep_between_tries: 1
  max_reattempts: 5
