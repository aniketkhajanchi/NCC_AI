Configuring Alarm Service pod time zone

This procedure provides instructions on how to configure the time zone for a Alarm Service pod by using a custom YAML file that is used to override the default Helm values.

  1. Create a custom YAML (timezone_calm.yaml) file in the input-generator/overrides directory.
    Sample content: 
    calm:
  global:
    timeZone:
      timeZoneEnv: 'America/Chicago'
  2. Open the Inputs.xlsx spreadsheet.
  3. Select the worksheet that corresponds to the namespace you want to update.
  4. Verify the Enabled value is set to true in the BTEL section.
  5. Enter the override file name (timezone_calm.yaml) in the Input > Overrides > Value field of the BTEL section.
    Note:
    The Overrides value is merged left to right, so right takes priority
  6. Update the Helm chart to apply the override.
