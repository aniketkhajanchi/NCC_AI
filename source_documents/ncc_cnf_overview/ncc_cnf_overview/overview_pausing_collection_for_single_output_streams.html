Pausing collection for a single output stream

Kafka output stream and File output stream both consist of multiple nodes and use file links for passing data though the nodes. These file links can become overloaded if some of the nodes are not able to process the data fast enough.

To prevent the overloading of file links, the first node in a stream (CG File Collector Node in File output stream or CG Collector Node in Kafka output stream) monitors the number of files in the link buffers of the subsequent nodes. The nodes stop collecting new data if the number of files in any file link buffer of that stream instance reaches the number defined in node parameter MaxFilesInLinkBuffer. The collection is resumed when all file link buffers of the stream are empty. Setting node parameter MaxFilesInLinkBuffer value to zero disables overload protection for file links.

To avoid problems caused by increasing stream link buffer sizes in the output stream, whether File or Kafka output stream, there is an option to temporarily stop collecting files when the number of files in any link buffer in the stream reaches a specified limit. The limit is defined with node parameter MaxFilesInLinkBuffer in CG File Collector Node/CG Collector Node.

The size of file link buffers is monitored for each stream instance independently. If one stream instance is paused because of file link overload, other instances of the stream continue operating normally if overload protection is not triggered for their file links.
