X-button UI

The X-button feature enables selecting a new UI type to run test cases with a defined number of buttons rather than to list each test case in detail after a test package is loaded on CCTF.

The X-buttons are labeled with the value of buttonLabel defined in cctf_buttons.json in the test package. These buttons trigger the execution of test groups for different purposes respectively.

The following figure presents an example of the X-button UI, with buttons labeled as REGRESSION, NEW FEATURE and PERFORMANCE. Each button is associated with a REPORT button for checking the reports from the latest execution

Figure: X-button UI

A test package uploaded to CCTF has either the normal UI or X-button UI, but not both. CCTF decides automatically which type of UI is available based on the analysis on the test package loaded (that is, whether cctf_buttons.json exists).

Note:
There is also a set of REST APIs for this feature, which could provide the same functionalities as the X-button UI. For security reasons, these REST APIs are turned off by default. For information on how to turn on the REST APIs, see Enabling X-button REST API.
