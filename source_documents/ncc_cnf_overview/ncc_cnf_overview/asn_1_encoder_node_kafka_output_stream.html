ASN.1 Encoder Node

ASN.1 Encoder Node converts the batch of output records into given ASN.1 format. By default, NCC generic ASN.1 schema is used. For more information on the schema, see the CDR/EDR Reference in the NCC documentation set.

For Kafka output stream processing, records must be encoded together with CDR headers, and the file header must be omitted. The default node parameter configuration defines the required headers to be included in the output. For more information about file and CDR headers, see 3GPP TS 32.297.

Rejected data storage

ASN.1 Encoder Node can reject records if there is a problem in encoding them in the given ASN.1 schema. If the node rejects a record, the record is stored in CGF Rejected Data storage area called DEFAULT. This area can be inspected at any time, and the records can be reprocessed either manually or via a schedule, while other options include taking the records offline or removing them.
