Disaster recovery for Monetization System

For network deployments that include more than one Monetization system, only one system is ACTIVE (primary) at a time. The other system is deemed as STANDBY and functions only as data storage. Switching from the ACTIVE Monetization to the STANDBY Monetization is a manual process. The ACTIVE Monetization has all the vital services deployed within it including the Kafka Mirror Maker and Rerate Manager services. However, the STANDBY Monetization does not include those services and they are scaled down to zero replicas automatically during the initial deployment. In case the ACTIVE Monetization goes down, you must preform these steps to manually switch the ACTIVE and STANDBY Monetization systems.

  1. Change the Primary EOC Site value to true in the SPS section of the STANDBY Monetization namespace worksheet in the Inputs.xlsx file.
  2. Change the Primary EOC Site value to false in the SPS section of the ACTIVE Monetization namespace worksheet in the Inputs.xlsx file.
  3. Generate new Helm override files by using the generate.py script.
  4. Perform a Helm upgrade on the Kafka Mirror Maker and Rerate Manager Helm charts to update the ACTIVE/STANDBY Monetization system.
