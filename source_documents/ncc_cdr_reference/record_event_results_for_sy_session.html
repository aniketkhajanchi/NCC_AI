Record event results for Sy session

Whenever a session/event request has occurred, it is either successful or unsuccessful depending upon the logic processing in NCC. The RecordEventResult tag contains the numeric number created by NCC explaining the cause of session/event closure.

Example: RecordEventResult = 21

This section contains the details about the values and corresponding description of the result code generated by the SY interface:

  • 00-20: Related to successful call
  • 21-40: Related to unsuccessful call
  • 300-320: Session audit result codes for successful/unsuccessful result due to different events. These are listed in Record event results for session audit CDRs
