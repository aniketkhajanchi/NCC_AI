Providing Helm chart inputs directly

This procedure provides general instructions on how to prepare a custom YAML file that is used to override any Helm values. For more instructions on how to create a custom YAML file to override particular Helm values, see Configuring Vendor Specific Resource Definition (VSRD) values by using a custom YAML file.

  1. Create a custom YAML file in the input-generator/overrides folder.
  2. Add to this file the parameters to override.
  3. Set these parameters to the new values.
  4. Enter the name of this file in the Inputs.xlsx file as follows:
    1. In the Inputs.xlsx file, click the worksheet that corresponds to the namespace for which you updated the values.
    2. In that tab, go to the section of the corresponding Helm chart.
    3. In that section, go to the Overrides row and enter the name of the YAML file as the value.
      Note:
      The Overrides value is a space/comma-separated list of override YAML files for the Helm chart. The value is merged left to right, so right takes priority.
