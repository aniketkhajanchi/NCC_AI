Provisioning EDRs

This chapter provides information about event data records (EDRs) generated for provisioning of entities in MEs.

Applicable application preferences

Configure the application preferences as suggested in the following table.
Note:
It is not recommended to use the Hierarchical EDR Flag the for provisioning EDRs.
Table: Application preference configuration for provisioning EDR
Application Preference Configure as: Description
EDR Generator Flag true Enable or disable provisioning EDR generation.

Valid values: boolean (true, false)

Default: false

Old and new EDRs for update action

For the following entities, EDRs generated for update operation include both old and new values of updated parameters:

  • Device

  • User

  • Group

  • Account

  • Usage control
  • Subscription

The format of the tags for the old and new parameters are old_<tagName> and <tagName>, respectively.

Example: old_subscriptionIndex and subscriptionIndex
